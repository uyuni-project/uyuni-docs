[[auth-methods-sso-example]]
= Example SSO Implementation
:description: Implement SSO by exposing endpoints with SUSE Multi-Linux Manager and using Keycloak as its identity service provider, enabling secure single sign-on
:revdate: 2026-06-24
:page-revdate: {revdate}
:page-author: SUSE Product & Solution Documentation Team
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

In this example, SSO is implemented by exposing three endpoints with {productname}, and using Keycloak 26.5.3 or later as the identity service provider (IdP).

To set up Single Sign-On (SSO) with {productname} and Keycloak, you first configure the Keycloak identity provider, obtain its public SAML certificate, and set up the client and users. After that, you configure the {productname} Server to trust Keycloak.

[WARNING]
====
This example is provided for illustrative purposes only.
{suse} does not recommend or support third-party identity service providers, and is not affiliated with Keycloak.
For Keycloak support, see https://www.keycloak.org/.
====

You can install Keycloak directly on your machine, or run it in a container.
In this example, we run Keycloak in a Podman container.
For more information about installing Keycloak, see the Keycloak documentation at https://www.keycloak.org/guides#getting-started.

[#how-to-set-up-keycloak]
== Setting up the identity service provider (Keycloak)

First, run Keycloak, create a realm, obtain the SAML public certificate, and configure the client.

[#pro-setting-up-keycloak]
.Procedure: Setting up Keycloak
[role=procedure]
____
. Install Keycloak in a Podman container, according to the Keycloak documentation.
. Run the container using the ``-td`` argument to ensure the process remains running:
+
----
podman run -td --name keycloak -p 8080:8080 -e KEYCLOAK_ADMIN=admin -e KEYCLOAK_ADMIN_PASSWORD=admin quay.io/keycloak/keycloak:26.1.1 start-dev
----
. Sign in to the Keycloak Admin Console as the ``admin`` user.
. Create an authentication realm:
* In the top-left corner, click the realm selector dropdown (showing ``master``) and select [guimenu]``Manage realms``.
* On the [guimenu]``Manage realms`` page, click the btn:[Create realm] button in the top-right corner.
* In the [guimenu]``Realm name`` field, enter a name (for example, ``MLM``).
* Click btn:[Create].
. Obtain the SAML public certificate:
* From the left navigation menu, click menu:Realm settings[].
* Click the [guimenu]``Keys`` tab.
* Find the active **RSA** key (with provider ``rsa-generated``) in the active keys table.
* Click the btn:[Certificate] button in that row.
* Copy the entire Base64-encoded certificate from the modal that opens. Keep this certificate handy for the server configuration.
. Add {productname} as a Client:
* From the left navigation menu, click menu:Clients[].
* Click btn:[Create client].
* In the [guimenu]``Client type`` field, select ``SAML``.
* In the [guimenu]``Client ID`` field, enter the {productname} Metadata endpoint.
  For example: ``https://<FQDN_MLM>/rhn/manager/sso/metadata``
* Click btn:[Next].
* In the [guimenu]``Valid redirect URIs`` field, enter the {productname} Assertion Consumer Service (ACS) endpoint.
  For example: ``https://<FQDN_MLM>/rhn/manager/sso/acs``
* Click btn:[Save].
. Fine-tune client settings:
* Under the [guimenu]``Settings`` tab, scroll down to the [guimenu]``SAML capabilities`` section:
** Toggle [guimenu]``Sign assertions`` to [guimenu]``On``.
** In the [guimenu]``Signature algorithm`` field, select ``RSA_SHA1``.
** In the [guimenu]``SAML Signature Key Name`` field, select ``Key ID``.
** Click **Save**.
* Under the [guimenu]``Keys`` tab:
** Toggle [guimenu]``Client signature required`` to [guimenu]``Off``.
* Under the [guimenu]``Advanced`` tab, scroll down to the [guimenu]``Fine Grain SAML Endpoint Configuration`` section:
** In both the [guimenu]``Assertion Consumer Service POST Binding URL`` and [guimenu]``Assertion Consumer Service Redirect Binding URL`` fields, enter the {productname} ACS endpoint.
  For example: ``https://<FQDN_MLM>/rhn/manager/sso/acs``
** In the [guimenu]``Logout Service POST Binding URL`` field, enter the {productname} SLS endpoint.
  For example: ``https://<FQDN_MLM>/rhn/manager/sso/sls``
** In the [guimenu]``Logout Service Redirect Binding URL`` field, enter the root URL of your {productname} instance.
  For example: ``https://<FQDN_MLM>/``
** Click **Save**.
. Configure Client Scope and Mappers:
* From the left navigation menu, click menu:Client scopes[].
* Click [guimenu]``role_list`` in the list.
* Click the [guimenu]``Mappers`` tab, and then click the [guimenu]``role list`` mapper.
* Toggle [guimenu]``Single Role Attribute`` to [guimenu]``On``, and click btn:[Save].
* Go back to menu:Clients[], click your {productname} client ID (``https://<FQDN_MLM>/rhn/manager/sso/metadata``), and click the [guimenu]``Client scopes`` tab.
* Click the dedicated scope (for example, ``https://<FQDN_MLM>/rhn/manager/sso/metadata-dedicated``).
* Click btn:[Configure a new mapper].
* Select [guimenu]``User Attribute`` from the list, and configure the mapper with these details:
** In the [guimenu]``Name`` field, enter ``uid``.
** In the [guimenu]``Property`` field, select ``username``.
** In the [guimenu]``SAML Attribute Name`` field, enter ``uid``.
* Click btn:[Save].
. Create Users:
* From the left navigation menu, click menu:Users[].
* Click btn:[Add user].
* In the [guimenu]``Username`` field, enter a username that matches an existing {productname} user (for example, the administrative user).
* Click btn:[Create].
* Click the [guimenu]``Credentials`` tab, click btn:[Set password], set a password for the user, and toggle [guimenu]``Temporary`` to [guimenu]``Off``, click btn:[Save] and confirm by clicking btn:[Save password].
____


[#how-to-configure-{productname}-server]
== Setting up the {productname} server

After Keycloak is ready, configure the {productname} Server to use single sign-on.

[IMPORTANT]
====
Ensure that the users whom you intend to authenticate using SSO already exist in {productname}. You can create users manually in the {productname} {webui}, or use the command-line interface or API (using PAM-enabled or passwordless authentication).
====

[#pro-setting-up-server]
.Procedure: Setting up the {productname} server
[role=procedure]
____
. On the {productname} Server, open the [path]``/etc/rhn/rhn.conf`` configuration file.
. Add the following parameters. Replace ``<FQDN_MLM>`` with the fully qualified domain name of your {productname} installation, ``<FQDN_IDP>`` with the FQDN of your Keycloak server, ``<REALM>`` with your realm name (for example, ``MLM``), and ``<CERTIFICATE>`` with the SAML public certificate Base64-encoded string obtained from Keycloak in the previous steps:
+
----
java.sso = true

java.sso.onelogin.saml2.debug = true

java.sso.onelogin.saml2.sp.entityid                       = https://<FQDN_MLM>/rhn/manager/sso/metadata
java.sso.onelogin.saml2.sp.assertion_consumer_service.url = https://<FQDN_MLM>/rhn/manager/sso/acs
java.sso.onelogin.saml2.sp.single_logout_service.url      = https://<FQDN_MLM>/rhn/manager/sso/sls

java.sso.onelogin.saml2.idp.entityid                   = http://<FQDN_IDP>:8080/realms/<REALM>
java.sso.onelogin.saml2.idp.single_sign_on_service.url = http://<FQDN_IDP>:8080/realms/<REALM>/protocol/saml
java.sso.onelogin.saml2.idp.single_logout_service.url  = http://<FQDN_IDP>:8080/realms/<REALM>/protocol/saml

java.sso.onelogin.saml2.idp.x509cert = -----BEGIN CERTIFICATE----- <CERTIFICATE> -----END CERTIFICATE-----

# Organization Metadata (Optional)
java.sso.onelogin.saml2.organization.name = SUSE Multi-Linux Manager
java.sso.onelogin.saml2.organization.displayname = SUSE Multi-Linux Manager
java.sso.onelogin.saml2.organization.url = https://<FQDN_MLM>
java.sso.onelogin.saml2.organization.lang =

# Contacts (Optional)
java.sso.onelogin.saml2.contacts.technical.given_name = SUSE Multi-Linux Manager Admin
java.sso.onelogin.saml2.contacts.technical.email_address = admin@example.com
java.sso.onelogin.saml2.contacts.support.given_name = SUSE Multi-Linux Manager Admin
java.sso.onelogin.saml2.contacts.support.email_address = admin@example.com
----
. Save your changes and restart the spacewalk service to apply them:
+
----
mgradm restart
----
. Navigate to the {productname} {webui}. You should be redirected to the Keycloak login screen, and successfully log in back to {productname} upon authentication.
____

[#how-to-troubleshoot-sso]
== Troubleshooting

If you encounter issues when authenticating with single sign-on, use these steps to diagnose:

* **Verify Logs:** Look for SSO errors in [path]``/var/log/rhn/rhn_web_ui.log``. Detailed logging is enabled when ``java.sso.onelogin.saml2.debug = true`` is set in [path]``/etc/rhn/rhn.conf``.
* **User Not Found:** If you see "User not found" or redirection fails after Keycloak authentication, ensure the username in Keycloak matches the username in {productname} exactly. Users must be pre-created in {productname}.
* **Signature and certificate errors:** If you see "Signature verification failed", verify that the certificate in ``java.sso.onelogin.saml2.idp.x509cert`` matches the active RSA certificate displayed under the [guimenu]``Keys`` tab in Keycloak Realm Settings, and ensure that [guimenu]``Sign assertions`` is toggled to [guimenu]``On`` in the client settings.
* **Invalid Redirect URI:** If Keycloak displays "Invalid parameter: redirect_uri" on redirection, verify that [guimenu]``Valid redirect URIs`` in client settings in Keycloak is set to ``https://<FQDN_MLM>/rhn/manager/sso/acs``.
