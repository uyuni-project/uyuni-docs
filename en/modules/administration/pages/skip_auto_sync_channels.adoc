[[workflow-autosync-channel]]
= Disable automatically Synchronize Channels
:revdate: 2026-05-08
:page-revdate: {revdate}

By default, channels are automatically synchronized with the upstream repositories to ensure that clients have access to the latest content and updates. However, in some cases, you may want to disable this automatic synchronization for specific channels.

This chapter describes how configure a channel to not be synchronized automatically.




[[get-channel-sync-satus]]
== Verify channel synchronization status

To check if a channel is set to be automatically synchronized user can call an API.

Here is an example of how to check the synchronization status of a channel using the API (replace `<Channel_Label>` with the actual label of the channel you want to check):

[source,console]
----
spacecmd api -- --args "<Channel_Label>" channel.software.isAutoSync
----

This information is also available in the Web UI.
To check the status, navigate to the channel details page and look for the field "Automatic Repository Synchronization".

[[set-channel-sync-satus]]
== Set a Channel to Skip Automatic Synchronization

To enable or disable automatic synchronization for a channel, you can use the API call `channel.software.isAutoSync`.

Here is an example of how to set a channel to skip automatic synchronization using the API (replace `<Channel_Label>` with the actual label of the channel you want to update):
[source,console]
----
spacecmd api -- --args '["<Channel_Label>", true] channel.software.isAutoSync
----

[[manual-channel-sync]]
== Manually Synchronize a Channel

Even if you have disabled automatic synchronization for a channel, you can still manually synchronize it whenever needed.

[IMPORTANT]
====
Use [literal]``mgrctl term`` before running steps inside the server container.
====

To manually synchronize a channel, you can use the command(replace `<Channel_Label>` with the actual label of the channel you want to synchronize):

[source,console]
----
spacewalk-repo-sync -c <Channel_Label>
----
