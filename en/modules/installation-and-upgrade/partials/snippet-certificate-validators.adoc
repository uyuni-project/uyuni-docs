[IMPORTANT]
====
Before upgrading, verify that every root and intermediate CA certificate used by {productname} marks `X509v3 Basic Constraints` as `critical` and includes `CA:TRUE`.
Strict certificate validators, including Python 3.13 and later, reject CA certificates that miss the `critical` flag.

If your installation uses an older self-signed CA generated by {productname}, you might need to create a new CA and new server certificates, then deploy the new root CA to all clients.

You can rotate the CA certificate on a Podman installation using the new `mgradm ssl` command. 
For more information, see xref:administration:ssl-certs-ca-migration.adoc[].

Note that the command can only be used *after* upgrading both `mgradm` and {productname}. 
You can safely upgrade affected installations as long as you rotate the CA certificates and ensure clients recognize them before starting the server.

Not rotating the CA certificate may not be fatal, but is heavily discouraged.
====
