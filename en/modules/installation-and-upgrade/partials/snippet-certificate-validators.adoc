[IMPORTANT]
====
Before upgrading, verify that every root and intermediate CA certificate used by {productname} marks [literal]``X509v3 Basic Constraints`` as [literal]``critical`` and includes [literal]``CA:TRUE``.
Strict certificate validators, including Python 3.13 and later, reject CA certificates that miss the [literal]``critical`` flag.

If your installation uses an older self-signed CA generated by {productname}, you might need to create a new CA and new server certificates, then deploy the new root CA to all clients.
For the verification command, certificate replacement, and client root-CA deployment procedure, see xref:administration:ssl-certs-imported.adoc#ssl-certs-verify-ca-basic-constraints[], xref:administration:ssl-certs-selfsigned.adoc#ssl-certs-selfsigned-create-replace[], and xref:administration:ssl-certs-imported.adoc#ssl-certs-import-deploy-root-ca[].
====
