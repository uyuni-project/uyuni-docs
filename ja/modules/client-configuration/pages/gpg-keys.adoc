= GPGキー
:description: 利用可能なGPGキーをリストし、新しいキーを作成して、クライアントのソフトウェアパッケージの認証を設定します。
:revdate: 2022-07-19
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメントチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

クライアントは、ソフトウェアパッケージがインストールされる前に、その真正性を確認するためにGPGキーを使用します。
信頼できるソフトウェアのみがクライアントにインストールできます。

ほとんどの場合、クライアントにソフトウェアをインストールできるようにGPG設定を調整する必要はありません。

RPMパッケージは直接署名できますが、Debianベースのシステムはメタデータのみを署名し、パッケージを保護するためにチェックサムのチェーンを使用します。
ほとんどのRPMベースのシステムは、署名されたパッケージに加えて署名されたメタデータも使用します。

Lists keys and certificates available for inclusion in {kickstart} profiles and provides a means to create new ones.

This is especially important for customers of {productname} or the Proxy Server because systems kickstarted by them must have the server key imported into {productname} and associated with the relevant {kickstart} profiles.

Import a profile by creating a new key on this page and then make the profile association in the `GPG and SSL keys` subtab of the `Autoinstallation Details` page.

To create a key or certificate, click the `Create Stored Key/Cert` link in the upper-right corner of the page.
Enter a description, select the type, upload the file, and click the btn:[Update Key] button.
A unique description is required.

[IMPORTANT]
====
The GPG key you upload to {productname} must be in ASCII format.
Using a GPG key in binary format causes anaconda, and therefore the {kickstart} process, to fail.
