[[clients-opensuseleapmicro]]
= {leapmicro} クライアントの登録
:description: openSUSE Leap Microを実行しているクライアントの登録方法と、信頼性の高いエッジコンピューティング環境を実現するためのチャネル管理の設定方法について学びます。
:revdate: 2026-06-01
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

このセクションでは、{leapmicro}オペレーティングシステムを{x86_64}および{aarch64}アーキテクチャ上で実行しているクライアントの登録について説明します。

{leapmicro}は、エッジコンピューティングのために特別に構築された超信頼性のある軽量オペレーティングシステムです。
それは、SUSE Linux Enterpriseの企業向けの堅牢なセキュリティとコンプライアンスコンポーネントを活用し、現代的で不変の開発者フレンドリーなOSプラットフォームと統合しています。

{leapmicro}はトランザクション更新を使用します。
トランザクション更新はアトミックであり(すべての更新が成功した場合にのみすべての更新が適用されます)、ロールバックをサポートします。
それらは、システムが再起動されるまで変更が有効化されないため、実行中のシステムには影響しません。
この情報は、menu:システム[詳細> 概要]サブタブに表示されます。

トランザクション更新と再起動に関する詳細は、 https://documentation.suse.com/sles/15-SP7/html/SLES-all/cha-transactional-updates.html.を参照してください。

== クライアントの登録

[WARNING]
====
{leapmicro}クライアントは、登録後に再起動が必要です。

起動プロセスの後、{leapmicro}での自動起動が無効になります。この変更は、ブートストラップソルトステートの適用結果をSaltが伝える前に発生する可能性のある間欠的な自動再起動のために実施されました。

再起動は、登録が完了した後に自動的にスケジュールされますが、デフォルトの再起動マネージャのメンテナンスウィンドウを尊重しています。
このウィンドウは、クライアントが登録された後に数時間になる場合があります。
{leapmicro}の登録を迅速化するために、登録スクリプトが終了した後にクライアントを手動で再起動してください。
====
