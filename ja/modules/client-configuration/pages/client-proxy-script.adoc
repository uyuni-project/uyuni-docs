[[script-client-proxy]]
= ブートストラップスクリプトを使用してクライアントをプロキシに登録する
:description: SUSE Multi-Linux Manager Serverでmgr-bootstrapコマンドラインツールを使用して作成されたブートストラップスクリプトを使い、クライアントがプロキシサーバに接続するように設定します。
:revdate: 2025-03-20
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメントチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1


クライアントは、ブートストラップスクリプトを使用して{productname}プロキシを通じて登録できます。
これは、{productname}サーバに直接クライアントを登録するのとほぼ同じ方法で行われます。

`mgr-bootstrap`コマンドラインツールを使用して、{productname}サーバーコンテナ内で起動するスクリプトを作成します。`mgr-bootstrap`には、プロキシの完全修飾ドメイン名（FQDN）、アクティベーションキー、またはGPGキーなどのパラメータが必要です。
プロキシのFQDNは必須です。
他のすべてのパラメータは、特定のセットアップに依存します。


その後、ブートストラップスクリプトは、クライアントに必要すべての情報を配備します。



.プロシージャ:ブートストラップスクリプトを使用してクライアントをプロキシに登録する

. {productname}サーバで{webui}を使用してクライアントアクティベーションキーを作成します。
  詳細については、xref:client-configuration:activation-keys.adoc[]を参照してください。
. サーバのコンテナホストで、コンテナ内で端末を起動します。

+

[source,shell]
----
mgrctl term
----

+

--
.. コンテナ内で起動するスクリプトを作成するには、たとえば次のオプションを指定して`mgr-bootstrap`コマンドを実行します。

+

[source,shell]
----
mgr-bootstrap --hostname=MLM_PROXY --activation-keys=ACTIVATION_KEY \
  --script bootstrap-MLM_PROXY.sh
----

+

`MLM_PROXY`をお使いのプロキシのホスト名に置き換えてください。

+
`--script`オプションを使用すると、説明的な名前で起動するスクリプトを保存できます。
   必要に応じて、追加のコマンドラインオプションを使用してブートストラップスクリプトをカスタマイズします。
   利用可能なオプションを表示するには、コマンドラインから`mgr-bootstrap --help`と入力します。

.. オプション:結果として得られたブートストラップスクリプトを編集します。
--

. ブートストラップスクリプトを実行します。
  次の2つのオプションがあります：

+

* スクリプトをクライアントに転送し、クライアント上で直接実行します。
  代わりに、プロキシがリクエストをサーバーに転送するため、クライアントは http://MLM_PROXY/pub/bootstrap-MLM_PROXY.sh.からダウンロードできます。`MLM_PROXY`をお使いのプロキシのホスト名に置き換えてください。

* プロキシから`ssh`を使用して起動するスクリプトを実行してください。
  `MLM_PROXY`をお使いのプロキシのホスト名に、` <client.example.com>`をクライアントのホスト名にそれぞれ置き換えてください：

+

[source,shell]
----
cat /srv/www/htdocs/pub/bootstrap/bootstrap-MLM_PROXY.sh | ssh root@<client.example.com> /bin/bash
----
