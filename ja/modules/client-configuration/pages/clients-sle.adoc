以下を含むすべての{sle}製品を準備する際には、この章の手順を使用してください。

これらの手順は、古い{sle}バージョンおよびサービスパックにも使用できます。

古い製品のチャンネル名を見つけるには、{productname}サーバーのコマンドプロンプトで、ルートとして、`mgr-sync`コマンドを使用します:

----
mgr-sync list --help
----

次に、興味のある引数を指定します。
例: `channels`:

----
mgr-sync list channels [-c]
----

== GPGキーの管理

クライアントは、ソフトウェアパッケージがインストールされる前に、その真正性を確認するためにGPGキーを使用します。
信頼できるソフトウェアのみがクライアントにインストールできます。

[IMPORTANT]
====
クライアントのセキュリティのために、GPGキーを信頼することは重要です。
どのキーが必要で信頼できるかを決定するのは管理者の仕事です。
GPG キーが信頼されていない場合、ソフトウェア チャンネルは使用できないため、クライアントにチャンネルを割り当てる決定はキーを信頼する決定に依存します。
====

GPGキーに関する詳細は、xref:client-configuration:gpg-keys.adoc[]をご覧ください。

[NOTE]
====
{sles}{nbsp}15および{sles}{nbsp}12クライアントの両方に同じGPGキーを使用してください。
正しいキーは `sle12-gpg-pubkey-39db7c82.key` と呼ばれます。
====
