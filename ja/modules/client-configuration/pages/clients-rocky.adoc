== ソフトウェアチャンネルの追加


{productname}サーバに{rocky}クライアントを登録する前に、必要なソフトウェアチャネルを追加して同期する必要があります。

現在サポートされているアーキテクチャは、`x86_64`と`aarch64`であり、バージョン9ではさらに{ppc64le}と{s390x}が追加されます。
サポートされている製品とアーキテクチャの完全なリストについては、xref:client-configuration:supported-features.adoc[]をご覧ください。

ifeval::[{uyuni-content} == true]
[IMPORTANT]
====
{rocky}{nbsp}10、{rocky}{nbsp}9および{rocky}{nbsp}8クライアントの場合、BaseおよびAppStreamチャネルの両方を追加してください。
両方のチャンネルからパッケージが必要です。
両方のチャネルを追加しないと、パッケージが不足しているため、ブートストラップリポジトリを作成できません。
====

[NOTE]
====
アップストリームと{productname}チャンネルの間でAppStreamチャンネルに利用可能なパッケージの数に不一致が見られるかもしれません。
異なる時点で追加された同じチャンネルを比較すると、異なる数が表示されることもあります。
これは、{rocky}がリポジトリを管理する方法によるものです。{rocky}は新しいバージョンがリリースされると古いバージョンのパッケージを削除しますが、{productname}は年齢に関係なくすべてを保持します。
====

{rocky} 8のモジュラーチャネルを使用している場合、クライアントでPython 3.6モジュールストリームを有効にする必要があります。
Python 3.6 を提供しない場合、`spacecmd`パッケージのインストールは失敗します。

== アクティベーションキーの作成

{rocky}チャネルと関連付けられているアクティベーションキーを作成する必要があります。

アクティベーションキーの詳細については、xref:client-configuration:activation-keys.adoc[]を参照してください。



== GPGキーの管理

クライアントは、ソフトウェアパッケージがインストールされる前に、その真正性を確認するためにGPGキーを使用します。
信頼できるソフトウェアのみがクライアントにインストールできます。

[IMPORTANT]
====
クライアントのセキュリティのために、GPGキーを信頼することは重要です。
どのキーが必要で信頼できるかを決定するのは管理者の仕事です。
GPGキーが信頼されていない場合、ソフトウェアチャネルをクライアントに割り当てることはできません。
====

GPGキーに関する詳細情報は、xref:client-configuration:gpg-keys.adoc[]を参照してください。
