= その他のサードパーティプロバイダを使用した仮想化
:description: SUSE Multi-Linux ManagerでVMwareなどのサードパーティ仮想化プロバイダを含むインポート設定を構成します。
:revdate: 2024-09-03
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメントチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

Xen、KVM、またはVMware以外のサードパーティ仮想化プロバイダを使用する場合、JSON設定ファイルを{productname}にインポートできます。

同様に、APIへの直接アクセスを提供しないVMwareインストール環境の場合、ファイルベースのVHMが基本的な管理機能を提供します。

[NOTE]
====
このオプションは、`virtual-host-gatherer`ツールで作成されたファイルをインポートするためのものです。
手動で作成されたファイルには対応していません。
====

.プロシージャ:JSONファイルのエクスポートとインポート

. VMネットワーク上で`virtual-host-gatherer`を実行して、JSON設定ファイルをエクスポートします。
. 生成されたファイルを、{productname}サーバがアクセス可能な場所に保存します。
. {productname} {webui}で、menu:Systems[Virtual Host Managers]に移動します。
. btn:[作成]をクリックし、`File-based`を選択します。
. `Add a file-based Virtual Host Manager`セクションで、以下のパラメータを使用します。
* `Label`フィールドに、VHMのカスタム名を入力します。
* `Url`フィールドに、エクスポートしたJSON設定ファイルへのパスを入力します。
. btn:[Create]をクリックして変更を保存し、VHMを作成します。
. `Virtual Host Managers`ページで、新しいVHMを選択します。
. `Properties`ページで、btn:[データの更新]をクリックして、新しいVHMを一覧に反映させます。

詳細については、{productname}サーバの`virtual-host-gatherer`のマニュアルページを参照してください:

----
man virtual-host-gatherer
----


そのパッケージの`README`ファイルには、ハイパーバイザーの`type`に関する背景情報が記載されています:

----
/usr/share/doc/packages/virtual-host-gatherer/README.md
----

マニュアルページおよび`README`ファイルには、設定ファイルのサンプルも含まれています。
