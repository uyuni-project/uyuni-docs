このセクションでは、{sll}オペレーティングシステムを実行しているクライアントの登録について説明します。{sll}クライアントは{rhel}または{centos}に基づいています。


[NOTE]
====
{sll}クライアントは、時々{sleses}（SLESES）、Liberty、RES、または{redhat} {es}とも呼ばれます。
====

{sll}ソフトウェアチャンネルは{suse}によって提供され、パッケージの更新のみを提供し、パッケージ自体は提供しません。
{sll}クライアントを登録するには、必要なベースチャンネルを作成するために{sll}製品（以下に概説）を登録し、その後、必要な{redhat}または{centos}パッケージをカスタム子チャンネルとしてインポートする必要があります。
{sll}ソフトウェアチャンネルによって提供される更新を適用する前に、初期パッケージを{redhat}または{centos}から直接取得する必要があります。

== ソフトウェアチャンネルの追加


{sll}クライアントの場合、必要なパッケージの一部は{rhel}または{centos}のインストールメディアに含まれています。
{sll}クライアントを登録する前に、これらのパッケージをインストールしておく必要があります。

{sll}製品は{scclongform}によって提供されます。
これにはクライアントツールパッケージも含まれます。

{sll}クライアントを{productname}サーバに登録する前に、必要なソフトウェアチャンネルを追加して同期する必要があります。

2つの異なるチャンネルセットを選択する必要があります。一方は{sll}用、他方はクライアントツール用です。

正しい{sll}チャネルに関連付けられたアクティベーションキーが必要です。
アクティベーションキーに関する詳細は、xref:client-configuration:activation-keys.adoc[]を参照してください。

[[add.base.media]]
=== ベースメディアの追加

{sll}ソフトウェアチャネルは、パッケージ自体ではなく、更新のみを提供します。
{sll}クライアントを登録するには、まず{sll}製品を登録して（以下に説明）、ベースチャネルを作成し、次に必要なRed HatまたはCentOSパッケージをカスタム子チャネルとしてインポートします。
初期パッケージは、{redhat}または{centos}から直接取得する必要があります。その後、{sll}からの更新を適用します。
重要なことに、Red Hatのサブスクリプションを維持する必要はありませんが、移行中にRed Hatへの継続的な支払い義務があるかどうかを確認するために法務部門に相談してください。
必要なすべてのパッケージを確保するために、最小限の{jeos}イメージではなく、完全なDVDイメージを使用してください。

{productname}カスタムチャネルを使用して、{rhel}または{centos}メディアを設定できます。
ベースメディアのすべてのパッケージは、子チャネルにミラーリングする必要があります。

チャンネルの名前は自由に選択できます。

[IMPORTANT]
====
{redhat} 10、9、および8クライアントの場合、ベースチャネルとAppStreamチャネルの両方を追加してください。
両方のチャネルからのパッケージが必要です。
両方のチャンネルを追加しないと、パッケージが不足しているため、ブートストラップリポジトリを作成できません。
====

モジュラー チャンネルを使用している場合は、クライアントで Python 3.6 モジュール ストリームを有効にする必要があります。
Python 3.6 を提供しない場合、`spacecmd` パッケージのインストールは失敗します。


.プロシージャ:ベースメディアをカスタムチャネルに追加する（RHEL 7.0の例）
[role=procedure]
____
[WARNING]
====
{redhat}からのオリジナルのインストールメディアを使用する必要があります。
メジャーバージョンリリース（例えば`7`）を使用する必要があり、マイナーバージョンリリース（例えば`7.9`）は使用しないでください。
====
. ベースメディアイメージを{productname}サーバーコンテナホストの`/tmp/`ディレクトリにコピーします。

. ベースメディアイメージを{productname} Serverコンテナに取得するには、次のコマンドを使用します:

+
----
mgradm distribution copy /tmp/rhel-server-7.0-x86_64-dvd.iso rhel-server-7.0
----

+
コンテナ内の宛先は`/srv/www/distributions/rhel-server-7.0`になります。

. 前に作成した子チャンネルにパッケージをインポートします。

+

[source,shell]
----
mgrctl exec -- spacewalk-repo-sync -c <channel-label> -u file:///srv/www/distributions/rhel-server-7.0/
----

____

バージョン9および8クライアントの場合は、ラベルとディレクトリを適宜調整し、さらにベースチャンネルとAppStreamチャンネルの両方を同期します。

[source,shell]
----
mgrctl exec -- spacewalk-repo-sync -c <channel-label> -u file:///srv/www/distributions/rhel-server-9.0/BaseOS
mgrctl exec -- spacewalk-repo-sync -c <channel-label> -u file:///srv/www/distributions/rhel-server-9.0/AppStream
----

または

[source,shell]
----
mgrctl exec -- spacewalk-repo-sync -c <channel-label> -u file:///srv/www/distributions/rhel-server-8.0/BaseOS
mgrctl exec -- spacewalk-repo-sync -c <channel-label> -u file:///srv/www/distributions/rhel-server-8.0/AppStream
----


==== オプション:コンテンツURLからベースメディアを追加します。

または、{redhat} CDNまたは{centos}が提供するコンテンツURLにアクセスできる場合、カスタムリポジトリを作成してパッケージをミラーリングできます。

[[clients-sleses-el-migration]]
== Enterprise Linux (EL)クライアントを{sll}に移行します。

{rhel}や{centos}などのEnterprise Linux (EL)クライアントがすでに{productname}のminionとして登録されており、ユーザがそれを{sll}に移行したい場合は、再アクティベーションキーを使用して、移行を推進するアクティベーションキーを適用できます。

再アクティベーションキーに関する詳細は、xref:client-configuration:activation-keys.adoc#activation-keys-reactivation[]を参照してください。

再アクティベーションキーはminionごとに発行され、{webui}またはAPIを使用して生成できます。
詳細については、 https://documentation.suse.com/multi-linux-manager/5.1/api/suse-manager/api/system.html#apidoc-system-obtainReactivationKey-loggedInUser-sid.を参照してください。

クライアントを再アクティベーションするには、ユーザーはクライアント上で起動するスクリプトを実行し、再アクティベーションキーを環境変数として渡します。
例:

----
REACTIVATION_KEY=<KEY> ./bootstrap_liberate9.sh
----

別の方法として、{salt}（または`/etc/venv-salt-minon/minion.d/susemanager.conf`）にある`/etc/salt-minon/minion.d/susemanager.conf`クライアント設定ファイルに特別なフラグを追加します（この内容を既存の内容に結合してください）。

----
grains:
    susemanager:
        activation_key: "<KEY_ID>"
        management_key: "MINION_REACTIVATION_KEY"
----

`susemanager.conf`ファイルを変更した後、`salt-minion`サーバー上で{salt}サービスを再起動する必要があります。 デフォルトでは:

----
systemctl restart venv-salt-minon
----

または、レガシー{salt}の場合は、

----
systemctl restart salt-minon
----
