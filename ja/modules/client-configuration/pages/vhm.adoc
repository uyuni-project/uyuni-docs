[[virt-vhm]]
= 仮想ホストマネージャ
:description: 仮想ホストマネージャーを使用して仮想クライアントを構成し、SUSE Multi-Linux Manager Web UIに表示されるデータを収集します。
:revdate: 2025-04-15
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

仮想ホストマネージャ(VHM)は、さまざまなクライアントの種類から情報を収集するために使用します。

VHMsは、プライベートまたはパブリッククラウドインスタンスを収集し、それらを仮想化グループに整理するために使用できます。
このように整理された仮想化クライアントを使用すると、Taskomaticはクライアントのデータを収集し、{productname} {webui}に表示します。
VHMsは、仮想化クライアントでサブスクリプションマッチングを使用することも可能です。

{productname}サーバー上にVHMを作成し、利用可能なパブリッククラウドインスタンスのインベントリを作成するために使用できます。
また、{kube}で作成されたクラスターを管理するためにVHMを使用することもできます。

* Amazon Web ServicesでVHMを使用する方法についての詳細は、xref:client-configuration:vhm-aws.adoc[]を参照してください。
* Microsoft AzureでVHMを使用する方法についての詳細は、xref:client-configuration:vhm-azure.adoc[]を参照してください。
* NutanixでVHMを使用する方法についての詳細は、xref:client-configuration:vhm-nutanix.adoc[]を参照してください。
* VMWare vSphereでVHMを使用する方法についての詳細は、xref:client-configuration:vhm-vmware.adoc[]を参照してください。
* 他のホストでVHMを使用する方法についての詳細は、xref:client-configuration:vhm-file.adoc[]を参照してください。
