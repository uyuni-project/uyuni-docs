= 無人プロビジョニング

MACアドレスで識別されるクライアントと自動インストールプロファイルの関連付けを宣言するために、API呼び出しを使用できます。
次回システムが再起動すると、指定されたプロファイルに基づいてインストールが開始されます。


.プロシージャ:手動で宣言したプロファイルからの再インストール

. {productname}サーバー上のコマンドプロンプトで、`system.createSystemRecord` API呼び出しを使用します。
  この例では、`name`をクライアントの名前に、`<profile>`をプロファイルに、`<iface>`をクライアントのインターフェース名（例：`eth0`）、`<hw_addr>`をハードウェアアドレス（例：`00:25:22:71:e7:c6`）に置き換えます。
+
----
$ spacecmd api -- --args '["<name>", "<profile>", "", "", \
  [ {"name": "<iface>", "mac": "<hw_addr>"} ]]' \
  system.createSystemRecord
----
. クライアントの電源をオンにします。
  ネットワークからブートし、インストール用の正しいプロファイルが選択されます。

このコマンドは、{cobbler}にシステムレコードを作成します。
カーネルオプション、クライアントのIPアドレス、ドメイン名などの追加パラメータを指定することもできます。
詳細については、`createSystemRecord call`のAPIマニュアルを参照してください。
