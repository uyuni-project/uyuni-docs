[[registration-methods]]
= 登録メソッド
:description: YaSTツールやコマンドラインコマンドなど、さまざまな方法を使用して、SUSEサーバにクライアントを登録します。
:revdate: 2024-02-28
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

クライアントを{productname}サーバに登録する方法は複数あります。

* クライアントの数が少ない場合、{productname}の{webui}を使用してクライアントを登録することをお勧めします。
    詳細については、xref:client-configuration:registration-webui.adoc[]を参照してください。
* プロセスをより詳細に制御したい場合、または多数のクライアントを登録する必要がある場合は、ブートストラップスクリプトの作成をお勧めします。
    詳細については、xref:client-configuration:registration-bootstrap.adoc[]を参照してください。
* さらに詳細にプロセスを制御するには、コマンド行でsingleコマンドを実行すると便利です。
    詳細については、xref:client-configuration:registration-cli.adoc[]を参照してください。

クライアントは、登録する前に{productname}サーバと日時が正しく同期されている必要があります。

最初にアクティベーションキーを作成する必要があります。起動スクリプトまたはコマンドライン方式を使用するためです。
アクティベーションキーの作成に関する詳細については、xref:client-configuration:activation-keys.adoc[]を参照してください。


[IMPORTANT]
====
{productname}サーバを自分自身に登録しないでください。
{productname}サーバは、個別に管理するか、別の{productname}サーバを使用して管理する必要があります。
複数のサーバを使用することに関する詳細については、xref:specialized-guides:large-deployments/hub-multi-server.adoc[]を参照してください。
====
