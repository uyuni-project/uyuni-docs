新しいクライアントを{productname}サーバに登録するには、まず{webui}またはコマンドラインツールを使用して、新しいクライアントのオペレーティングシステムとアーキテクチャのソフトウェアチャンネルを同期します。

ifeval::[{uyuni-content} == true]

次に、コマンドラインツールを使用して、新しく追加されたオペレーティングシステムのクライアントツールリポジトリを同期します。

その方法および詳細は、クライアントのオペレーティングシステムによって異なります。
これは次のセクションで説明されています。



== 準備手順または確認事項


始める前に、クライアントで{productname}サーバと日時が正しく同期していることを確認してください。

アクティベーションキーも作成しておく必要があります。
アクティベーションキーの作成に関する詳細は、xref:client-configuration:activation-keys.adoc[]を参照してください。



[IMPORTANT]
====
{productname}サーバを自分自身に登録しないでください。
{productname}サーバは、個別に管理するか、別の{productname}サーバを使用して管理する必要があります。
複数のサーバを使用することに関する詳細は、xref:specialized-guides:large-deployments/hub-multi-server.adoc[]を参照してください。
====



[[suse-registration-overview-check]]
== 以前の登録ステータスの確認

クライアントが以前にSCC、SMT、またはRMTに登録されていた場合、クライアントを`SUSEConnect`サーバに移行する前に、{productname}でその登録を削除する。
最初に古い登録を削除することを忘れないでください。
そうしない場合、不要なリポジトリが、クライアントが{suse}付きの新しいバージョンのオペレーティングシステム{productname}へ移行した後、クライアントに追加されます。

{sle} 12または15クライアントの登録解除を行うには、クライアント上でルート権限で次のコマンドを実行してください。

----
SUSEConnect --de-register
SUSEConnect --cleanup
----

詳細なクリーンアップコマンドおよび検証手順に関する詳細は、 https://www.suse.com/de-de/support/kb/doc/?id=000019054.を参照してください。
