== ソフトウェアチャンネルの追加


[NOTE]
====
{almalinux}クライアントを{productname}に登録することは、`enforcing`のデフォルトSELinux設定および`targeted`ポリシーでテストされています。
{almalinux}クライアントを{productname}に登録するためにSELinuxを無効にする必要はありません。
====

{almalinux}クライアントを{productname}サーバに登録する前に、必要なソフトウェアチャネルを追加して同期する必要があります。

現在サポートされているアーキテクチャは、`x86_64`と`aarch64`であり、バージョン9ではさらに{ppc64le}と{s390x}が追加されます。
サポートされている製品とアーキテクチャの完全なリストについては、xref:client-configuration:supported-features.adoc[]をご覧ください。

ifeval::[{uyuni-content} == true]
[IMPORTANT]
====
{almalinux} 10、{almalinux} 9、および{almalinux} 8クライアントの場合、BaseチャンネルとAppStreamチャンネルの両方を追加してください。
両方のチャンネルからパッケージが必要です。
両方のチャンネルを追加しない場合、パッケージが不足しているため、起動するリポジトリを作成できません。
====
endif::[]

モジュラーチャンネルを使用している場合、{almalinux} 8クライアントでPython 3.6モジュールストリームを有効にする必要があります。
Python 3.6 を提供しない場合、`spacecmd`パッケージのインストールに失敗します。

[NOTE]
====
アップストリームと{productname}チャンネルの間でAppStreamチャンネルに利用可能なパッケージの数に不一致が見られるかもしれません。
異なる時点で追加された同じチャンネルを比較すると、異なる数が表示されることもあります。
これは、{almalinux}がリポジトリを管理する方法によるものです。{almalinux}は新しいバージョンがリリースされると古いバージョンのパッケージを削除しますが、{productname}は年齢に関係なくすべてを保持します。
====

== GPGキーの管理

クライアントは、ソフトウェアパッケージがインストールされる前に、その真正性を確認するためにGPGキーを使用します。
信頼できるソフトウェアのみがクライアントにインストールできます。

[IMPORTANT]
====
GPGキーを信頼することは、クライアントのセキュリティにとって重要です。
どのキーが必要で信頼できるかを決定するのは管理者の仕事です。
GPGキーが信頼されていない場合、ソフトウェアチャンネルは使用できないため、クライアントにチャンネルを割り当てる決定はキーを信頼する決定に依存します。
====

GPG キーの詳細については、xref:client-configuration:gpg-keys.adoc[] を参照してください。
