オペレーティングシステムは、自分のGPGキーを直接信頼するか、少なくとも最小システムにインストールして出荷します。
しかし、異なるGPGキーで署名されたサードパーティパッケージは手動で処理する必要があります。
クライアントは、GPGキーが信頼されていなくても正常に起動できます。
ただし、キーが信頼されるまで、新しいクライアントツールパッケージをインストールしたり、更新したりすることはできません。

クライアントは、ソフトウェアチャネルのために入力されたGPGキー情報を使用して、信頼されたキーを管理します。
GPGキー情報を持つソフトウェアチャネルがクライアントに割り当てられると、チャネルが更新されるか、このチャネルから最初のパッケージがインストールされると、キーが信頼されます。

ソフトウェアチャネルページのGPGキーURLパラメータには、「空白」で区切られた複数のキーURLを含めることができます。
ファイルURLの場合、ソフトウェアチャネルを使用する前に、GPGキーファイルをクライアントにデプロイする必要があります。

{redhat}ベースのクライアントのクライアントツールチャネル用GPGキーは、クライアントの`/etc/pki/rpm-gpg/`にデプロイされ、ファイルURLで参照できます。

// SUSE Liberty Linux not available at Uyuni for now

ifeval::[{mlm-content} == true]
{sll}クライアントのGPGキーの場合も同様です。
endif::[]

ソフトウェアチャンネルがクライアントに割り当てられている場合にのみ、システムによってインポートされて信頼されます。

[NOTE]
====
Debianベースのシステムはメタデータのみを署名するため、単一のチャネルに対して追加のキーを指定する必要はありません。
ユーザーがxref:administration:repo-metadata.adoc[]の「独自のGPGキーを使用する」に記載されているように、メタデータに署名するために独自のGPGキーを設定すると、そのキーのデプロイメントと信頼が自動的に行われます。
====


== ユーザ定義のGPGキー

ユーザは、クライアントに配備するカスタムのGPGキーを定義できます。

いくつかのpillarデータを提供し、{salt}ファイルシステムにGPGキーファイルを提供することで、自動的にクライアントにデプロイされます。

これらのキーは、RPMベースのオペレーティングシステムでは`/etc/pki/rpm-gpg/`に、Debianシステムでは`/usr/share/keyrings/`にデプロイされます。

キーをデプロイする対象のクライアントのpillarキー[literal`custom_gpgkeys`]を定義し、キーファイルの名前を一覧にします。
// I think the cat command is here to show the contents of the file.
// In this case, the cat is superfluous.
----
cat /srv/pillar/mypillar.sls
custom_gpgkeys:
  - my_first_gpg.key
  - my_second_gpgkey.gpg
----

さらに、{salt}ファイルシステム上で、`gpg`という名前のディレクトリを作成し、`custom_gpgkeys`のpillarデータで指定された名前のGPGキーファイルを保存します。
// I think the ls command is here to show the contents of the directory.
// In this case, the ls is superfluous.

----
ls -la /srv/salt/gpg/
/srv/salt/gpg/my_first_gpg.key
/srv/salt/gpg/my_second_gpgkey.gpg
----

キーは、クライアントの`/etc/pki/rpm-gpg/my_first_gpg.key`および`/etc/pki/rpm-gpg/my_second_gpgkey.gpg`にデプロイされます。

最後のステップは、ソフトウェアチャネルのGPGキーURLフィールドにURLを追加することです。
menu:ソフトウェア[管理> チャネル]に移動し、変更したいチャネルを選択します。
`GPG key URL`に値`file:///etc/pki/rpm-gpg/my_first_gpg.key`を追加します。


== ブートストラップスクリプトのGPGキー

.プロシージャ:ブートストラップスクリプトを使用してクライアントでGPGキーを信頼する
. {productname}サーバーのコマンドプロンプトで、`/srv/www/htdocs/pub/`ディレクトリの内容を確認します。
  このディレクトリには、すべての利用可能な公開鍵が含まれています。
  登録しているクライアントに割り当てられたチャネルに適用されるキーをメモしておきます。
. 関連する起動スクリプトを開き、`ORG_GPG_KEY=`パラメータを見つけて、必要なキーを追加します。
  例:
+
----
uyuni-gpg-pubkey-0d20833e.key
----
+
以前保存したキーを削除する必要はありません。


[IMPORTANT]
====
GPGキーを信頼することは、クライアントのセキュリティにとって重要です。
どのキーが必要で信頼できるかを決定するのは管理者の仕事です。
GPGキーが信頼されていない場合、ソフトウェアチャネルはクライアントに割り当てることができません。
====
