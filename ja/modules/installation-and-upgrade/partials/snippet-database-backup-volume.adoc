= データベースバックアップボリューム
:revdate: 2025-07-29
:page-revdate: {revdate}

`mgradm migration`または`mgradm upgrade`を使用したサーバーの移行やアップグレードにより、データベースのバックアップを含むボリュームが作成される場合があります。

{postgresql}データベースのバージョンが上がると、アップグレードを実行する前に古いデータベースを別の場所に保存する必要があります。
この目的のために、`mgradm`は動的にボリューム`var-pgsql-backup`を作成します。
移行またはアップグレードが完了し、ユーザーが新しいシステムが期待通りに動作していることを確認したら、このボリュームは安全に削除することができます。
