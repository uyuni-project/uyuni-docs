[[persistant-volume-list]]
= 永続コンテナボリューム
:description: SUSE Multi-Linux Managerの永続コンテナボリュームへのアクセス方法および管理方法を学びます。
:revdate: 2025-07-29
:page-revdate: {revdate}
:page-author: SUSE Product & Solution Documentation Team
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

コンテナ内で行われた変更は保持されません。
永続ボリュームの外で行われた変更は破棄されます。
以下は、{productname} {productnumber}の永続ボリュームのリストです。

デフォルトのボリュームの場所をカスタマイズするには、`podman volume create`コマンドを使用して、ポッドを初めて起動する前に必要なボリュームを作成しておくことを確認してください。

[NOTE]
====
この表は、Helmチャートおよびsystemctlサービス定義の両方で示されているボリュームマッピングに正確に従っています。
====


== サーバ


次のボリュームは、サーバ上の**Podman**のデフォルトのストレージの場所に保存されます。

.永続ボリューム:**Podmanデフォルトストレージ**
[cols="name,directory"]
|===
|ボリューム名 | ボリュームディレクトリ

| **Podmanストレージ**
| `/var/lib/containers/storage/volumes/`
|===
