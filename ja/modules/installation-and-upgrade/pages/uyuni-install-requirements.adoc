{productname}プロキシは`/var/cache/`ディレクトリにパッケージをキャッシュします。
`/var/cache/`に十分な空き容量がない場合、プロキシは古い未使用のパッケージを削除し、新しいパッケージと置き換えます。

この動作の結果は以下のとおりです。

* プロキシ上の`/var/cache/`ディレクトリの容量が大きいほど、プロキシと{productname}サーバ間のトラフィックが少なくなります。

* プロキシ上の`/var/cache/`ディレクトリを`/var/spacewalk/`サーバ上の{productname}と同じサイズにすることで、最初の同期後のトラフィック量の増大を防止できます。

* `/var/cache/`サーバ上の{productname}ディレクトリは、プロキシ上のものと比べて小さくても構いません。
    サイズ推定のガイドについては、<<server-hardware-requirements>>セクションを参照してください。
