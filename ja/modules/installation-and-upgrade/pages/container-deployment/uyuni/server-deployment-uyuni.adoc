== 配備の準備


このセクションでは、{productname}サーバのセットアップとデプロイメントに関する専門知識を習得できます。
このプロセスには、`Podman`と`Uyuni container utilities`のインストール、デプロイメント、そして`mgrctl`を通じたコンテナとの対話の開始が含まれます。

[NOTE]
====
このセクションでは、{leapmicro} {microversion}ホストサーバを設定済みであることを想定しています。物理マシンまたは仮想環境内のどちらで実行されているかは関係ありません。

https://download.opensuse.org/distribution/leap-micro/
====

== コンテナで使用するためにUyuniツールをインストールする


.プロシージャ:{leapmicro} {microversion}にUyuniツールをインストールする
. ローカルホストで、端末のウィンドウを開くか、{leapmicro} {microversion}が実行される仮想マシンを起動します。
. ログインします。
. `transactional-update shell`を入力してください：
+
----
transactional-update shell
----
+
. 次のリポジトリを{leapmicro} {microversion}サーバに追加します:
+
----
zypper ar https://download.opensuse.org/repositories/systemsmanagement:/Uyuni:/Stable/images/repo/Uyuni-Server-POOL-$(arch)-Media1/ uyuni-server-stable
----
+
. リポジトリのリストを更新してキーを受け入れます。
+
----
zypper ref
----
+
. コンテナツールをインストールします。
+
[source, shell]
----
zypper in mgradm mgrctl mgradm-bash-completion mgrctl-bash-completion uyuni-storage-setup-server
----
+
. トランザクションシェルを終了します。
+
----
transactional update # exit
----
. ホストを再起動します。

Uyuniコンテナユーティリティに関する詳細は、link:https://build.opensuse.org/repositories/systemsmanagement:Uyuni:Stable:ContainerUtils[Uyuni Container Utilities]を参照してください。


== カスタム永続ストレージの設定

このステップはオプションです。
ただし、ご使用のインフラストラクチャにカスタム永続ストレージが必要な場合は、`mgr-storage-server`ツールを使用します。

* 詳細については、`mgr-storage-server --help`を参照してください。
詳細については、[command]\``mgr-storage-server \--help``を参照してください。このツールを使用すると、コンテナストレージとデータベースボリュームの作成が容易になります。

このコマンドは次のように使用します。
----
mgr-storage-server <storage-disk-device> [<database-disk-device>]
----

例:
----
mgr-storage-server /dev/nvme1n1 /dev/nvme2n1
----

[NOTE]
====
このコマンドは、`/var/lib/containers/storage/volumes`に永続ストレージボリュームを作成します。

詳細については、xref:installation-and-upgrade:container-management/persistent-container-volumes.adoc[]を参照してください。
====

=== 永続ボリューム

多くのユーザが永続ボリュームの場所を指定したいと考えています。

[NOTE]
====
{productname}をテストしている場合は、これらのボリュームを指定する必要はありません。`mgradm`が正しいボリュームをデフォルトでセットアップします。

ボリュームの場所の指定は一般的に、大規模な運用配備で使用されます。
====

デフォルトでは、`podman`はそのボリュームを`/var/lib/containers/storage/volumes/`に保存します。

このパスまたはその内部の期待されるボリュームパスにディスクをマウントすることで、ボリュームのカスタムストレージを提供できます。例えば、`/var/lib/containers/storage/volumes/var-spacewalk`のように。
これは特にデータベースとパッケージのミラーにとって重要です。

コンテナ内のすべての永続ボリュームのリストについては、以下を参照してください。

* xref:installation-and-upgrade:container-management/persistent-container-volumes.adoc[]
* xref:administration:troubleshooting/tshoot-container-full-disk.adoc[]
