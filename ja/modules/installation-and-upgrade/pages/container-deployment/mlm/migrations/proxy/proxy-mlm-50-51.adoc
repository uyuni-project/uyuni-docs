= プロキシの移行
:description: SLE Micro 5.5からSLE Micro 6.1、SLES 15 SP6からSLES 15 SP7への移行に関する検証済みの手順を使用して、プロキシとホストオペレーティングシステムをアップグレードします。
:revdate: 2026-01-29
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

== 概要


このドキュメントは、**ホストオペレーティングシステム**と**プロキシ拡張機能**の両方を、**{productname}**で管理される環境において移行するためのテスト済みかつ検証済みの手順を提供します。特に、**{productname} Proxy 5.0**を配備したシステムを対象としています。

対象となるアップグレードシナリオは次のとおりです。

* **SUSE Linux Enterprise Micro (SLE Micro) 5.5**から**SLE Micro 6.1**への移行
* **SUSE Linux Enterprise Server (SLES) 15 SP6**から**SLES 15 SP7**への移行
* **{productname}プロキシ拡張機能**を**バージョン 5.0**から**バージョン {productnumber}**にアップグレードします。

== {sle-micro} 5.5から{sl-micro} 6.1

このセクションでは、{sle-micro} 5.5ホストを{productname} 5.0プロキシでデプロイし、{sl-micro} 6.1に{productname} {productnumber}プロキシでアップグレードするためのテスト済みの手順を提供します。


=== 前提条件

[IMPORTANT]
====
プロキシを移行する前に、まずSUSE Manager 5.0サーバを{productname} {productnumber}に移行する必要があります。
====

* {productname} 5.0プロキシが{sle-micro} 5.5でインストールされ、実行中です。
* プロキシシステムが{productname}サーバに登録されています。


=== ディストリビューションのアップグレードとプロキシの移行

.プロシージャ:{productname} 5.0プロキシを{productname} {productnumber}プロキシに移行します。
[role=procedure]
____

. システムと{productname}ツールのバージョンを確認します。

+

[source,console]
----
cat /etc/os-release
mgrpxy --version
----

+

確認:

* オペレーティングシステム: `SLE Micro 5.5`
* ツールバージョン: `mgrpxy version 0.1.29`以上

+

. 実行中のコンテナを確認する:

+

[source,console]
----
podman ps
----

+

以下のコンテナが実行されていることを確認します。

- `proxy-squid`
- `proxy-ssh`
- `proxy-httpd`
- `proxy-tftpd`
- `proxy-salt-broker`

+

. 製品カタログを更新します。

+

* {productname} {webui}で、menu:管理者[セットアップウィザード> 製品]に移動します。
* 右側にある`Refresh`ボタンをクリックして、SUSEカスタマーセンターから製品カタログを更新します。

. {productname}サーバに新しいプロキシ製品を同期します。
  詳細については、xref:client-configuration:products.adoc[]を参照してください。

+

Web UIを使用して同期します。

* `SUSE Linux Micro 6.1`
* `{productname} Proxy Extension {productnumber}`

+

. 移行の実行
  詳細については、xref:client-configuration:client-upgrades-product-migration.adoc[]を参照してください。

+

プロキシシステムに移動し、menu:システム[概要] > menu:ソフトウェア[製品移行]を選択します。

+

移行元

* `SLE Micro 5.5` + `SUSE Manager Proxy 5.0 Extension`

+

移行先

* `SL Micro 6.1` + `{productname} Proxy Extension {productnumber}`

+

[IMPORTANT]
====
プロンプトが表示されたときに、必要であることが確認されていない限り、オプションのチャンネルを**選択しないでください**。
====

+

[NOTE]
====
アップグレードする前に、サーバのバックアップを作成することをお勧めします。
====

+

. 移行アクションを監視する

+

menu:システム[詳細> イベント]タブのでプロセスを追跡できます{webui}。

+

. アップグレードが完了したら、プロキシコンテナを停止して、システムを再起動します。

+

[source,console]
----
reboot
mgrpxy stop
----

+

. 再起動後のチェック:

+

アップグレードされたOSと{productname}拡張機能を確認してください。

+

[source,console]
----
cat /etc/os-release
SUSEConnect --status-text
----

+

. {productname}ツールのバージョンを確認してください。

+

[source,console]
----
mgrpxy --version
----

+

予想される出力:

- `mgrpxy version 5.1.11`以上。

+

. ルートSSHアクセスを有効にします（_必要な場合_）。
  SL Micro 6.1は、デフォルトでSSH経由のルートログインを無効にします。
  `/etc/ssh/sshd_config.d/sshd.conf`を編集します。

+

[source,console]
----
PermitRootLogin yes
----

+

サービスを再起動します。

+

[source,shell]
----
systemctl restart sshd
----

+

詳細については、xref:administration:troubleshooting/tshoot-remote-root-on-micro.adoc[]を参照してください。


. RPMパッケージとして新しいプロキシコンテナイメージをインストールします。

+

[source,shell]
----
transactional-update pkg install suse-multi-linux-manager-5.1-<arch>-proxy*
----

. プロキシを再起動します。

. プロキシコンテナをアップグレードします。

+

[source,console]
----
mgrpxy start
mgrpxy upgrade podman
----

+

. プロキシコンテナが動作していることを確認する

+

[source,console]
----
podman ps
----

+

すべての予期されるプロキシコンテナが稼働中である必要があります。

- `proxy-salt-broker`
- `proxy-httpd`
- `proxy-squid`
- `proxy-tftpd`
- `proxy-ssh`

____


=== 移行の完了

プロキシホストシステムは、更新された{productname}{productnumber}プロキシパッケージと同期された製品チャンネルを備えた{sl-micro}{microversion}を実行しています。


== {sles} 15 SP6から15 SP7へ。

このセクションでは、{sles} SP6ホストを{productname} 5.0プロキシから、{productname}{productnumber}プロキシを備えた{sles}{sp-version}にアップグレードする手順を提供します。


=== 前提条件

* {productname}プロキシ5.0は、{sles} 15 SP6にインストールされ、実行されています。
* プロキシシステムが{productname}サーバに登録されています。


=== ディストリビューションのアップグレードとプロキシの移行

.プロシージャ:{sles} 15 SP6で{productname}プロキシコンポーネントを更新します。
[role=procedure]
____

. オペレーティングシステムと{productname}ツールのバージョンを確認してください。

+

[source,console]
----
cat /etc/os-release
mgrpxy --version
----

+

確認:

* オペレーティングシステム: `SUSE Linux Enterprise Server 15 SP6`
* ツールバージョン: `mgrpxy version 0.1.29`以上


+

. 実行中のプロキシコンテナを一覧表示する

+

[source,console]
----
podman ps
----

+

次のコンテナが実行されていることを確認します。

- `proxy-salt-broker`
- `proxy-httpd`
- `proxy-squid`
- `proxy-tftpd`
- `proxy-ssh`
- サポートインフラストラクチャコンテナ

+

. {productname}サーバで新しいプロキシ製品を同期します。
  詳細については、xref:client-configuration:products.adoc[]を参照してください。

+

{webui}を使用して、以下を同期します。

* `SUSE Linux Enterprise Server 15 SP7`
- `{productname} Proxy Extension for SLE 5.1`

+

[NOTE]
====
プロキシ拡張機能は`Basesystem Module 15 SP7` > `Containers Module 15 SP7`の下に表示されます。
====

+

. 移行の実行
  詳細については、xref:client-configuration:client-upgrades-product-migration.adoc[]を参照してください。

+

プロキシシステムに移動し、menu:システム[概要> ソフトウェア> 製品移行]を選択します。

+

移行元

+

* `SUSE Linux Enterprise Server 15 SP6` + `SUSE Manager Proxy Extension 5.0`

+

移行先

+

* `SUSE Linux Enterprise Server 15 SP7` + `{productname} Proxy Extension for SLE 5.1`

+

[IMPORTANT]
====
プロンプトが表示されたとき、必要であることが確認されていない限り、オプションのチャンネルを**選択しないでください**。
====

+

[NOTE]
====
アップグレードする前に、サーバのバックアップを作成することをお勧めします。
====

+

. 移行アクションを監視する

+

menu:システム[詳細> イベント]タブのでプロセスを追跡できます{webui}。

+

. アップグレードが完了したら、プロキシコンテナを停止して、システムを再起動します。

+

[source,console]
----
reboot
mgrpxy stop
----

+

. 再起動後のチェック:

+

アップグレードされたOSと{productname}拡張機能を確認してください。

+

[source,console]
----
cat /etc/os-release
SUSEConnect --status-text
----

+

以下が表示されるはずです。

* `PRETTY_NAME="SUSE Linux Enterprise Server 15 SP7"`
* `SUSE Multi-Linux Manager Proxy 5.1 Extension for SLE`

+

. {productname}ツールのバージョンを確認してください。

+

[source,console]
----
mgrpxy --version
----

+

予想される出力:

* `mgrpxy version 5.1.11`以上。


+

. RPMパッケージとして新しいプロキシコンテナイメージをインストールします。

+

[source,shell]
----
zypper install suse-multi-linux-manager-5.1-x86_64-proxy*
----

. プロキシコンテナをアップグレードします。

+

[source,console]
----
mgrpxy start
mgrpxy upgrade podman
----

+

. プロキシコンテナが動作していることを確認する

+

[source,console]
----
podman ps
----

+

すべての予期されるプロキシコンテナが稼働中である必要があります。

- `proxy-salt-broker`
- `proxy-httpd`
- `proxy-squid`
- `proxy-tftpd`
- `proxy-ssh`

____


=== 移行の完了

{sles}プロキシホストシステムは現在、更新された{productname} {productnumber}プロキシパッケージと同期された製品チャンネルを備えた15 SP7を実行しています。
