= ミラーソースパッケージ
:description: SUSE® Multi-Linux Managerサーバを設定して、rhn.conf設定ファイルでこの機能をグローバルに有効にすることでソースパッケージをミラーリングし、その後再起動します。
:revdate: 2024-05-10
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

独自のパッケージをローカルに構築する場合、または法的な理由でパッケージのソースコードが必要な場合は、{productname}サーバ上のソースパッケージをミラーリングできます。

[NOTE]
====
ソースパッケージをミラーリングすると、大量のディスク容量が消費される可能性があります。
====

[IMPORTANT]
====
サーバコンテナ内でステップを実行する前に、`mgrctl term`を使用してください。
====

.プロシージャ:ソースパッケージのミラーリング
. 設定ファイル`/etc/rhn/rhn.conf`を開き、この行を追加します:
+

----
server.sync_source_packages = 1
----
. Spacewalkサービスを再起動して、変更を取得します。
+

----
mgradm restart
----

現在、この機能はすべてのリポジトリに対してグローバルにのみ有効にできます。
個別のリポジトリをミラーリングすることはできません。

この機能が有効化されると、次回のリポジトリ同期後にソースパッケージが{productname} {webui}で利用可能になります。
それらはバイナリパッケージのソースとして表示され、{webui}から直接ダウンロードできます。
ソースパッケージは、{webui}を使用しているクライアントにインストールすることはできません。
