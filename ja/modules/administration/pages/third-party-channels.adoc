[[third-party-channels]]
= サードパーティチャンネル
:description: SUSE® Multi-Linux Managerコマンドを使用して、リポジトリの同期のためにサードパーティのGPGキーを安全にインポートおよび管理します。
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

{productname}には、オプションのサードパーティリポジトリの内容を、管理可能な製品の一部と同期する機能が含まれています。

一部のサードパーティのGPGキーは、デフォルトで{productname}データベースに含まれていますが、一部は含まれていません。
含まれていないサードパーティのキーについては、そのようなサードパーティリポジトリを同期する際に、関連するキーをインポートする必要があります。

GPGキーをインポートするには、次のコマンド構文を使用し、{productname}ホストサーバでコマンドを実行します。

----
mgradm gpg add <path-to-gpg-key-file-or-URL>
----

. 例:ファイルからGPGキーを追加する:
+
----
mgradm gpg add repomd.xml.key
----

. 例:URLを使用してリモートリポジトリからGPGキーを追加する:
+
----
mgradm gpg add https://<3rd-party-domain>/path/to/repository/repodata/repomd.xml.key
----

{productname} GPGデータベースに現在保持されているキーを一覧表示するには、次のコマンドを実行します。

----
mgrctl exec -ti -- gpg --homedir /var/lib/spacewalk/gpgdir --list-keys
----

[NOTE]
====
クライアントにパッケージをインストールする際、クライアント自体でGPGキーを信頼する必要がある場合もあります。
これには、GPGキーがそこでも利用可能である必要があります。

詳細については、以下を参照してください: xref:client-configuration:gpg-keys.adoc[]。
====
