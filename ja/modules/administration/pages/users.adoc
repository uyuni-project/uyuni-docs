= ユーザー
:description: ユーザ権限を設定し、パスワードポリシーを管理し、SUSE Multi-Linux Managerの柔軟な管理ツールを使用してユーザを追加/削除することができます。
:revdate: 2026-05-13
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

{productname} 管理者は新しいユーザを追加し、権限を付与し、ユーザを無効化または削除できます。
多数のユーザを管理している場合、ユーザをシステムグループに割り当てて、グループレベルで権限を管理できます。
{webui}のシステムデフォルト、言語やテーマのデフォルトを変更することもできます。


[NOTE]
====
`Users`メニューは、{productname}管理者アカウントでログインしている場合にのみ使用できます。
====


{productname}ユーザを管理するには、menu:ユーザ[ユーザリスト> すべて]に移動して、{productname}サーバー内のすべてのユーザを確認します。
リスト内の各ユーザは、ユーザ名、実名、割り当てられた役割、最後にサインインした日付、およびユーザの現在のステータスを表示します。
btn:[ユーザの作成]をクリックして、新しいユーザアカウントを作成します。
ユーザー名をクリックして、`User Details`ページに移動します。

新しいユーザを組織に追加するには、btn:[ユーザの作成]をクリックして、新しいユーザの詳細を入力し、btn:[ログインの作成]をクリックします。


[[users.password.requirments]]
== パスワード要件


{productname}は、デフォルト値が選択された状態で出荷されます。

すべての新しいユーザパスワードが組織のセキュリティ基準に確実に準拠するように、{productname}管理者にはパスワード作成ルールを強制するオプションがあります。

{webui}で、menu:管理者[マネージャ設定> パスワードポリシー]に移動して、パスワード要件を定義します。
以下のフィールドの組み合わせを使用します：

最小パスワード長::
このフィールドを使用して、パスワードの最小長を定義します。

最大パスワード長::
このフィールドを使用して、パスワードの最大長を定義します。

数字が必要::
このフィールドを使用して、パスワードに数字(0～9)を含める必要があるかどうかを指定します。

小文字が必要::
このフィールドを使用して、パスワードに小文字(a～z)を含める必要があるかどうかを指定します。

大文字が必要::
このフィールドを使用して、パスワードに大文字(A～Z)を含める必要があるかどうかを指定します。

連続文字を制限::
このフィールドを使用して、連続文字が制限されるかどうかを指定します。

特殊文字が必要::
このフィールドを使用して、パスワードに特殊文字を含める必要があるかどうかを指定します。

使用できる特殊文字::
このフィールドは、`Require Special Characters`が選択されている場合にのみ有効になります。
許可される特殊文字を指定するために使用します。例えば、`!@#$%&*`などです。

Restrict Character Occurences (文字の出現を制限する)::
このフィールドを使用して、制限される文字の出現を指定します。

Max Character Occurences (最大文字出現回数)::
このフィールドを使用して、最大文字出現回数を指定します。

btn:[保存]をクリックして、変更したパスワード設定を保存します。

btn:[リセット]を使用して、すべての設定をデフォルトに戻します。


[NOTE]
====
{productname}は、次のデフォルト値で出荷されます。

* 最小パスワード長:4
* 最大パスワード長:32
* 大文字が必要: チェック済み
====


== アカウントの無効化と削除

不要になった場合は、ユーザアカウントを無効化または削除できます。
無効化されたユーザアカウントは、いつでも再有効化できます。
削除されたユーザアカウントは表示されず、復元することはできません。

ユーザは独自のアカウントを無効化できます。
ただし、ユーザに管理者ロールがある場合は、アカウントを無効化する前に、ロールを削除する必要があります。

無効化されたユーザは、{productname} {webui}にログインしたり、アクションをスケジュールしたりすることはできません。
無効化前にユーザによってスケジュールされたアクションは、アクションキューに残ります。
無効化されたユーザは、{productname}の管理者によって再有効化できます。


[[administrator-roles]]
== ユーザロール

ユーザには複数のロールを割り当てることができ、同時に複数のユーザが任意のロールを保持することができます。
常に少なくとも1人のアクティブな{productname}管理者が必要です。

{productname}管理者ロールを除くユーザのロールを変更するには、menu:ユーザ[ユーザ一覧> すべて]に移動して、変更するユーザを選択し、必要に応じて管理者ロールをオンまたはオフにします。

{productname}管理者ロールを変更するには、menu:管理者[ユーザー]に移動し、必要に応じて`{productname} Admin?`をオンまたはオフにします。


[cols="1,1", options="header"]
.ユーザロールの許可
|===
| Role Name | 説明
| {productname}管理者 | 他のユーザの権限を変更するなど、すべての機能を実行できます。
| 組織の管理者 | アクティベーションキー、設定、チャネル、システムグループを管理します。
| アクティベーションキーの管理者 | アクティベーションキーを管理します。
| イメージ管理者 | イメージプロファイル、ビルド、ストレージを管理します。
| 設定管理者 | システム設定を管理します。
| チャンネル管理者 | ソフトウェアチャネルを管理し、チャネルを全体で購読可能にしたり、新しいチャネルを作成したりします。
| システムグループ管理者 | システムグループを管理し、システムグループの作成や削除、既存グループへのクライアントの追加、グループへのユーザーアクセスの管理を行います。
| 一般ユーザー | 標準レベルのアクセスを提供します。新しく作成されたユーザーは自動的にこの役割に割り当てられます。
|===



== 追加ロールの作成

{productname}のロールベースのアクセス制御を使用すると、ユーザーの権限を微調整するために追加のロールを作成できます。
これらの役割に関する詳細情報はxref:administration:role-based-access-control.adoc[]を参照してください。



== ユーザ許可とシステム

クライアントを管理するシステムグループを作成している場合は、管理するグループをユーザに割り当てることができます。

ユーザーをシステムグループに割り当てるには、menu:ユーザー[ユーザーリスト]に移動し、ユーザー名をクリックして編集し、`System Groups`タブに移動します。
割り当てるグループをチェックし、btn:[デフォルトを更新する]をクリックします。

ユーザーのために1つ以上のデフォルトのシステムグループを選択することもできます。
ユーザーが新しいクライアントを登録すると、それはデフォルトで選択されたシステムグループに割り当てられます。
これにより、ユーザーは新しく登録されたクライアントにすぐにアクセスできます。

外部グループを管理するには、menu:ユーザー[システムグループ設定]に移動し、`External Authentication`タブに移動します。
新しい外部グループを作成するには、btn:[外部グループを作成]をクリックします。
グループに名前を付け、適切なシステムグループに割り当てます。

システムグループに関する詳細情報はxref:reference:systems/system-groups.adoc[]を参照してください。

== ユーザとチャンネルの許可

チャンネルからコンテンツを消費するサブスクライバとして、またはチャンネル自体を管理できる管理者として、ユーザを組織内のソフトウェアチャンネルに割り当てることができます。

ユーザーをチャンネルに登録するには、menu:ユーザー[ユーザーリスト]に移動し、ユーザー名をクリックして編集し、menu:チャンネルの権限[サブスクリプション]タブに移動します。
割り当てるチャンネルを確認し、btn:[権限を更新]をクリックします。

ユーザーにチャンネル管理権限を付与するには、menu:ユーザー[ユーザーリスト]に移動し、ユーザー名をクリックして編集し、menu:チャンネルの権限[管理]タブに移動します。
割り当てるチャンネルを確認し、btn:[権限を更新]をクリックします。

リスト内の一部のチャンネルは、登録できない場合があります。
これは通常、ユーザーの管理者ステータスまたはチャンネルのグローバル設定が原因です。



== ユーザのデフォルト言語

新しいユーザーを作成する際に、{webui}に使用する言語を選択できます。
ユーザーが作成された後、menu:ホーム[マイ設定]に移動することで言語を変更できます。

デフォルトの言語は、`rhn.conf`設定ファイルで設定します。
デフォルトの言語を変更するには、`/etc/rhn/rhn.conf`ファイルを開き、この行を追加または編集します:

----
web.locale = <LANGCODE>
----

パラメータが設定されていない場合、デフォルトの言語は`en_US`です。

これらの言語は{productname}で利用可能です。

=== ユーザデフォルトのインタフェーステーマ

デフォルトでは、{productname} {webui}は、インストールされている製品に適したテーマを使用します。
テーマを変更して、{uyuni}または{susemgr}の色を反映させることができます。
{susemgr}テーマには、ダークオプションも用意されています。

`rhn.conf`設定ファイルでデフォルトテーマを変更できます。
デフォルトテーマを変更するには、`/etc/rhn/rhn.conf`ファイルを開き、この行を追加または編集します:

----
web.theme_default = <THEME>
----

[[webui-themes]]
.使用可能なWebUIテーマ
[cols="1,1,1", options="header"]
|===
| テーマ名     | 色    | スタイル
| `suse-light` | {susemgr} | ライト
| `suse-dark`  | {susemgr} | ダーク
| `uyuni`      | {uyuni}   | ライト
|===
