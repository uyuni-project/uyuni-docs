= タスクスケジュール
:description: SUSE Multi-Linux Managerで事前定義されたタスクスケジュールを管理および構成し、定期的なメンテナンスタスクを自動化する方法を学びます。
:revdate: 2026-04-20
:page-revdate: {revdate}
:page-author: SUSE 製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1


すべての事前定義されたタスクスケジュールは、menu:Admin[Task Schedules]の下に一覧表示されます。

image::admin_task_schedules.png[scaledwidth=80%]


menu:{productname}スケジュール[スケジュール名]をクリックして、menu:スケジュール名[基本的なスケジュール詳細]を開きます。
それを無効にするか、頻度を変更できます。

btn:[スケジュールの編集]をクリックして、設定でスケジュールを更新します。

右上隅のbtn:[スケジュールを無効にする]をクリックして、スケジュールを無効にします。

[WARNING]
====
本当に必要だと確信している場合にのみスケジュールを無効にしてください。なお、{productname}が正しく動作するためには、このスケジュールが不可欠です。
====

タスクが無効化されると、リストに表示され続けます。
menu:{productname}スケジュール[スケジュール名]をクリックすると、btn:[スケジュールを有効化]をクリックすることで、ジョブを再度有効化できます。


バンチ名をクリックすると、そのバンチタイプのランとそのステータスのリストが表示されます。

開始時間のリンクをクリックすると、menu:スケジュール名[基本的なスケジュール詳細]に戻ります。



== 事前定義済みのタスクバンチ


次の事前定義済みのタスクバンチはデフォルトでスケジュールされており、設定できます。

menu:auto-errata-default[]::
必要に応じて自動エラータ更新をスケジュールします。

menu:channel-repodata-default[]::
リポジトリメタデータファイルを(再)生成します。

menu:cleanup-data-default[]::
古いパッケージ変更ログをクリーンアップし、データベースから時系列データを監視します。

menu:clear-taskologs-default[]::
ジョブタイプに応じて、指定した日数よりも古いタスクエンジン(taskomatic)履歴データをデータベースからクリアします。

menu:cobbler-sync-default[]::
{productname}から{cobbler}への配布およびプロファイルデータを同期します。
{cobbler}によって提供される自動インストールに関する詳細は、xref:client-configuration:autoinst-intro.adoc[]を参照してください。

menu:compare-configs-default[]::
設定チャネルに保存されている設定ファイルと、すべての設定が有効なサーバーに保存されているファイルを比較します。
比較を確認するには、menu:システム[]タブをクリックし、興味のあるシステムを選択します。
menu:設定[ファイルを比較]に移動します。
詳細については、xref:reference:systems/system-details/sd-configuration.adoc#sd-config-compare-files[]を参照してください。

menu:cve-server-channels-default[]::
menu:Audit[CVE Audit]ページに表示される結果に使用される内部の事前計算されたCVEデータを更新します。
menu:Audit[CVE Audit]ページの検索結果は、このスケジュールの最後の実行に更新されます。
詳細については、xref:reference:audit/audit-cve-audit.adoc[]を参照してください。

menu:daily-status-default[]::
関連するアドレスに毎日の報告メールを送信します。
特定のユーザーの通知を設定する方法についての詳細は、xref:reference:users/user-details.adoc[]を参照してください。

menu:errata-advisory-map-sync-default[]::
内部の{suse}パッチベンダーアドバイザリーデータベーステーブルを更新します。
利用可能な場合、{suse}によって提供された元のアドバイザリーは、各パッチ詳細のベンダーアドバイザリーセクションに表示されます。

menu:errata-cache-default[]::
各サーバーの更新が必要なパッケージを検索するために使用される内部パッチキャッシュデータベーステーブルを更新します。
また、特定のパッチに興味があるかもしれないユーザーに通知メールを送信します。
パッチに関する詳細については、xref:reference:patches/patches-menu.adoc[]を参照してください。

menu:errata-queue-default[]::
自動更新(パッチ)を受信するように設定されているサーバのキューに入れます。

menu:gatherer-matcher-default[]::
仮想ホストマネージャーで設定されたVirtual Host Gathererを実行して、仮想ホストデータを収集します。
更新されたデータが利用可能になると、サブスクリプションマッチャーのジョブが実行されます。

menu:kickstart-cleanup-default[]::
古い{kickstart}セッションデータをクリーンアップします。

menu:kickstartfile-sync-default[]::
設定ウィザードによって作成された{kickstart}プロファイルに対応する{cobbler}ファイルを生成します。

menu:mgr-forward-registration-default[]::
クライアント登録データを{scclongform}と同期します。
デフォルトでは、新しい、変更された、または削除されたクライアントデータが転送されます。
`/etc/rhn/rhn.conf` で設定された同期を無効にするには、次を実行します。
+
----
server.susemanager.forward_registration = 0
----
+

[WARNING]
====
{scc}とのデータ同期を無効にすると、RMT、SMT、{productname}、および{scc}に直接登録されたクライアント間の管理対象クライアントの可視性が低下します。

データの同期により、登録されたすべてのクライアントの統一されたビューが確保されます。

https://suselinux.fra1.qualtrics.com/jfe/form/SV_0ooNnrY0rYuQScS[サービス向上のためにオプトアウトの理由を共有してください]
====

menu:mgr-sync-refresh-default[]::
{scclongform} (`mgr-sync-refresh`) と同期します。
デフォルトでは、すべてのカスタムチャネルもこのタスクの一部として同期されます。
カスタムチャネルの同期に関する詳細は、xref:administration:custom-channels.adoc#_custom_channel_synchronization[]を参照してください。

menu:minion-action-chain-cleanup-default[]::
古い動作チェーンデータをクリーンアップします。

menu:minion-action-cleanup-default[]::
ファイルシステムから古いクライアントアクションデータを削除します。
最初に、{salt}ジョブキャッシュに保存されている対応する結果を調べて、未完了のアクションを完了しようとします。
サーバーがアクションの結果を見逃した場合、未完了のアクションが発生する可能性があります。
正常に完了したアクションについては、実行されたスクリプトファイルなどのアーティファクトを削除します。

menu:minion-checkin-default[]::
クライアントに対して定期的なチェックインを実行します。

menu:notifications-cleanup-default[]::
期限切れの通知メッセージをクリーンアップします。

menu:oval-data-sync-default[]::
CVE監査クエリの精度を向上させるために必要なOVALデータを生成します。

menu:package-cleanup-default[]::
ファイルシステムから古いパッケージファイルを削除します。

menu:reboot-action-cleanup-default[]::
6時間以上保留中の再起動アクションは失敗としてマークされ、関連データはデータベースからクリーンアップされます。
再起動アクションのスケジューリングに関する詳細は、xref:reference:systems/system-details/sd-provisioning.adoc#sd-power-management[]を参照してください。

menu:session-cleanup-default[]::
古いWebインタフェースセッションをクリーンアップします。通常は、ユーザがログインし、ログアウトする前にブラウザを閉じたときに一時的に保存されるデータです。

menu:ssh-service-default[]::
クライアントが{productname}連絡方法で設定されている場合、クライアントにSSH経由で`SSH Push`にチェックインするようにプロンプトを表示します。
再起動後にアクションチェーンも再開します。

menu:system-overview-update-queue-default[]::
システム概要データを更新します。

menu:system-profile-refresh-default[]::
すべてのシステムでハードウェアのリフレッシュを実行します。
これは月に一度のみ発生し、{productname}サーバーへの負荷が増加する可能性があります。
このジョブはxref:specialized-guides:salt/salt-rate-limiting.adoc[Salt Rate Limiting]を使用します。
バッチサイズの調整については、xref:specialized-guides:large-deployments/tuning.adoc#java-salt-batch-size[]を参照してください。

menu:token-cleanup-default[]::
{salt}クライアントがパッケージとメタデータをダウンロードするために使用する期限切れのリポジトリトークンを削除します。

menu:update-payg-default[]::
構成された{payg}のクラウドインスタンスから認証データを収集します。

menu:update-reporting-default[]::
ローカルレポーティングデータベースを更新します。

menu:update-reporting-hub-default[]::
周辺機器{productname}サーバからすべてのレポーティングデータを収集し、ハブレポーティングデータベースを更新します。

menu:update-system-overview-default[]::
システム概要データが最新であることを定期的に確認します。

menu:uuid-cleanup-default[]::
古いUUIDレコードをクリーンアップします。
