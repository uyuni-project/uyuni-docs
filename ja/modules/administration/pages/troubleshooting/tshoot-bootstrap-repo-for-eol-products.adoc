[[troubleshooting-bootstrap-repo-for-eol-products]]
= サポート終了製品のブートストラップリポジトリのトラブルシューティング
:description: サービス終了製品の手動ブートストラップリポジトリを構成し、それらを一覧表示および作成するには、mgr-create-bootstrap-repoコマンドを使用します。
:revdate: 2024-10-31
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメントチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

サポートされている製品が同期されると、ブートストラップリポジトリは{productname}サーバー上で自動的に作成および再生成されます。
製品がサービス終了に達し、サポート対象外になると、製品を引き続き使用する場合は、ブートストラップリポジトリを手動で作成する必要があります。

ブートストラップリポジトリの詳細については、xref:client-configuration:bootstrap-repository.adoc[]を参照してください。



.プロシージャ:サービス終了製品のブートストラップリポジトリの作成

. {productname}コンテナホストのコマンドプロンプトで、ルートとしてサーバコンテナに入ります。
+

----
mgrctl term
----

. コンテナ内で次の手順を実行します。
+

--
.. `--force`オプションを指定すると、使用可能なサポート対象外のブートストラップリポジトリが一覧表示されます。たとえば：
+

----
mgr-create-bootstrap-repo --list --force
1. SLE-12-SP2-x86_64
2. SLE-12-SP3-x86_64
----

.. 製品ラベルとして適切なリポジトリ名を使用して、ブートストラップリポジトリを作成します。
+

----
mgr-create-bootstrap-repo --create SLE-12-SP2-x86_64 --force
----
--

ブートストラップリポジトリを手動で作成しない場合、必要な製品およびブートストラップリポジトリでLTSSが使用できるかどうかを確認できます。
