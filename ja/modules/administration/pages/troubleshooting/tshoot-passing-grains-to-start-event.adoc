[[troubleshooting-passing-grains-to-start-event]]
= grainを開始イベントに渡す場合のトラブルシューティング
:description: Saltの新機能を使用して、スタートイベントにグレインを渡す際の問題をトラブルシューティングする方法を学びます。
:revdate: 2022-10-18
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

Saltクライアントが起動するたびに、`machine_id`グレインが{productname}に渡されます。{productname}はこのグレインを使用して、クライアントが登録されているかどうかを判断します。
このプロセスには、同期Saltコールが必要です。同期Saltコールは他のプロセスをブロックするため、同時に多くのクライアントが起動すると、プロセスが大幅な遅延を引き起こす可能性があります。

この問題を克服するために、別々の同期Saltコールを回避するための新しい機能がSaltに導入されました。

この機能を使用するには、この機能をサポートしているクライアントのクライアント設定に設定パラメータを追加できます。

この処理を簡単にするために、`mgr_start_event_grains.sls`ヘルパーSaltステートを使用できます。

[NOTE]
====
これは、すでに登録されているクライアントにのみ適用されます。
最近Saltクライアントを登録した場合、この設定パラメータはデフォルトで追加されます。
====


{productname}サーバのコマンドプロンプトで、次のコマンドを使用して`start_event_grains`設定ヘルパーを有効にします：

----
salt '*' state.sls util.mgr_start_event_grains
----

このコマンドは、クライアントの設定ファイルに必要な設定を追加し、クライアントが再起動されるときに適用されます。
多数のクライアントがいる場合は、代わりにバッチモードでコマンドを実行できます：

----
salt --batch-size 50 '*' state.sls mgr_start_event_grains
----
