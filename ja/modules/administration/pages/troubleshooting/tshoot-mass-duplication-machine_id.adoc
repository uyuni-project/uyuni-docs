[[troubleshooting-mass-machineID-duplication]]
= 大量のMachine_idの重複
:description: SUSE Multi-Linux ManagerでVMクローンを管理するには、マシンIDを変更したり、偽のIDを生成したりする技術を使用します。
:revdate: 2025-10-23
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

仮想マシンを管理するために{productname}を使用している場合、VMのクローンを作成することが役立つかもしれません。
クローンとは、既存のディスクの正確なコピーであるプライマリディスクを使用するVMです。

仮想マシンのクローンを作成すると時間を大幅に節約できますが、ディスク上の識別情報が重複しているために問題が発生する可能性があります。

一方が他方のクローンである2台のマシンを登録しようとすると、{productname}がそれらを2つの別々のクライアントとして登録することを望むかもしれません。
ただし、元のクライアントとクローンの両方のマシンIDが同じ場合、{productname}は両方のクライアントを1つのシステムとして登録し、2台目のマシンが登録されるとデータが上書きされます。

これは、クローンのマシンIDを変更することで解決でき、{productname}がそれらを2つの異なるクライアントとして認識できるようになります。
マシンIDを変更する方法に関する詳細情報と手順については、xref:administration:troubleshooting/tshoot-registerclones.adoc[]を参照してください。

ただし、状況によっては、すべてのシステムでマシンIDを変更することが実行可能でも実用的でもない場合があります。これは、既存のマシンIDに依存する他のアプリケーションに影響を与える可能性があるためです。その場合、{productname}のコンテキストで使用するために偽のマシンIDを生成するオプションが利用可能です。

.プロシージャ:登録時に{productname}のためのマシンIDを生成する
[role=procedure]
____

. 起動するスクリプトを作成します（すでに持っている場合はこのステップをスキップしてください）。詳細については、xref:client-configuration:registration-bootstrap.adoc[]を参照してください。

. ファイルを保存し、エディタを終了します。

+

----
GENERATE_OWN_MACHINEID=1
----

. 厳密なブートスクラップの使用を開始すると、マシンIDが競合しなくなります。
