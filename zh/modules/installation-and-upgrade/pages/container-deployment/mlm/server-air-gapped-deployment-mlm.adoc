== 什么是隔离的部署？


隔离部署是指任何网络系统的设置和操作，该系统与不安全的网络（尤其是互联网）物理隔离。
这种类型的部署通常用于高安全性环境，如军事设施、金融系统、关键基础设施，以及处理敏感数据并必须保护免受外部威胁的任何地方。



== 部署

{productname} 支持两种部署变体。


=== 通过虚拟机部署

建议的安装方法是使用所提供的 {productname} 虚拟机映像选项，因为所需的全部工具和容器映像都已预先加载并且随时可用。

有关安装 {productname} 服务器虚拟机的更多信息，请参见 xref:container-deployment/mlm/server-deployment-vm-mlm.adoc[将服务器作为虚拟机部署]。

要升级 {productname} 服务器，用户应升级系统中的所有软件包，并遵循 xref:container-management/updating-server-containers.adoc[服务器升级] 中定义的程序。


=== 在 {productname} 上部署 {sl-micro}

{productname} 还提供了所有必需的容器映像，这些映像以 RPM 包的形式提供，可在系统上安装。

[NOTE]
====
用户应在内部网络上提供所需的 RPM。
这可以通过使用第二个 {productname} 服务器或 RMT 服务器来完成。
====

.过程：升级在隔离的部署中，于 {productname} 上安装 {sl-micro}。
[role=procedure]
____
. 安装 {sl-micro}。

. 更新系统。

. 安装工具软件包和映像包（将 $ARCH$ 替换为正确的体系结构）

+

[source,shell]
----
transactional-update pkg install mgradm* mgrctl* suse-multi-linux-manager-5.1-$ARCH$-server-*
----

+

. 重引导。

. 使用 {productname} 部署 `mgradm`。

____

有关在 {productname} 上安装 {sl-micro} 服务器的更详细信息，请参见 xref:container-deployment/mlm/server-deployment-mlm.adoc[将服务器作为虚拟机部署]。

要升级 {productname} 服务器，用户应升级系统中的所有软件包，并遵循 xref:container-management/updating-server-containers.adoc[服务器升级] 中定义的程序。


== PTF

PTF 映像不以软件包形式提供。
这意味着需在连接到互联网的计算机上使用 `podman` 拉取映像，然后将其保存为存档文件，传输至隔离的计算机并在该计算机上加载。

.过程：升级在连接到互联网的计算机上拉取映像
[role=procedure]
____
. 安装 `podman`。

. 使用 SCC 身份凭证向 {suse} 注册表进行身份验证：

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-login-into-suse-registry.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../partials/snippet-login-into-suse-registry.adoc[]
endif::[]

+

. 运行以下命令部署 \{productname}：

+

[NOTE]
====
此示例假设 PTF 是针对 `server` 映像的，如果 PTF 涉及其他映像，请调整示例。
====

+

[source,shell]
----
SCC_USERID=aXXXX
PTFID=12345
echo "registry.suse.com/a/$SCC_USERID/$PTFID/suse/multi-linux-manager/5.1/x86_64/server:latest-ptf-$PTFID" >>/tmp/ptf-images
----

+


. 验证文件是否包含 PTF 映像的完整路径。
  如有需要，可以将其他映像的完整路径添加到文件中，每行一个。

. 拉取 PTF 的每个容器映像，然后将其保存为 tar 归档。


+

[source,shell]
----
for image in `cat /tmp/ptf-images`; do
    podman pull $image
done
podman save -o /tmp/ptf-images.tar $(cat /tmp/ptf-images)
----
. 将服务器上的 `/tmp/ptf-images.tar` 映像存档传输至补丁服务器。

____

.过程：升级在要修补的服务器上加载映像。
[role=procedure]
____
. 确保 `ptf-images.tar` 文件在服务器上可用。

. 从存档中加载映像：

+

[source,shell]
----
podman load -i ptf-images.tar
----

+

. 按照联网环境的操作方式安装 PTF，使用以下命令：

+

[shell, source]
----
mgradm support ptf podman [--test|--ptf] NUMBER --user SCC_ACCOUNT
----

+

由于映像已加载，系统不会再次拉取映像。
____
