[[minimmal.kvm.settings]]
== 初始 KVM 设置

.过程：升级创建初始设置
. 使用下载的 {minimal} KVM 映像创建一个新虚拟机，并选择 `Import existing disk image`。
. 配置 RAM 和 CPU 数量（至少 16 GB RAM，4 个 CPU）。
. 为您的 KVM 计算机命名。
. 单击 btn:[开始安装] 以从映像引导。
. 在 JeOS 首次引导屏幕上选择“开始”以继续。
+

image::jeos-firstboot.png[]

. 选择键盘布局。
+

image::select-keyboard.png[]

.  接受许可协议。
+

image::accept-license-agreement.png[]

. 选择您的时区。
+

image::enter-timezone.png[]

. 输入 root 口令。
+

image::root-password.png[]

. 安装完成后，以 root 身份登录。

. 继续阅读下一节。

.过程：升级注册 {sl-micro} 和 {productname} {productnumber}
. 引导虚拟机。
. 以 `root` 身份登录。
+
// note for 5.0 backport: 5.0 should have sle-micro
. 使用 SCC 注册 {sl-micro}。
+

----
transactional-update register -r <REGCODE> -e <your_email>
----

. 重引导。

. 使用 注册 {productname} {productnumber} {scclongform}。
+

----
transactional-update register -p Multi-Linux-Manager-Server/5.1/x86_64 -r <REGCODE>
----

. 重引导。

. 更新系统：
+

----
transactional-update
----

. 如果已应用更新，请重引导。

. 该步骤可选。
  但是，如果您的基础设施需要自定义的永久性存储，请使用 `mgr-storage-server` 工具。

+

有关详细信息，请访问 `mgr-storage-server --help`。
有关详细信息，请参见 [command]\\``mgr-storage-server \--help``。此工具可以简化容器存储和数据库卷的创建。
如下所示使用命令：

+

----
mgr-storage-server <storage-disk-device> [<database-disk-device>]
----
+
例如：
+

----
mgr-storage-server /dev/nvme1n1 /dev/nvme2n1
----

+
[NOTE]
====
此命令会将 `/var/lib/containers/storage/volumes` 处的永久性存储卷移动到指定的存储设备。

有关详细信息，请参见

* xref:installation-and-upgrade:container-management/persistent-container-volumes.adoc[]
* xref:administration:troubleshooting/tshoot-container-full-disk.adoc[]
====
