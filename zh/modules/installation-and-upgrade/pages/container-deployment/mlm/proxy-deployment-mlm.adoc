本指南概述了在 {sl-micro} {microversion} 或 {sles} {bci-mlm} 上部署 {productname} {productnumber} 代理容器的过程。
本指南假设您已经成功部署了 {productname} {productnumber} 服务器。

[IMPORTANT]
====
目前仅支持将 {sl-micro} 作为常规 minion（`default` 联系方式）。
我们也在努力将其作为 Salt SSH 客户端（`salt-ssh` 联系方式）进行管理。
====

[NOTE]
====
可以将现有客户端转换为代理。
有关详细信息，请参见 xref:installation-and-upgrade:container-deployment/mlm/proxy-conversion-from-client-mlm.adoc[]。
====

要成功部署新的代理，请按照以下程序操作：


// Add certificate tasks item
.过程：升级部署代理
[role=procedure]
_____

. 查看硬件要求。

. 在服务器上同步 {sl-micro} {microversion} 或 {sles} {bci-mlm} 父通道和代理扩展子通道。

. 在裸机上安装 {sl-micro} 或 {sles}。

. 创建带有代理扩展的 {salt} 激活密钥。

. 使用 `default` 连接方法将代理启动为客户端。

. 生成代理配置。

. 将服务器中的代理配置传输到代理。

. 使用 \{webui} 安装软件包

. 使用代理配置将客户端作为代理注册到 {productname}。

_____

.代理容器主机支持的操作系统
[NOTE]
====
容器主机支持的操作系统为 {sl-micro} {microversion} 和 {sles} {bci-mlm}。

容器主机::
容器主机是配备有容器引擎（如 Podman）的服务器，允许其管理和部署容器。
这些容器包含应用程序及其基本部分，如库，但不包含完整的操作系统，从而使其轻量化。
此设置确保应用程序在不同环境中以相同方式运行。
容器主机提供这些容器所需的资源，如处理器、内存和存储。
====

== 同步父通道和代理扩展子通道

本节假定您已在服务器的 menu:管理员[设置向导> 组织凭据] {webui} 下输入了您的组织凭据。
产品列在 menu:管理员[设置向导> 产品] 页面上。
此通道必须在服务器上完全同步，并选择子通道 `Proxy` 作为扩展选项。


.过程：升级同步父通道和代理扩展
[role=procedure]
_____

. 在 {productname} {webui} 中选择 menu:管理员[产品]。

. 在产品页面的过滤器字段中输入 {sl-micro} 或 {sles}。

. 接下来，在下拉列表中选择所需的体系结构。
  在本示例中 {x86_64}。

. 在 `Product Description` 字段中，选中 {sl-micro} {microversion} 或 {sles} {bci-mlm} 复选框，然后使用下拉菜单选择 `{productname} Proxy Extension 5.1 x86_64` 扩展。

. 单击 btn:[添加产品] 按钮。

. 等待同步完成。

_____

== 配置自定义永久性存储

配置永久性存储是可选的，但这是避免容器因磁盘已满而引发严重问题的唯一方法。
如果您的基础设施需要自定义的永久性存储，请使用 `mgr-storage-proxy` 工具。

* 有关详细信息，请访问 `mgr-storage-proxy --help`。
此工具可以简化容器存储和 Squid 缓存卷的创建。

如下所示使用命令：

[source,shell]
----
mgr-storage-proxy <storage-disk-device>
----

例如：

[source, shell]
----
mgr-storage-proxy /dev/nvme1n1
----



[NOTE]
====
此命令将在 `/var/lib/containers/storage/volumes` 处创建持久存储卷。

有关详细信息，请参见

* xref:installation-and-upgrade:container-management/persistent-container-volumes.adoc[]
* xref:administration:troubleshooting/tshoot-container-full-disk.adoc[]
====

== 启动 {productname} 代理

现在可以使用 `mgrpxy` 命令启动容器：

.过程：升级启动并检查代理状态
[role=procedure]
_____

. 调用以下命令启动代理：

+

----
mgrpxy start
----

. 调用以下命令检查容器状态：

+

----

----

应该会显示五个 {productname} 代理容器，并且它们应该属于 `proxy-pod` 容器 Pod：

* proxy-salt-broker
* proxy-httpd
* proxy-tftpd
* proxy-squid
* proxy-ssh

== 为服务使用自定义容器映像

默认情况下，{productname} 代理套件配置为对其每个服务使用相同的映像版本和注册表路径。
但是，可以使用以`-tag`和`-image`结尾的安装参数覆盖特定服务的默认值。

例如：

[source, shell]
----
mgrpxy install podman --httpd-tag 0.1.0 --httpd-image registry.opensuse.org/uyuni/proxy-httpd /path/to/config.tar.gz
----

它会在重启httpd服务之前调整其配置文件，其中`registry.opensuse.org/uyuni/proxy-httpds`是要使用的镜像，`0.1.0`是版本标签。

要重置为默认值，请再次运行 install 命令但不要指定这些参数：

[source,shell]
----
mgrpxy install podman /path/to/config.tar.gz
----

此命令首先将所有服务的配置重置为全局默认值，然后重新装载配置。
