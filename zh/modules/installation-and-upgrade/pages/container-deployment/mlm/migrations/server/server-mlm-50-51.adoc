== 要求和注意事项

=== SSL 证书

[NOTE]
====
从 SUSE Multi-Linux Manager 5.1 开始，PostgreSQL 数据库在一个单独的容器中运行，该容器在迁移过程中需要自己的 SSL 证书。
这就是为什么即使服务器证书和 CA 链从源复制，数据库证书仍然是必需的。
====

== 客户端工具品牌更新

{productname} {productnumber} 引入了一套重新品牌的客户端工具，适用于所有受支持的操作系统。此过渡过程无缝，进行新产品同步的用户只会注意到更新的通道名称。

名为 `SUSE Manager Client Tools for XYZ` 的通道（由之前注册到 {productname} 4.3 或 5.0 的客户端使用）在 5.1 版本中已不再可用，并且在 5.1 中将不再接收更新。

尽管迁移后旧版通道仍会分配给现有客户端，但对应的储存库已被去除。

为确保客户端能持续接收更新，用户必须执行以下操作：

* 镜像相关产品的新 `SUSE Multi-Linux Manager Client Tools for XYZ` 通道，并将其分配给相应的客户端。
* 取消分配过时的 `SUSE Manager Client Tools for XYZ` 通道。

这也意味着，任何基于旧客户端工具的 CLM 项目都必须相应调整。

有关示例工作流程，请参见 xref:common-workflows:workflow-switch-to-new-client-tools-channels.adoc[]。



== {sle-micro} 5.5 到 {sl-micro} 6.1



本文件提供了将部署了 {sle-micro} 5.0 服务器的 {productname} 5.5 主机升级到 {sl-micro} 6.1 并迁移到 {productname} {productnumber} 的测试程序。


=== 先决条件

* {productname} 5.0 已安装并在 {sle-micro} 5.5 上运行。
* 系统已注册到 {scc} 且拥有有效的订阅。


=== 发行套件升级和服务器迁移

.过程：升级从 {productname} 5.0 迁移到 {productname} {productnumber}
[role=procedure]
____

. 校验当前产品状态

+

[source,console]
----
SUSEConnect --status-text
----

+

确认以下信息：

* 基础操作系统：`SUSE Linux Enterprise Micro 5.5`

* 扩展：`SUSE Manager Server 5.0 Extension`

+

. 确保系统已更新

+

[source,console]
----
transactional-update patch
----

+

--

* 如果已应用补丁，请在继续迁移前\\**重引导系统**：

+

[source,console]
----
mgradm stop
reboot
----

* 如果未发现更新，可直接进入迁移步骤。

--

+

. 启动迁移流程：

+

[source,console]
----
transactional-update migration --auto-agree-with-licenses --gpg-auto-import-keys
----

+

按照提示操作，选择可用的迁移目标：**SUSE Linux Micro 6.1** 和 **{productname} Server Extension 5.1**。

+

. 停止服务器后重引导，使更改生效。

+

[source,console]
----
mgradm stop
reboot
----

+

. 重引导后的检查：

+

验证升级后的操作系统和 {productname} 扩展：

+

[source,console]
----
cat /etc/os-release
SUSEConnect --status-text
----

+

您应看到以下内容：

* `PRETTY_NAME="SUSE Linux Micro 6.1"`

* `SUSE Multi-Linux Manager Server 5.1 Extension`

+

. 启用 root SSH 访问（_如有需要_）。
+

SL Micro 6.1 默认禁用通过 SSH 的根登录。
+
编辑 `/etc/ssh/sshd_config.d/sshd.conf`：

+

[source,console]
----
PermitRootLogin yes
----

+

重启动 spacewalk 服务：

+

[source,shell]
----
systemctl restart sshd
----

+

有关更多信息，请参见 xref:administration:troubleshooting/tshoot-remote-root-on-micro.adoc[]。

+

. 验证 {productname} 工具

+

[source,console]
----
mgradm --version
----

+

预期输出：

+

* 版本 `5.1.11` 或更高版本

* 参考 `5.1.0` 或更高版本

. 升级服务器容器

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

按照提示操作，然后等待设置过程完成。


. 检查正在运行的容器：

+

[source,console]
----
podman ps
----

+

您应看到以下内容：

* `server:5.1.0` 或更高版本

* `server-postgresql:5.1.0` 或更高版本

____


[NOTE]
====
如果之前未安装 `uyuni-db` 或 `saline` 等服务，升级过程中出现的相关缺失服务错误可忽略。
====


=== 迁移完成

系统现在在 SL Micro 6.1 上运行 {productname} 5.1。
在恢复生产操作之前，请验证您的设置。
如果您有一个连接到此服务器的 {productname} 5.0 代理，请参阅 xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-51.adoc[] 指南以升级代理主机。

== {sles} 15 SP6 升级到 15 SP7

本文提供了经过测试的步骤，用于将部署了 SUSE Multi-Linux Manager 5.0 服务器的 SUSE Linux Enterprise Server 15 SP6 主机升级到部署了 SUSE Multi-Linux Manager 5.1 的 SUSE Linux Enterprise Server 15 SP7。


=== 先决条件

* SUSE Multi-Linux Manager 5.0 已在 SUSE Linux Enterprise Server 15 SP6 上安装并运行。
* 系统已注册到 SUSE Customer Center (SCC) 且拥有有效的订阅。
* 继续操作前请务必创建备份。



=== 发行套件升级和服务器迁移

.过程：升级 从 {productname} 5.0 迁移到 {productname} {productnumber}
[role=procedure]
____

. 校验当前产品状态

+

[source,console]
----
SUSEConnect --status-text
----

+

确认以下信息：

+

* 基础操作系统：`SUSE Linux Enterprise Server 15 SP6`

* 扩展：`SUSE Manager Server 5.0 Extension`

+

. 应用所有系统补丁

+

[source,console]
----
zypper patch
----

+

. 如果更新了更新栈，则停止服务器后重引导。

+

[source,console]
----
mgradm stop
reboot
----

+

. 启动 Zypper 迁移工具

+

[source,console]
----
zypper migration
----

+

Zypper 会显示可能的迁移目标以及详细摘要。

+

. 选择适当的目标，并按照提示完成迁移。

. 完成迁移后，重引导系统：

+

[source,console]
----
mgradm stop
reboot
----

+

. 重引导后的检查：

+

验证升级后的操作系统和 {productname} 扩展。
+

[source,console]
----
cat /etc/os-release
SUSEConnect --status-text
----

+

您应看到以下内容：

* `VERSION="15-SP7"`

* `SUSE Multi-Linux Manager Server 5.1 Extension for SLE`

+

. 验证 {productname} 工具版本。

+

[source,console]
----
mgradm --version
----

+

预期输出：

* 版本 `5.1.11` 或更高版本

* 镜像标签 `5.1.0` 或更高版本

+

. 升级服务器容器

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

按照提示拉取新的容器映像并重新配置环境。

+

. 检查正在运行的容器：

+

[source,console]
----
podman ps
----

+

更新容器

* `server:5.1.0` 或更高版本

* `server-postgresql:5.1.0` 或更高版本

____


=== 迁移完成

系统现在在 SUSE Linux Enterprise Server 15 SP7 上运行 {productname} 5.1。
在恢复生产操作之前，请验证您的设置。
如果您有一个连接到此服务器的 {productname} 5.0 代理，请按照 xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-51.adoc[] 进行操作。
