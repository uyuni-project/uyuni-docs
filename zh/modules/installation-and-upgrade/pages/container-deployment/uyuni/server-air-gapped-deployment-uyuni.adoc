== 什么是隔离的部署？


隔离部署是指任何网络系统的设置和操作，该系统与不安全的网络（尤其是互联网）物理隔离。
这种类型的部署通常用于高安全性环境，如军事设施、金融系统、关键基础设施，以及处理敏感数据并必须保护免受外部威胁的任何地方。

您可以在能够访问互联网的计算机上轻松使用 `Podman` 或 `Docker` 拉取容器映像。

.过程：升级拉取映像
. 拉取所需的映像，然后将其保存为 `tar` 归档。
  例如：

+
.Podman
----
podman pull registry.opensuse.org/uyuni/server:latest
podman save --output server.tar registry.opensuse.org/uyuni/server:latest
----
+
.Docker
----
docker pull registry.opensuse.org/uyuni/server:latest
docker save --output images.tar registry.opensuse.org/uyuni/server:latest
----
+
. 将生成的 `server-image.tar` 传输到服务器容器主机，并使用以下命令加载它：
+
.装载服务器映像
----
podman load -i images.tar
----

== 在隔离环境中获取 {salt} 公式的容器映像

一些公式，如 Bind 和 DHCP (Kea)，也使用容器。
如果您计划在隔离环境中使用它们，您需要拉取它们的映像，将其保存到存档中，并在您的 {productname} 代理或其他管理系统上加载它们。

这些映像可从 `registry.opensuse.org` 获取。

.过程：升级为隔离环境获取公式映像
. 在能够访问互联网的系统上，拉取所需映像。例如：
 +
 [source,shell]
 ----
 podman pull registry.opensuse.org/opensuse/bind:latest
 podman pull registry.opensuse.org/opensuse/kea:latest
 ----
 . 从归档中加载映像：
 +
 [source,shell]
 ----
 podman save -m -o formula-images.tar registry.opensuse.org/opensuse/bind:latest registry.opensuse.org/opensuse/kea:latest
 ----
 . 将 `formula-images.tar` 文件传输到您的隔离系统。
 . 从归档中加载映像：
 +
 [source,shell]
 ----
 podman load -i formula-images.tar
 ----

=== 在 {productname} 上部署 {leapmicro}

{productname} 还以 RPM 软件包的形式提供了所有所需的容器映像，可在系统上安装。

[NOTE]
====
用户应在内部网络上提供所需的 RPM。这可以通过使用第二个 {productname} 服务器或任何类型的镜像来完成。
====

.过程：升级在隔离环境中，在 {productname} 上安装 {leapmicro}.
. 安装 {leapmicro}。
. 更新系统。
. 安装工具软件包和映像包（将 $ARCH$ 替换为正确的体系结构）

+

[source,shell]
----
transactional-update pkg install mgradm* mgrctl* uyuni-server*-image*
----
+
. 重引导。
. 使用 {productname} 部署 `mgradm`。在隔离的环境中，您可能希望使用选项 `--pullPolicy Never`。


有关在 {leapmicro} 上安装 {productname} 服务器的更详细信息，请参见 xref:container-deployment/uyuni/server-deployment-uyuni.adoc[服务器部署]。

要升级 {productname} 服务器，用户应升级系统中的所有软件包，并遵循 xref:container-management/updating-server-containers.adoc[服务器升级] 中定义的程序。
