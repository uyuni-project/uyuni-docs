. 部署 {productname}。

+

[NOTE]
====
如果您使用虚拟机镜像作为迁移目标，请在此处作为最后一步执行命令`mgradm migrate`而不是`mgradm install`。
====

+

根据 SSL 证书的变体（自签名或第三方），执行以下命令之一。
  将 `<FQDN>` 替换为您的 {productname} 服务器的完全限定的域名：

+

--
* 使用 {productname} 提供的自签名证书：

+

[source,shell]
----
mgradm install podman <FQDN>
----

* 使用第三方 SSL 证书标志导入相应 SSL 证书（如果无需全部证书，可调整示例）：

+

[source,shell]
----
mgradm install podman <FQDN> \
  --ssl-ca-intermediate <strings> \
  --ssl-ca-root <string> \
  --ssl-server-cert <string> \
  --ssl-server-key <string> \
  --ssl-db-ca-intermediate <strings> \
  --ssl-db-ca-root <string> \
  --ssl-db-cert <string> \
  --ssl-db-key <string>
----

+

有关详细信息，请访问 `mgradm install podman --help`。

--
