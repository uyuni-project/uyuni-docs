= 注册 {sle-micro} 客户端
:description: 通过同步软件通道并通过 Web UI 或命令行将 SLE Micro 客户端注册到 SUSE Multi-Linux Manager 服务器
:revdate: 2026-06-01
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

本节包含有关注册运行 {sle-micro} 操作系统 5.1、5.2、5.3、5.4 和 5.5（{x86_64}、{arm64} 和 {zsystems} (s390x) 体系结构）的客户端的信息。

{sle-micro} 使用事务更新。
事务更新具有原子性（仅当所有更新都成功时，才会应用所有更新），且支持回滚。
它们不会影响正在运行的系统，因为在系统重引导之前，任何更改都不会生效。

有关事务更新和重启的更多信息，请参见 https://documentation.suse.com/sles/15-SP7/html/SLES-all/cha-transactional-updates.html.

== 注册客户端

[WARNING]
====
{sle-micro} 客户端在注册后需要重启。

在启动过程后，{sle-micro} 客户端的自动启动被禁用。此更改是由于间歇性的自动重启，这可能发生在 {salt} 能够传递应用启动 {salt} 状态的结果之前。

尽管在注册完成后会自动安排重启，但它会遵循默认的重启管理器维护窗口。
该窗口可能在客户端注册后几个小时。
建议在注册脚本完成后手动重引导客户端，以加快注册速度并使系统出现在系统列表中。
====

== 重启 {sle-micro}

{sle-micro} 是一个事务型系统。
事务更新通常支持多种重引导方法。
建议在由 `systemd` 管理的系统中使用 {productname} 进行重启。
使用其他方法可能会导致不希望的行为。

在 {productname} 上启动事务型系统时，如果系统处于默认配置下，`systemd` 将被配置为重启方法（`REBOOT_METHOD`）。
这样的配置允许 {productname} 控制重引导操作，重引导可以立即执行或根据需要与 {productname} 安排。


=== 背景信息

默认情况下，客户端安装期间的重启方法设置为 `auto`。
使用 `auto` 启动方法时，如果服务正在运行，`rebootmgrd` 将根据配置的策略重启系统。
策略可以立即重引导或在维护窗口期间进行重引导。
有关更多信息，请参见 `rebootmgrd` (8) 手册页。
否则，如果 `rebootmgrd` 未运行，{productname} 将调用 `systemctl reboot`。

[NOTE]
====
使用任何不同于 `systemd` 的方法可能会导致意外行为。
====
