= 客户端联系方法
:description: 学习如何使用 SUSE Multi-Linux Manager 服务器配置和部署最佳的 Salt 联系方法，以满足客户端通信需求
:revdate: 2024-09-27
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

有三种联系方法可供 {productname} 服务器与客户端通信。
它们基于 {salt} 协议。

使用哪种 {salt} 联系方法取决于客户端类型和您的网络架构：

默认::
除非有特定的用户需求，否则推荐使用。
默认情况下，将部署 {salt} 软件包实现。
有关更多信息，请参见 xref:client-configuration:contact-methods-default.adoc[]。

SSH 推送::
仅在网络限制使客户端无法与服务器建立联系时才有用。
仅支持 {salt} 软件包。
此联系方法有严重的限制。
有关更多信息，请参见 xref:client-configuration:contact-methods-saltssh.adoc[]。

SSH 推送（使用隧道）::
与 SSH Push 相同，但使用安全的通信隧道。
有关更多信息，请参见 xref:client-configuration:contact-methods-saltssh-tunnel.adoc[]。


通信软件{salt}选项的实现：

Salt 捆绑包 (`venv-salt-minion`)::
是一个包含 {salt} 受控端、Python 3、所需的 Python 模块和库的单个二进制软件包。
因此，{salt} 联系方法与客户端上安装的软件无关。

+

////
Using the {salt} Bundle is the default.
It is the only supported implementation for the SSH Push or SSH Push (with tunnel) contact method.
////

+

有关更多信息，请参见 xref:client-configuration:contact-methods-saltbundle.adoc[]。

Salt minion (`salt-minion`)::
是安装在客户端系统上的 {salt} 软件，并且通常不再支持 {productname}。
如有需要，您可以仅在 {sle} 15 客户端及其衍生版本（如 {sle-micro}）上安装，仅用于启动。
然后，在启动时，它将被替换为 `venv-salt-minion`。



[IMPORTANT]
====
所谓的传统联系协议在 {productname} 5.0 及更高版本中不再受支持。
在从 {productname} 4 升级到 5 之前，任何现有的传统客户端，包括传统代理，必须迁移到 {salt} 或替换为 {salt} 代理。

有关将传统 {productname} 4 客户端迁移到 {salt} 客户端的更多信息，请参见 https://documentation.suse.com/suma/4.3/en/suse-manager/client-configuration/contact-methods-migrate-traditional.html。
====
