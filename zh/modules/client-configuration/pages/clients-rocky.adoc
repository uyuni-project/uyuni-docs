== 添加软件通道


在将 {rocky} 客户端注册到您的 {productname} 服务器之前，您需要添加所需的软件通道，并同步这些通道。

当前支持的架构有：`x86_64` 和 `aarch64`，在 9 版本中额外支持 {ppc64le} 和 {s390x}。
有关支持的产品和架构的完整列表，请参见 xref:client-configuration:supported-features.adoc[]。

ifeval::[{uyuni-content} == true]
[IMPORTANT]
====
对于 {rocky}{nbsp}10、{rocky}{nbsp}9 和 {rocky}{nbsp}8 客户端，请添加 Base 和 AppStream 通道。
您需要从这两个通道获取软件包。
如果您不添加这两个通道，由于缺少软件包，您将无法创建启动储存库。
====
endif::[]

[NOTE]
====
您可能会注意到，上游与 {productname} 通道中可用的 AppStream 软件包数量存在一些差异。
如果您在不同时间点比较同一通道，您也可能会看到不同的数字。
这是由于 {rocky} 管理其存储库的方式。{rocky} 在发布新版本时会删除旧版本的软件包，而 {productname} 则会保留所有软件包，无论其年龄。
====

如果您使用的是 {rocky} 8 的模块化通道，您必须在客户端上启用 Python 3.6 模块流。
如果您不提供 Python 3.6，则 `spacecmd` 软件包的安装将失败。

== 创建激活密钥

您需要创建与您的 {rocky} 通道关联的激活密钥。

有关激活密钥的更多信息，请参见 xref:client-configuration:activation-keys.adoc[]。



== 管理 GPG 密钥

客户端使用 GPG 密钥在安装软件包之前检查其真实性。
只有受信任的软件包才能在客户端上安装。

[IMPORTANT]
====
信任 GPG 密钥对客户端的安全性至关重要。
决定需要哪些密钥并可以信任哪些密钥是管理员的任务。
当 GPG 密钥不被信任时，软件通道无法分配给客户端。
====

有关 GPG 密钥的更多信息，请参见 xref:client-configuration:gpg-keys.adoc[]。
