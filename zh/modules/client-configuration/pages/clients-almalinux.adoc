== 添加软件通道


[NOTE]
====
将 {almalinux} 客户端注册到 {productname} 的测试是在 `enforcing` 的默认 SELinux 配置下，使用 `targeted` 策略进行的。
您无需禁用 SELinux 即可将 {almalinux} 客户端注册到 {productname}。
====

在将 {almalinux} 客户端注册到您的 {productname} 服务器之前，您需要添加所需的软件通道，并同步这些通道。

当前支持的架构有：`x86_64` 和 `aarch64`，在 9 版本中额外支持 {ppc64le} 和 {s390x}。
有关支持的产品和架构的完整列表，请参见 xref:client-configuration:supported-features.adoc[]。

ifeval::[{uyuni-content} == true]
[IMPORTANT]
====
对于 {almalinux} 10、{almalinux} 9 和 {almalinux} 8 客户端，请添加基础和应用流通道。
您需要来自两个通道的软件包。
如果您不添加两个通道，您将无法创建启动储存库，因为缺少软件包。
====
endif::[]

如果您使用模块化通道，您必须在 {almalinux} 8 客户端上启用 Python 3.6 模块流。
如果您不提供 Python 3.6，则 `spacecmd` 软件包的安装将失败。

[NOTE]
====
您可能会注意到上游和 {productname} 通道之间的应用流通道中可用软件包数量存在差异。
如果您在不同时间点比较同一通道，您也可能会看到不同的数字。
这是由于 {almalinux} 管理其储存库的方式。{almalinux} 在发布新版本时会删除旧版本的软件包，而 {productname} 则会保留所有版本，无论其年龄。
====

== 创建激活密钥

您需要创建与您的 {almalinux} 通道关联的激活密钥。

有关激活密钥的更多信息，请参见 xref:client-configuration:activation-keys.adoc[]。



== 管理 GPG 密钥

客户端使用 GPG 密钥在安装软件包之前检查其真实性。
只有受信任的软件包才能在客户端上安装。

[IMPORTANT]
====
信任 GPG 密钥对于客户端的安全性至关重要。
管理员的任务是决定哪些密钥是必要的并且可以被信任。
因为当 GPG 密钥不被信任时，软件通道无法使用，所以将通道分配给客户端的决定取决于信任该密钥的决定。
====

有关 GPG 密钥的更多信息，请参见 xref:client-configuration:gpg-keys.adoc[]。
