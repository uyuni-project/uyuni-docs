[[client-upgrades]]
= 客户端升级
:description: 使用 SUSE Multi-Linux Manager Web UI 中的 YaST 工具或 Content Lifecycle Manager 自动化并执行客户端升级。
:revdate: 2024-11-29
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

客户端采用底层操作系统的版本控制系统，需要定期升级。

对于使用 {scc} 操作系统的 {suse} 个注册客户端，您可以在 {productname} {webui} 内执行升级。
有关支持的 {sle}{nbsp}15 升级路径，请参见 {sles-base-os-documentation}/cha-upgrade-paths.html。

要将运行 SLE{nbsp}12 的客户端升级到 SLE{nbsp}15，升级是自动化的，但在开始之前需要进行一些准备步骤。
有关更多信息，请参见 xref:client-configuration:client-upgrades-major.adoc[]。

您还可以使用 Content Lifecycle Manager 自动化并执行客户端升级。
有关更多信息，请参见 xref:client-configuration:client-upgrades-lifecycle.adoc[]。

有关产品迁移的更多信息，例如服务包升级、openSUSE Leap 小版本升级或 openSUSE Leap 到 {sle} 的迁移，请参见 xref:client-configuration:client-upgrades-product-migration.adoc[]。

ifeval::[{uyuni-content} == true]

有关升级未注册的 openSUSE Leap 客户端的更多信息，请参见 xref:client-configuration:client-upgrades-uyuni.adoc[]。
endif::[]
