:description: 了解如何为 SUSE Multi-Linux Manager 服务器访问注册和配置 SUSE Linux Micro 客户端，以便获得访问权限

本节包含有关注册运行 {sl-micro} 操作系统 6.0 {x86_64}、{arm64} 和 {zsystems}（s390x）客户端的信息。

{sl-micro} 使用事务更新。
事务更新具有原子性（仅当所有更新都成功时，才会应用所有更新），且支持回滚。
它们不会影响正在运行的系统，因为在系统重引导之前，任何更改都不会生效。

有关事务更新和重引导的更多信息，请参见 https://documentation.suse.com/sles/15-SP7/html/SLES-all/cha-transactional-updates.html.

== 注册客户端

[WARNING]
====
{sl-micro} 客户端在注册后需要重引导。
尽管在注册完成后会自动安排重引导，但它会遵循默认的重引导管理器维护窗口。
该窗口可能在客户端注册后几个小时。
建议在注册脚本完成后手动重引导客户端，以加快注册速度并使系统出现在系统列表中。
====

ifndef::backend-pdf[]
include::client-configuration:partial$create_bootstrap_repo_register.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../partials/create_bootstrap_repo_register.adoc[]
endif::[]

在使用启动脚本与 {sl-micro} 系统时，请确保脚本的证书部分包含以下内容：

----
ORG_CA_CERT=RHN-ORG-TRUSTED-SSL-CERT
ORG_CA_CERT_IS_RPM_YN=0
----

直接编辑引导脚本并添加该设置，或使用以下参数创建引导脚本：

----
mgrctl exec -ti -- mgr-bootstrap --script=bootstrap-sl-micro.sh \
    --ssl-cert=/srv/www/htdocs/pub/RHN-ORG-TRUSTED-SSL-CERT
----



[[clients-slmicro-reboot]]
== 重引导 {sl-micro}

{sl-micro} 是一个事务型系统。
事务更新通常支持多种重引导方法。
建议在由 `systemd` 管理的系统中使用 {productname} 进行重启。
使用其他方法可能会导致不希望的行为。

在 {productname} 上启动事务型系统时，如果系统处于默认配置下，`systemd` 将被配置为重启方法（`REBOOT_METHOD`）。
这样的配置允许 {productname} 控制重引导操作，重引导可以立即执行或根据需要与 {productname} 安排。


=== 背景信息

默认情况下，客户端安装期间的重启方法设置为 `auto`。
使用 `auto` 启动方法时，如果服务正在运行，`rebootmgrd` 将根据配置的策略重启系统。
策略可以立即重引导或在维护窗口期间进行重引导。
有关更多信息，请参见 `rebootmgrd` (8) 手册页。
否则，如果 `rebootmgrd` 未运行，{productname} 将调用 `systemctl reboot`。

[NOTE]
====
使用任何不同于 `systemd` 的方法可能会导致意外行为。
====
