= 客户端注册
:description: 学习如何通过考虑各种注册方法和特定操作系统的要求，在 SUSE Multi-Linux Manager 服务器上有效地注册客户端。
:revdate: 2025-02-06
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

有多种方法可将客户端注册到您的 {productname} 服务器。
本节涵盖了各种可用的方法。
它还包含您打算在客户端上运行的操作系统的特定信息。

开始前，请进行以下检查：

* 注册前客户端的日期和时间已与 {productname} 服务器正确同步。
* 您已创建激活密钥。
  有关创建激活密钥的更多信息，请参见 xref:client-configuration:activation-keys.adoc[]。


.引导后处理安装程序更新通道
[IMPORTANT]
====
一旦客户端系统已启动，**安装程序更新** 通道应被去除。
标准更新通道已经包含必要的更新，使该通道变得多余。

此外，在迁移期间，该通道也不是必需的，不应使用。
====


[IMPORTANT]
====
请勿将 {productname} 服务器基础操作系统注册到 {productname} 自身。
{productname} 服务器基础操作系统必须单独管理，或通过另一台单独的 {productname} 服务器进行管理。

要管理{productname}服务器容器，请使用`mgradm`工具。

====

ifeval::[{mlm-content} == true]


[IMPORTANT]
====
从旧版 {productname} 迁移到新版本后，我们强烈建议在引入新系统之前重新生成启动脚本，以防出现任何潜在问题。
====

endif::[]
