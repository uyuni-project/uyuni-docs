本节包含有关注册运行 {leap} 操作系统的客户端的信息。{productname} 支持使用 {leap} 的 15 和 {leap} 16 客户端 {salt}。

支持启动 {leap} 客户端，并执行诸如设置仓库和更新控制文件等初始状态操作。



== 添加软件通道

在将 {leap} 客户端注册到您的 {productname} 服务器之前，您需要添加所需的软件通道，并同步这些通道。

目前支持的体系结构有 \{arm64}。

* {leap} 15: `x86_64` 和 `aarch64`。
* {leap} 16: `x86_64`、`aarch64`、`ppc64le` 和 `s390x`。

有关支持的产品和架构的完整列表，请参见 xref:client-configuration:supported-features.adoc[]。

== 管理 GPG 密钥

客户端使用 GPG 密钥在安装前检查软件包的真实性。
只有受信任的软件才能在客户端上安装。

[IMPORTANT]
====
信任 GPG 密钥对客户端的安全性至关重要。
决定哪些密钥是必要的并可以被信任是管理员的任务。
由于在未信任 GPG 密钥时无法使用软件通道，因此将通道分配给客户端的决定取决于是否信任该密钥。
====

有关 GPG 密钥的更多信息，请参见 xref:client-configuration:gpg-keys.adoc[]。
