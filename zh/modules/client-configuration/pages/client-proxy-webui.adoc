= 使用 {webui} 将客户端注册到代理
:description: 通过系统引导部分在 SUSE Multi-Linux Manager Web UI 中配置客户端，方法是通过代理服务器注册它们
:revdate: 2024-11-21
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

可以使用 {productname} 将客户端注册到 {webui} 代理。

一般来说，非 SLE 客户端和早于版本 15 的 SLE 客户端都需要启动储存库。
//A bootstrap repository offers packages for installing {salt} on clients and for registering {salt} clients.

有关创建启动储存库的信息，请参见 xref:client-configuration:bootstrap-repository.adoc[]。


.过程：升级使用 {webui} 将客户端注册到代理

. 在 {productname} {webui} 中，导航到 menu:系统[引导]。
. 在 `Host` 字段中，键入用于启动客户端的完全限定的域名 (FQDN)。
. 在 `SSH Port` 字段中，键入用于连接和启动客户端的 SSH 端口号。
    SSH 端口默认为 `22`。
. 在 `User` 字段中，键入用于登录客户端的用户名。
    默认情况下，用户名为 `root`。
. 在 `Authentication Method` 字段中，选择用于启动客户端的身份验证方法。
+
* 如果选择的是密码身份验证，请在 `Password` 字段中，键入用于登录客户端的密码。
* 如果选择的是 SSH 私用密钥身份验证，请输入私用密钥和关联的通行口令。
    该密钥的存储期截止到引导过程完成。
. 在 `Activation Key` 字段中，选择与您要用于启动客户端的软件通道关联的激活密钥。
. 在 `Proxy` 字段中，选择要注册到的代理服务器。
. 默认情况下，选中 `Disable SSH Strict Key Host Checking` 复选框。
    如此可让引导过程自动接受 SSH 主机密钥，而无需您手动进行身份验证。
. 可选：选中 `Manage System Completely via SSH` 复选框。
    如果您选中此选项，客户端将会配置为使用 SSH 来连接服务器，且不再配置其他连接方法。
. 单击 btn:[启动] 开始注册。

启动过程完成时，您的客户端会列在menu:系统[系统列表]中。
