[[third-party-channels]]
= 第三方通道
:description: 使用 SUSE Multi-Linux Manager 命令安全地导入和管理第三方 GPG 密钥，以便进行储存库同步
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

{productname} 支持将可选的第三方储存库的内容与其管理的部分产品进行同步。

在 {productname} 数据库中，某些第三方 GPG 密钥是默认包含的，而某些则不是。
对于尚未包含的第三方 GPG 密钥，在选择同步这些第三方储存库时，需要导入相关密钥。

如需导入 GPG 密钥，请在 {productname} 主机服务器上执行以下命令：

----
mgradm gpg add <path-to-gpg-key-file-or-URL>
----

. 示例：从文件添加 GPG 密钥：
+
----
mgradm gpg add repomd.xml.key
----

. 示例：使用 URL 从远程储存库添加 GPG 密钥：
+
----
mgradm gpg add https://<3rd-party-domain>/path/to/repository/repodata/repomd.xml.key
----

如需列出当前储存在 {productname} GPG 数据库中的密钥，请执行以下命令：

----
mgrctl exec -ti -- gpg --homedir /var/lib/spacewalk/gpgdir --list-keys
----

[NOTE]
====
在客户端安装软件包时，您可能还需要在客户端本身信任 GPG 密钥。
这要求 GPG 密钥在该处也可用。

有关更多信息，请参见：xref:client-configuration:gpg-keys.adoc[]。
====
