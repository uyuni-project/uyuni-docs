= 任务日程安排
:description: 学习在 SUSE Multi-Linux Manager 中管理和配置预定义的任务日程安排，以自动化例行维护任务
:revdate: 2026-04-20
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1


所有预定义的任务组列在 menu:管理[任务日程安排] 下。

image::admin_task_schedules.png[scaledwidth=80%]


单击 menu:{productname} 日程安排[日程名称] 以打开 menu:日程名称[基本日程细节]。
您可以禁用它或更改其频率。

单击 btn:[编辑日程安排] 以使用您的设置更新该日程安排。

要禁用日程安排，请单击右上角的 btn:[禁用日程安排]。

[WARNING]
====
仅在您绝对确定必要时才禁用日程安排，因为它们对 {productname} 的正常运行至关重要。
====

当任务被禁用时，它仍然会显示在列表中。
当您单击 menu:{productname} 日程安排[日程名称] 时，您可以通过单击 btn:[激活日程安排] 再次激活该日程安排。


如果您单击某个组名称，将显示该组类型的运行列表和运行状态。

单击开始时间链接会返回 menu:日程名称[基本日程细节]。



== 预定义的任务组


系统默认会安排以下预定义的任务组，您可对其进行配置：

menu:auto-errata-default[]::
根据需要安排自动勘误更新。

menu:channel-repodata-default[]::
（重新）生成储存库元数据文件。

menu:cleanup-data-default[]::
从数据库中清理已过时的软件包更改日志和监控时序数据。

menu:clear-taskologs-default[]::
根据作业类型，从数据库中清除超过指定天数的任务引擎 (taskomatic) 历史数据。

menu:cobbler-sync-default[]::
从 {productname} 同步分发和控制文件数据到 {cobbler}。
有关由 {cobbler} 提供的自动安装的更多信息，请参见 xref:client-configuration:autoinst-intro.adoc[]。

menu:compare-configs-default[]::
将存储在配置通道中的配置文件与所有启用配置的服务器上存储的文件进行比较。
要查看比较结果，请单击 menu:系统[] 选项卡并选择感兴趣的系统。
转到 menu:配置[比较文件]。
有关更多信息，请参见 xref:reference:systems/system-details/sd-configuration.adoc#sd-config-compare-files[]。

menu:cve-server-channels-default[]::
更新用于在 menu:审计[CVE 审计] 页面上显示结果的内部预计算 CVE 数据。
在 menu:审计[CVE 审计] 页面中的搜索结果更新为此日程的最后一次运行。
有关更多信息，请参见 xref:reference:audit/audit-cve-audit.adoc[]。

menu:daily-status-default[]::
向相关地址发送每日报告电子邮件。
有关为特定用户配置通知的更多信息，请参见 xref:reference:users/user-details.adoc[]。

menu:errata-advisory-map-sync-default[]::
更新内部 {suse} 补丁供应商建议数据库表。
如果可用，{suse} 提供的原始建议将在每个补丁详细信息的供应商建议部分中显示。

menu:errata-cache-default[]::
更新内部补丁缓存数据库表，这些表用于查找每个服务器需要更新的软件包。
此外，这会向可能对某些补丁感兴趣的用户发送通知电子邮件。
有关补丁的更多信息，请参见 xref:reference:patches/patches-menu.adoc[]。

menu:errata-queue-default[]::
为配置为接收自动更新（补丁）的服务器将这些更新（补丁）排队。

menu:gatherer-matcher-default[]::
通过运行在虚拟主机管理器中配置的虚拟主机收集器来收集虚拟主机数据。
在更新的数据可用后，将运行 Subscription Matcher 作业。

menu:kickstart-cleanup-default[]::
清理过期的 {kickstart} 会话数据。

menu:kickstartfile-sync-default[]::
生成与配置向导创建的 {cobbler} 控制文件对应的 {kickstart} 文件。

menu:mgr-forward-registration-default[]::
与 {scclongform} 同步客户端注册数据。
默认情况下，新的、已更改的或已删除的客户端数据将被转发。
若要禁用 `/etc/rhn/rhn.conf` 中的同步，请运行：
+
----
server.susemanager.forward_registration = 0
----
+

[WARNING]
====
禁用与 {scc} 的数据同步将导致您所管理的客户端在 RMT、SMT、{productname} 以及 {scc} 直接注册之间的可见性降低。

通过同步数据，您可以确保所有注册客户端具有统一的视图。

https://suselinux.fra1.qualtrics.com/jfe/form/SV_0ooNnrY0rYuQScS[通过分享您选择退出的原因来帮助我们改善服务]
====

menu:mgr-sync-refresh-default[]::
与 {scclongform} (`mgr-sync-refresh`) 同步。
默认情况下，所有自定义频道也作为此任务的一部分进行同步。
有关自定义频道同步的更多信息，请参见 xref:administration:custom-channels.adoc#_custom_channel_synchronization[]。

menu:minion-action-chain-cleanup-default[]::
清理过时的操作链数据。

menu:minion-action-cleanup-default[]::
从文件系统中删除过期的客户端操作数据。
首先，它尝试通过查找存储在 {salt} 作业缓存中的相应结果来完成任何可能未完成的操作。
如果服务器错过了操作的结果，则可能会发生未完成的操作。
对于成功完成的操作，它会删除诸如已执行脚本文件等工件。

menu:minion-checkin-default[]::
在客户端上执行常规的签入。

menu:notifications-cleanup-default[]::
清理失效的通知消息。

menu:oval-data-sync-default[]::
生成所需的 OVAL 数据，以提高 CVE 审计查询的准确度。

menu:package-cleanup-default[]::
从文件系统中删除过时的软件包文件。

menu:reboot-action-cleanup-default[]::
任何待处理超过六小时的重启操作将被标记为失败，并且相关数据将从数据库中清除。
有关调度重启操作的更多信息，请参见 xref:reference:systems/system-details/sd-provisioning.adoc#sd-power-management[]。

menu:session-cleanup-default[]::
清理过时的 Web 界面会话，这通常是用户登录之后，在注销之前关闭浏览器时临时存储的数据。

menu:ssh-service-default[]::
如果客户端配置了 {productname} 联系方法，则提示客户端通过 SSH 与 `SSH Push` 进行检查。
重启后也会恢复操作链。

menu:system-overview-update-queue-default[]::
更新系统概览数据。

menu:system-profile-refresh-default[]::
对所有系统进行硬件刷新。
这仅在每月发生一次，并可能增加 {productname} 服务器的负载。
该作业使用 xref:specialized-guides:salt/salt-rate-limiting.adoc[Salt 限速]。
有关调整批处理大小的信息，请参见 xref:specialized-guides:large-deployments/tuning.adoc#java-salt-batch-size[]。

menu:token-cleanup-default[]::
删除 {salt} 客户端用来下载软件包和元数据的已失效储存库词元。

menu:update-payg-default[]::
从配置的 {payg} 云实例收集身份验证数据。

menu:update-reporting-default[]::
更新本地报告数据库。

menu:update-reporting-hub-default[]::
从外围 {productname} 服务器收集所有报告数据，并更新 Hub 报告数据库。

menu:update-system-overview-default[]::
定期检查系统概览数据是否保持最新。

menu:uuid-cleanup-default[]::
清理过时的 UUID 记录。
