[[tuning-changelogs]]
= 微调更改日志
:description: 在 SUSE Multi-Linux Manager 中配置更改日志元数据，通过设置参数值在 1 到无限之间来限制可下载的数据并节省磁盘空间。
:revdate: 2024-05-10
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

某些软件包有很长的更改日志条目列表。
默认情况下会下载这些数据，但并不总是有用的信息。
为了限制下载的更改日志元数据的数量并节省磁盘空间，您可以限制保留在磁盘上的条目数量。

[IMPORTANT]
====
在服务器容器内运行步骤之前，请使用 `mgrctl term`。
====

此配置选项位于`/etc/rhn/rhn.conf`配置文件中。
该参数的默认值为 `20`。
将此值更改为 `0` 将提供无限数量的条目。

----
java.max_changelog_entries = 20
----

如果您设置此参数，它只在同步新软件包后对其生效。

更改此参数后，请使用 `mgradm restart` 重启服务。

您可能想要删除再重新生成缓存的数据，以去除旧数据。

[IMPORTANT]
====
删除和重新生成缓存数据可能需要很长时间。
根据您拥有的频道数量和要删除的数据量，这可能需要几个小时。
该任务由Taskomatic在后台运行，因此您可以在操作完成时继续使用 {productname}，但您应该预期会有一些性能损失。
====

可以从命令行删除缓存的数据并请求重新生成数据：

----
spacewalk-sql -i
----
然后在 SQL 数据库提示符下输入：
----
DELETE FROM rhnPackageRepodata;
INSERT INTO rhnRepoRegenQueue (id, CHANNEL_LABEL, REASON, FORCE)
(SELECT sequence_nextval('rhn_repo_regen_queue_id_seq'),
        C.label,
        'cached data regeneration',
        'Y'
        FROM rhnChannel C);
\q
----
