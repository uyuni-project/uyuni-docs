= 生成报告
:description: 学习如何使用 SUSE Multi-Linux Manager 中的 Hub Reporting 工具生成自定义报告，然后连接支持您需求的报告工具。
:revdate: 2026-06-24
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

{productname} 允许用户生成各种报告。
这些报告对于盘点您订阅的系统、用户和组织非常有帮助。
使用报告通常比从 {productname} {webui} 手动收集信息更简单，尤其是当您管理许多系统时。

虽然命令行工具 `spacewalk-report` 可用于生成预配置的报告，但随着 xref:administration:reporting-database.adoc[Reporting Database] 的引入，现在也可以生成完全自定义的报告。
这可以通过将任何支持 SQL 语言的报告工具连接到报告数据库并直接提取数据来实现。
有关数据可用性和结构的更多信息，请参见报告数据库架构文档。

== 使用 `spacewalk-report`


[IMPORTANT]
====
在服务器容器内运行步骤之前，请使用 `mgrctl term`。
====

要生成报告，必须安装 `spacewalk-reports` 软件包。
`spacewalk-report` 命令允许您组织和显示有关内容、系统和用户资源的报告，跨越 {productname}。

[IMPORTANT]
====
由于引入了 xref:administration:reporting-database.adoc[Reporting Database]，`spacewalk-report` 现在默认从报告数据库中收集数据。
有关更多信息，请参见<<spacewalk-report-reporting-db>>。
====

可以针对以下各项生成报告：

系统清单::
列出所有已注册到 {productname} 的系统。

补丁:: 
列出所有与已注册系统相关的补丁。
可以按严重性以及适用于特定补丁的系统将补丁排序。

用户:: 
列出所有已注册用户以及与特定用户关联的所有系统。

要获取 CSV 格式的报告，请在服务器上的命令提示符下运行以下命令：

[source]
----
spacewalk-report <report_name>
----

[[spacewalk-report-reporting-db]]
== `spacewalk-report` 和报告数据库

`spacewalk-report` 默认使用新的报告数据库提取数据。
这意味着新生成的报告在数据的结构和格式上会有一些差异。
所有报告的共同差异是：

- 报告数据不会实时更改，而只能通过执行安排的任务来更新；
- 数据重复已被去除，之前被视为“multival”的列现在包含多个用 `;` 分隔的值。这也意味着命令行选项 `--multival-on-rows` 和 `--multival-separator` 不再适用于新报告，因为它们的行为现在是默认的；
- 在所有报告中引入了新列 `mgm_id` 和 `synced_date`，以标识中心方案中的管理服务器，以及上次从应用程序数据库更新信息的时间；
- 现在，所有布尔数据类型值都由 `True`/`False` 表示，而不是由 `1`/`0` 值表示；
- `org_id` 列已被 `organization` 取代，后者包含组织名称而不是数字标识符；
- 术语 "server" 已被 "system" 取代。因此，例如，`server_id` 列现在称为 `system_id`。

有关特定于报告的更改，请参见 <<report-list>>。

[IMPORTANT]
====
如果这种行为更改造成了问题，可以使用新选项 `--legacy-report` 回退到针对应用程序数据库执行的旧报告。
====

有关报告数据库的更多信息，请参阅 xref:administration:reporting-database.adoc[]。
有关在中心方案下对多个服务器数据进行特定聚合的更多信息，请参阅 xref:specialized-guides:large-deployments/hub-reporting.adoc[]。
