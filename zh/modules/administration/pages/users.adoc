{productname} 管理员可以添加新用户、授予权限以及停用或删除用户。
如果您管理大量用户，可以将用户分配到系统组，以便在组级别管理权限。
您还可以更改{webui}的系统默认设置，包括默认语言和默认主题。


[NOTE]
====
`Users` 菜单仅在您以 {productname} 管理员身份登录时可用。
====


要管理 {productname} 用户，请导航到 menu:用户[用户列表 > 所有] 以查看您 {productname} 服务器中的所有用户。
列表中的每个用户显示用户名、真实姓名、分配的角色、用户上次登录的日期以及用户的当前状态。
单击 btn:[创建用户] 以创建新用户帐户。
单击用户名以转到 `User Details` 页面。

要将新用户添加到您的组织，请单击 btn:[创建用户]，填写新用户的详细信息，然后单击 btn:[创建登录名]。


[[users.password.requirments]]
== 口令要求


{productname} 出厂时已预设一系列默认值。

为确保所有新用户密码均符合组织的安全标准，{productname}管理员可以选择强制执行密码创建规则。

在 {webui} 中，导航到 menu:管理员[管理配置 > 密码策略] 以定义密码要求。
使用以下字段的组合：

口令长度下限::
此字段用于定义口令的最小字符数。

口令长度上限::
此字段用于定义口令的最大字符数。

需要包含数字::
此字段用于指定口令是否必须包含数字 (0-9)。

需要包含小写字母::
此字段用于指定口令是否必须包含小写字母 (a-z)。

需要包含大写字母::
此字段用于指定口令是否必须包含大写字母 (A-Z)。

限制连续的字符::
此字段用于指定是否限制使用连续的字符。

需要包含特殊字符::
此字段用于指定口令是否必须包含特殊字符。

允许使用的特殊字符::
仅当选中 `Require Special Characters` 时，此字段才启用。
使用它来指定允许哪些特殊字符，例如 `!@#$%&*` 等。

限制重复的字符::
此字段用于指定禁止的字符重复次数。

重复字符数上限::
此字段用于指定字符的最大重复次数。

单击 btn:[保存] 以保存任何更改的密码设置。

使用 btn:[重置] 将任何设置更改为默认值。


[NOTE]
====
{productname} 出厂时已预设以下默认值：

* 最小密码长度：4
* 最大密码长度：32
* 需要包含大写字母：已选中
====


== 停用和删除帐户

如果不再需要，可以停用或删除用户帐户。
停用的用户帐户可以随时重新激活。
已删除的用户帐户不可见，且无法恢复。

用户可以停用自己的帐户。
但是，如果用户具有管理员角色，则必须先去除该角色，然后才能停用帐户。

停用的用户无法登录到{productname} {webui}或安排任何操作。
用户在停用之前安排的操作仍然保留在操作队列中。
停用的用户可以由{productname}管理员重新激活。


[[administrator-roles]]
== 用户角色

用户可以被分配多个角色，并且任何角色在任何时候可以有多个用户持有。
必须始终至少有一个活跃的{productname}管理员。

要更改用户的角色（{productname}管理员角色除外），请导航到menu:用户[用户列表> 所有]，选择要更改的用户，然后根据需要选中或取消选中管理员角色。

要更改用户的 {productname} 管理员角色，请导航到 menu:管理员[用户]，并根据需要选中或取消选中 `{productname} Admin?`。


[cols="1,1", options="header"]
.用户角色权限
|===
| 职能名称 | 说明
| {productname}管理员 | 可以执行所有功能，包括更改其他用户的权限。
| 组织管理员 | 管理激活密钥、配置、频道和系统组。
| 激活密钥管理员 | 管理激活密钥。
| 映像管理员 | 管理图像控制文件、构建和存储。
| 配置管理员 | 管理系统配置。
| 通道管理员 | 管理软件频道，包括使频道全球可订阅和创建新频道。
| 系统组管理员 | 管理系统组，包括创建和删除系统组、将客户端添加到现有组以及管理用户对组的访问权限。
| 普通用户 | 提供标准级别的访问权限。新创建的用户会自动分配到此角色。
|===



== 创建额外角色

在{productname}中使用基于角色的访问控制，您可以创建额外的角色以微调用户权限。
有关这些角色的更详细信息，请参见xref:administration:role-based-access-control.adoc[]。



== 用户权限和系统

如果您已创建系统组来管理客户端，可以将组指派到用户，让他们进行管理。

要将用户分配到系统组，请导航到 menu:用户[用户列表]，单击用户名进行编辑，然后转到 `System Groups` 选项卡。
检查要分配的组，然后单击btn:[更新默认值]。

您还可以为用户选择一个或多个默认系统组。
当用户注册新客户端时，默认情况下会将其分配到所选的系统组。
这使得用户可以立即访问新注册的客户端。

== 用户和通道权限

可以将用户作为使用通道内容的订阅者，或者作为可以自行管理通道的管理员，指派到组织中的软件通道。

要将用户订阅到频道，请导航到menu:用户[用户列表]，单击用户名进行编辑，然后转到menu:频道权限[订阅]选项卡。
检查要分配的频道，然后点击btn:[更新权限]。

要授予用户频道管理权限，请导航到menu:用户[用户列表]，点击用户名进行编辑，然后转到menu:频道权限[管理]选项卡。
检查要分配的频道，然后点击btn:[更新权限]。

列表中的某些频道可能无法订阅。
这通常是由于用户的管理员状态或频道的全局设置。



== 用户默认语言

创建新用户时，可以选择用于{webui}的语言。
用户创建后，可以通过导航到menu:主页[我的偏好]来更改语言。

默认语言在 `rhn.conf` 配置文件中设置。
要更改默认语言，请打开 `/etc/rhn/rhn.conf` 文件并添加或编辑此行：

----
web.locale = <LANGCODE>
----

如果未设置该参数，则默认语言为 `en_US`。

=== 用户默认界面主题

默认情况下，{productname} {webui}使用与您安装的产品相适应的主题。
您可以更改主题以反映{uyuni}或{susemgr}的颜色。
{susemgr}主题也有一个可用的暗黑选项。

您可以在 `rhn.conf` 配置文件中更改默认主题。
要更改默认主题，请打开 `/etc/rhn/rhn.conf` 文件并添加或编辑此行：

----
web.theme_default = <THEME>
----
