[[security-master-fingerprint]]
= 设置用于客户端到主控端验证的指纹
:description: 配置您的 Salt 客户端，使其通过在 minion 配置文件中输入特定主控端的指纹来进行验证。
:revdate: 2024-12-12
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1


在高度安全的网络配置中，您可能希望确保您的 {salt} 客户端连接到特定的主控端。
要设置从客户端到主控端的验证，请首先在 {salt} minion 配置文件中输入主控端的指纹：

* `/etc/salt/minion.d/custom.conf` 如果您的客户端使用经典 {salt} minion，或者
* `/etc/venv-salt-minion/minion.d/custom.conf` 如果您的客户端使用 {salt} Bundle

然后按以下过程进行操作：

[IMPORTANT]
====
要在服务器容器内访问外壳，请在容器主机上运行 `mgrctl term`。
====

.过程：升级将主控端的指纹添加到客户端
. 在主控端的命令提示符下，以 root 身份使用以下命令查找 `master.pub` 指纹：
+

----
salt-key -F master
----
+

在您的客户端上，打开 `/etc/salt/minion.d/custom.conf` 或 `/etc/venv-salt-minion/minion.d/custom.conf` 配置文件。
添加此行以输入主控端的指纹，替换示例指纹：
+

----
master_finger: 'ba:30:65:2a:d6:9e:20:4f:d8:b2:f3:a7:d4:65:11:13'
----

. 重启该服务。
  对于 salt-minion，请运行：
+

----
systemctl restart salt-minion
----
+ 
. 对于 venv-salt-minion，请运行：
+

----
systemctl restart venv-salt-minion
----

有关 {salt} 捆绑包的详细信息，请参见 xref:client-configuration:contact-methods-saltbundle.adoc[]。

有关从客户端配置安全性的详细信息，请参见 https://docs.saltproject.io/en/latest/ref/configuration/minion.html.
