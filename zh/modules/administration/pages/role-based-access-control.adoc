[[rbac]]
= 基于角色的访问控制 (RBAC)
:description: 通过定义角色和权限，使用SUSE Multi-Linux Manager有效实施基于角色的访问控制（RBAC），以增强安全性并简化管理。
:revdate: 2026-05-13
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

基于角色的访问控制（RBAC）是一种安全方法，根据用户分配的角色限制资源访问，仅允许授权用户访问。在{productname}中，RBAC确保用户只能执行其明确授权的操作并访问资源，从而增强安全性并简化管理。

RBAC 的核心原则包括：

* *最小权限原则:*仅授予用户执行其任务所需的必要访问权限。
* *精细化控制:*针对特定功能提供精细的控制能力。
* *责任分离：*防止单个用户对关键处理拥有过多控制权。
* *可审计性:*能够清晰跟踪用户的操作及权限配置。


[[rbac-key-concepts]]
== RBAC 核心概念


理解以下核心概念对于高效管理 RBAC 至关重要：

* *角色：*一组定义特定能力的权限集合，位于{productname}内。角色分配给用户后，用户将获得该角色对应的所有聚合权限。

+

[IMPORTANT]

====
角色分配给用户后，用户将获得该角色对应的所有聚合权限。
====

+

* *权限:*执行特定操作、访问特定网页或调用{productname}内特定API端点的原子授权。在{productname}中，权限由名称空间及其访问模式表示。
* *用户：*与{productname}交互的单个账户。用户可以被指派一个或多个角色。
* *名称空间:*以树状结构组织的细粒度访问控制单元。大多数名称空间具有独特的“查看”或“修改”模式。


[[user-roles]]
== {productname}中的用户角色

{productname} 提供了预定义角色，并允许定义额外的自定义角色，且可选择从多个其他角色的组合中继承。


=== 预定义角色

请参阅 xref:administration:users.adoc#administrator-roles[] 以获取预定义角色及其描述的完整列表。


=== 定义额外角色

要定义额外角色，您可以执行以下操作：

* 选择多个现有角色，从中继承权限。
* 指定额外的名称空间，以授予访问权限。


[[namespaces-fine-grained-access]]
== 用于精细化访问控制的名称空间

名称空间提供了细粒度的访问控制，组织成树状结构。对于大多数名称空间，名称空间内的访问进一步细分为 "查看" 和 "修改" 模式。

[[managing-rbac]]
== RBAC 管理

可以通过 API 或 {webui} 管理 RBAC 角色和权限。
要通过网页 UI 将角色分配给用户，请参阅 xref:administration:users.adoc[]。


=== 通过 API 管理 RBAC

{productname} API 提供了以编程方式管理角色、权限及用户分配的相关方法。


==== `access` API

以下 API 方法用于管理角色及其关联的访问权限：

* `listNamespaces`：列出 {productname} 中可用的名称空间、访问模式及其说明。
* `listPermissions`：列出角色的允许名称空间。
* `listRoles`：列出 {productname} 中的现有角色。
* `createRole`：创建新角色，可选择从现有角色复制权限。
* `deleteRole`：删除角色。
* `grantAccess`：授予对名称空间的访问权限。
* `revokeAccess`：撤销对名称空间的访问权限。


==== `user` API

以下 API 方法用于管理用户与角色的分配关系：

* `listPermissions`：列出用户的有效权限。
* `listRoles`：列出某个用户分配到的角色。
* `addRole`：为某个用户分配角色。
* `removeRole`：去除某个用户的角色。

有关详细的 API 文档（包括请求和响应格式），请参见 {productname} API 参考手册。


[[rbac-best-practices]]
== RBAC 最佳实践

遵循以下最佳实践有助于维护安全、高效且易于管理的 RBAC 环境：

* *最小权限原则:*始终授予用户执行其职责所需的最低权限。避免过于宽泛的权限。
* *定期审查:*定期审查用户分配到的角色和权限，确保其仍然适用且符合当前的安全策略。
* *记录角色:*清晰记录您创建的每个自定义角色的用途和权限。
* *职责分离:*实施能够强制执行职责分离的角色，以防止单个用户对关键处理拥有过多控制权。
