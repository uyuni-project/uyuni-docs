[[mgr.sync]]
= 使用 `mgr-sync`
:description: 使用 SUSE Multi-Linux Manager 配置通道、组织凭据和产品，并通过 mgr-sync 工具的各种命令和可选参数进行操作。
:revdate: 2024-11-29
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

`mgr-sync` 工具在命令提示符下使用。
它提供了使用 {productname} 的功能，这些功能并不总是在 {webui} 中可用。
`mgr-sync` 的主要用途是连接到 {scclongform}，检索产品和软件包信息，并准备与 {productname} 服务器同步的通道。

该工具旨在与 {suse} 支持订阅一起使用。
对于开源发行版，包括 {opensuse}、{centos} 和 {ubuntu}，不需要此工具。

此表中列出了 `mgr-sync` 的命令和参数。
对 `mgr-sync` 命令使用此语法：

----
mgr-sync [-h] [--version] [-v] [-s] [-d {1,2,3}] {list,add,refresh,delete}
----


[[mgr-sync-commands]]
[cols="1,1,1", options="header"]
.mgr-sync 命令
|===
| 命令 | 说明 | 示例用法
| list | 列出通道、组织凭据或产品 | `mgr-sync list channels`
| 添加 | 添加通道、组织凭据或产品 | `mgr-sync add channel <channel_name>`
| 刷新 | 刷新产品、通道和订阅的本地副本 | `mgr-sync refresh`
| delete | 从本地系统删除现有的 {scc} 组织凭据 | `mgr-sync delete credentials`
| sync | 同步指定的通道，或在留空时请求它| `mgr-sync sync channel <channel_name>`
|===


要查看特定于某个命令的完整选项列表，请使用以下命令：

----
mgr-sync <command> --help
----



[[mgr-sync-optional-args]]
[cols="1,1,1,1", options="header"]
.mgr-sync 可选参数
|===
| 选项 | 缩略选项 | 说明 | 示例用法
| help | `h` | 显示命令用法和选项 | `mgr-sync --help`
| version | 不适用 | 显示当前安装的 `mgr-sync` 版本| `mgr-sync --version`
| 冗长 | `v` | 提供冗长输出 | `mgr-sync --verbose refresh`
| 存储凭证 | `s` | 将凭证存储在本地隐藏文件中 | `mgr-sync --store-credentials`
| debug | `d` | 记录额外的调试信息。需要级别 1、2、3。3 提供最高量的调试信息 | `mgr-sync -d 3 refresh`
| 不同步 | 不适用 | 与 `add` 命令一起使用，以在不开始同步的情况下添加产品或通道 | `mgr-sync --no-sync add <channel_name>`
|===




`mgr-sync` 的日志位于：

* `/var/lib/containers/storage/volumes/var-log/_data/rhn/mgr-sync.log`
* `/var/lib/containers/storage/volumes/var-log/_data/rhn/rhn_web_api.log`
