RPC 连接有时会因网络缓慢或网络链接中断而超时。
这会导致软件包下载或批处理作业挂起或耗时比预期更长。
您可以通过编辑配置文件来调整 RPC 连接允许的最长时间。
虽然这并不能解决网络问题，但它确实会导致进程失败而不是挂起。


.过程：升级解决 RPC 连接超时

. 在 {productname} 服务器上，打开 `/etc/rhn/rhn.conf` 文件并设置最大超时值（以秒为单位）：

+

----
server.timeout =`number`
----

+

. 在 {productname} 代理上，打开 `/etc/uyuni/proxy/config.yaml` 文件并设置最大超时值（以秒为单位）。
代理容器需要重启以使更改生效。

+

----
timeout =`number`
----

+

. 在使用 zypper 的 {sles} 客户端上，打开 `/etc/zypp/zypp.conf` 文件并设置最大超时值（以秒为单位）：

+

----
## Valid values:  [0,3600]
## Default value: 180
download.transfer_timeout = 180
----

+

. 在使用 yum 的 {rhel} 客户端上，打开 `/etc/yum.conf` 文件并设置最大超时值（以秒为单位）：

+

----
timeout =`number`
----

[NOTE]
====
如果您将 RPC 超时限制为小于 `180` 秒，可能会中止完全正常的操作。
====
