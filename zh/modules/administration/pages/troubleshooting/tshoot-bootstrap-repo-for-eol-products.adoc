= 对生命周期已结束产品的引导储存库进行查错
:description: 使用 mgr-create-bootstrap-repo 命令配置用于终止服务的产品的手动引导储存库，以便列出并创建它们。
:revdate: 2024-10-31
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

当受支持的产品同步时，引导储存库会在 {productname} 服务器上自动创建和重新生成。
当产品达到终止服务状态并变为不受支持时，如果您希望继续使用该产品，则必须手动创建引导储存库。

有关引导储存库的更多信息，请参见 xref:client-configuration:bootstrap-repository.adoc[]。



.过程：升级为终止服务的产品创建引导储存库

. 在 {productname} 容器主机的命令提示符处，以 root 身份进入服务器容器：
+

----
mgrctl term
----

. 在容器内部，执行以下步骤：
+

--
.. 使用 `--force` 选项列出可用的不受支持的引导储存库，例如：
+

----
mgr-create-bootstrap-repo --list --force
1. SLE-12-SP2-x86_64
2. SLE-12-SP3-x86_64
----

.. 使用适当的储存库名称作为产品标签，创建引导储存库：
+

----
mgr-create-bootstrap-repo --create SLE-12-SP2-x86_64 --force
----
--

如果您不想手动创建引导储存库，可以检查您需要的产品和引导储存库是否有 LTSS。
