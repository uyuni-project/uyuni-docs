= 代理连接和 FQDN 查错
:description: 请设置通过代理连接的客户端，使其能够正确显示连接状态，方法是在客户端配置中添加额外的完全限定的域名（FQDN）作为 grains。
:revdate: 2022-10-18
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

有时，通过代理连接的客户端会出现在{webui}中，但并未显示它们是通过代理连接的。
如果您未使用完全限定的域名（FQDN）进行连接，且代理对{productname}未知，可能会出现这种情况。

要更正此行为，请在代理上的客户端配置文件中指定其他 FQDN 作为 grain：

----
grains:
  susemanager:
    custom_fqdns:
      - name.one
      - name.two
----
