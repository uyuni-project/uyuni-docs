[[troubleshooting-mass-machineID-duplication]]
= 批量计算机 ID 重复问题
:description: 在 SUSE Multi-Linux Manager 中管理 VM 克隆，使用更改机器 ID 或生成虚假 ID 等技术
:revdate: 2025-10-23
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

如果您使用 {productname} 来管理虚拟机，您可能会发现创建 VM 克隆是有用的。
克隆是一个虚拟机，它使用的主磁盘是现有磁盘的精确副本。

虽然克隆 VM 可以节省大量时间，但磁盘上复制的标识信息有时会导致问题。

如果您尝试注册两台机器，其中一台是另一台的克隆，您可能希望 {productname} 将它们注册为两个独立的客户端。
然而，如果原始客户端和克隆的机器 ID 相同，{productname} 会将两个客户端注册为一个系统，并且在注册第二台机器时数据会被覆盖。

可以通过更改克隆的机器 ID 来解决此问题，以便 {productname} 将它们识别为两个不同的客户端。
有关如何更改机器 ID 的更多信息和说明，请参见 xref:administration:troubleshooting/tshoot-registerclones.adoc[]。

然而，在某些情况下，改变所有系统上的机器 ID 既不可行也不实际，因为这可能会干扰依赖于现有机器 ID 的其他应用程序。在这些情况下，可以生成一个虚假的机器 ID，以便在 {productname} 的上下文中使用。

.过程：升级在注册时为 {productname} 生成机器 ID
[role=procedure]
____

. 创建一个启动脚本（如果您已经有了，请跳过此步骤）。有关更多信息，请参见 xref:client-configuration:registration-bootstrap.adoc[]。

. 打开该文件并更改相应参数：

+

----
GENERATE_OWN_MACHINEID=1
----

. 启用此引导脚本后，计算机 ID 将不再发生冲突。
