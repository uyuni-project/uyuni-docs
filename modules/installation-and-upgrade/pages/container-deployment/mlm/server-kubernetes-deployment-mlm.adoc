=== Credentials

[literal]``server-helm`` installs {productname} server.
The Helm chart does not handle SSL certificate generation.
Instead, users must create {kube} secrets containing the certificates prior to deployment.

The deployment requires four specific {kube} type secrets, each containing a username and a password key.

* [literal]``db-admin-credentials``: PostgreSQL administrator credentials.

* [literal]``db-credentials``: Credentials for the internal database user.

* [literal]``reportdb-credentials``: Credentials for the report database user.

* [literal]``admin-credentials``: Credentials for the server administrator.


The following TLS secrets are expected:

* [literal]``db-cert``: TLS certificate for the report database and needs to have the [literal]``db`` and [literal]``reportdb`` Subject Alternate Names, and the FQDN exposed to the outside world.

* [literal]``uyuni-cert``: TLS certificate for the ingress rule and needs to have the public FQDN as Subject Alternate Name.


=== ConfigMaps

The Root CA certificates for [literal]``db-ca`` and [literal]``uyuni-ca`` are expected in ConfigMaps names [literal]``db-ca`` and [literal]``uyuni-ca`` with the certificate in the [literal]``ca.crt`` key.


=== Storage

The new chart defines volumes as Persistent Volume Claims (PVCs).

[WARNING]
====
* The creation of the underlying PVCs is the responsibility of the cluster administrators.

* The PVCs use the [literal]``ReadWriteOnce`` access mode.
====
