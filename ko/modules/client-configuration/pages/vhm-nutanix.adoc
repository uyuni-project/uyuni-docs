가상 호스트 관리자(VHM)를 설정하여 {productname}과 함께 Nutanix AHV 가상 머신을 사용할 수 있습니다.
시작하려면 {productname} 서버에 VHM을 설정하고 사용 가능한 VM 호스트를 목록에 추가해야 합니다.



== VHM 설정



가상 호스트 관리자(VHM)는 {productname} 서버에서 실행됩니다.

`virtual-host-gatherer-Nutanix` 패키지를 {productname} 서버에 설치했는지 확인하십시오.

VHM을 실행하려면 {productname} 서버가 포트 9440을 열어 Nutanix Prism Element API에 액세스해야 합니다.


.절차 Nutanix VHM 생성

. {productname} {webui}에서 menu:Systems[Virtual Host Managers]로 이동합니다.
. btn:[생성]을 클릭하고 `Nutanix AHV`을(를) 선택합니다.
. `Add a Nutanix AHV Virtual Host Manager` 섹션에서 다음 매개변수를 사용합니다:
* `Label` 필드에 VHM의 사용자 지정 이름을 입력합니다.
* `Hostname` 필드에 전체 도메인 이름(FQDN) 또는 호스트 IP 주소를 입력합니다.
* `Port` 필드에 사용할 Prism Element API 포트를 입력합니다 (예: `9440`).
* `Username` 필드에 VM 호스트와 연결된 사용자 이름을 입력합니다.
* `Password` 필드에 VM 호스트 사용자와 연결된 비밀번호를 입력합니다.
. btn:[Create]를 클릭하여 변경 사항을 저장하고 VHM을 생성합니다.
. `Virtual Host Managers` 페이지에서 새 VHM을 선택합니다.
. `Properties` 페이지에서 btn:[데이터 새로 고침]을 클릭하여 새 VHM을 목록에 추가합니다.

어떤 객체 및 리소스가 목록에 포함되었는지 확인하려면 menu:시스템[시스템 목록> 가상 시스템]으로 이동하십시오.


[NOTE]
====
브라우저에서 HTTPS를 사용하여 Nutanix Prism API 서버에 연결하면 때때로 `invalid certificate` 오류가 발생할 수 있습니다.
이 경우 가상 호스트 관리자의 데이터 새로 고침이 실패합니다.
유효한 SSL 인증서(자체 서명되지 않음)가 Nutanix API 서버에 필요합니다.
Nutanix SSL 인증서에 사용자 지정 CA 기관을 사용하는 경우, 사용자 지정 CA 인증서를 `/etc/pki/trust/anchors`에 복사하여 {productname} 서버에 배치하십시오.

명령줄에서 `update-ca-certificates` 명령을 실행하여 인증서를 다시 신뢰하고 spacewalk 서비스를 다시 시작합니다.
====

VHM이 생성되고 구성된 후, Taskomatic이 자동으로 데이터 수집을 실행합니다.
데이터 수집을 수동으로 수행하려면 menu:시스템[가상 호스트 관리자]로 이동하여 적절한 VHM을 선택하고 btn:[데이터 새로 고침]을 클릭하십시오.

{productname}은 `virtual-host-gatherer`라는 도구와 함께 제공되며, 이 도구는 API를 사용하여 VHM에 연결하고 가상 호스트에 대한 정보를 요청할 수 있습니다. `virtual-host-gatherer`는 각 모듈이 특정 VHM을 활성화하는 선택적 모듈 개념을 유지합니다.
이 도구는 Taskomatic에 의해 매일 밤 자동으로 호출됩니다.
`virtual-host-gatherer` 도구의 로그 파일은 `/var/log/rhn/gatherer.log`에 있습니다.
