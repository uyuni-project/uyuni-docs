= 구성 관리
:description: SUSE Multi-Linux Manager와 YaST 도구를 사용하여 클라이언트의 중앙 분산 구성 파일을 관리합니다.
:revdate: 2026-04-20
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1


각 클라이언트를 수동으로 구성하는 대신에 구성 파일 및 채널을 사용해 클라이언트에 대한 구성을 관리할 수 있습니다.

구성 매개변수는 스크립트화되어 구성 파일에 저장됩니다.
구성 파일을 {productname} {webui}를 사용하여 직접 작성하거나, 다른 위치에 있는 파일을 업로드하거나 링크할 수 있습니다.

구성 채널은 구성 파일을 조직하는 데 사용됩니다.
클라이언트를 구성 채널에 구독시키고 필요에 따라 구성 파일을 배포할 수 있습니다.

구성 파일은 버전 관리되므로 구성 설정을 추가하고, 클라이언트에서 테스트한 후 필요에 따라 이전 버전으로 롤백할 수 있습니다.
구성 채널을 생성한 후에는 다양한 구성 파일 간의 비교와 동일한 구성 파일의 여러 버전 간 비교를 수행할 수 있습니다.

중앙에서 관리되는 구성 파일은 전역 구성 채널에서 제공합니다.

이 표에는 지원되는 기능이 나와 있으며, 여기서:

* {check} 기능은 {suse}에서 지원됩니다.
* {cross} 기능은 {suse}에서 지원되지 않습니다.
* {question} 기능은 현재 검토 중이며, 이후에 지원될 수도 있고 지원되지 않을 수도 있습니다.

== 구성 채널 생성


=== 구성 채널.

새로운 중앙 구성 채널을 생성하려면:

.절차 구성 채널 생성
[role=procedure]
____
. {productname} {webui}에서 menu:구성[채널]로 이동한 후 btn:[구성 채널 생성]을 클릭합니다.

. 채널의 이름을 입력합니다.

. 채널의 레이블을 입력합니다.
    이 필드는 글자, 숫자, 하이픈(`-`) 및 밑줄(`_`)만 포함해야 합니다.

. 다른 채널과 구별할 수 있게 해주는 채널 설명을 입력합니다.

. btn:[구성 채널 생성]을 클릭하여 새 채널을 생성합니다.
____



=== 상태 채널

상태 채널은 클라이언트의 {salt} 상태를 사용합니다:

.절차 {salt} 상태 채널 생성
[role=procedure]
____
. {productname} {webui}에서 menu:구성[채널]로 이동한 후 btn:[상태 채널 생성]을 클릭합니다.

. 채널의 이름을 입력합니다.

. 채널의 레이블을 입력합니다.
    이 필드는 글자, 숫자, 하이픈(`-`) 및 밑줄(`_`)만 포함해야 합니다.

. 다른 채널과 구별할 수 있게 해주는 채널 설명을 입력합니다.

. `SLS Contents` 파일의 `init.sls`을(를) 입력합니다.

. btn:[구성 상태 채널 생성]을 클릭하여 새 채널을 생성합니다.
____


== 구성 파일, 디렉토리 또는 심볼 링크 추가

구성 채널을 생성했으면 이제 구성 파일, 디렉토리 또는 심볼 링크를 추가할 수 있습니다.

.절차 구성 파일, 디렉토리 또는 소프트 링크 추가
[role=procedure]
____
. {productname} {webui}에서 menu:구성[채널]로 이동하여 구성 파일을 추가하려는 구성 채널의 이름을 클릭한 후, menu:파일 추가[파일 생성] 하위 탭으로 이동합니다.

. `File Type` 필드에서 텍스트 파일, 디렉토리 또는 소프트 링크 중 생성하려는 것을 선택합니다.
. `Filename/Path` 필드에 파일을 배포해야 하는 위치의 절대 경로를 입력합니다.

. 소프트 링크를 생성하는 경우 `Symbolic Link Target Filename/Path` 필드에 대상 파일 및 경로를 입력합니다.

. `Ownership` 필드에서 파일의 `User name`과 `Group name`를 입력하고, `File Permissions Mode`를 입력합니다.

. 클라이언트가 SELinux를 활성화한 경우 `SELinux contexts`를 구성하여 필수 파일 속성(예: 사용자, 역할, 파일 형식)을 활성화할 수 있습니다.
. Jinja 템플릿을 그레인 또는 필러 데이터와 함께 사용할 수 있으며, 레거시 매크로 구분자를 대신 사용할 수 있습니다.

. `File Contents` 텍스트 상자에 구성 파일 내용을 입력하고, 스크립트 드롭다운 상자를 사용하여 적절한 스크립팅 언어를 선택합니다.

. btn:[구성 파일 생성]을 클릭합니다.

____


== 클라이언트를 구성 채널에 가입

menu:시스템[시스템 목록]으로 이동하여 구독할 클라이언트를 선택한 후 `Configuration` 탭으로 이동하면 개별 클라이언트를 구성 채널에 구독시킬 수 있습니다.
여러 클라이언트를 구성 채널에 구독시키려면 시스템 세트 관리자(SSM)를 사용할 수 있습니다.


.절차 여러 클라이언트를 구성 채널에 구독시키기
[role=procedure]
____
. {productname} {webui}에서 menu:시스템[시스템 목록]으로 이동하여 작업할 클라이언트를 선택합니다.

. menu:시스템[시스템 세트 관리자]로 이동한 후, menu:구성[채널 구독] 하위 탭으로 이동하여 사용 가능한 구성 채널 목록을 확인합니다.

. 선택 사항: `Systems currently subscribed` 열의 숫자를 클릭하여 현재 어떤 클라이언트가 구성 채널에 구독되어 있는지 확인합니다.

. 구독하려는 구성 채널을 확인하고 btn:[계속]을 클릭합니다.

. 위 및 아래 방향 화살표를 사용해 구성 채널의 순위를 지정합니다.
    설정이 구성 채널 간에 서로 충돌하는 경우 목록 상단에 더 가까이 있는 채널이 우선합니다.

. 선택한 클라이언트에 채널이 적용되는 방식을 결정합니다.
    btn:[최저 우선 순위로 구독]를 클릭하여 새 채널을 현재 구독 중인 채널보다 낮은 우선 순위로 추가합니다.
    btn:[최고 우선 순위로 구독]를 클릭하여 새 채널을 현재 구독 중인 채널보다 높은 우선 순위로 추가합니다.
    btn:[기존 구독 교체]를 클릭하여 기존 구독을 제거하고 새 구독으로 교체합니다.

. btn:[구독 적용]을 클릭합니다.

____

[NOTE]
====
새 구성 채널 우선 순위가 기존 채널과 충돌하는 경우, 중복 채널은 제거된 후 새 우선 순위에 따라 교체됩니다.
클라이언트의 구성 우선 순위가 작업에 의해 재정렬될 경우, {webui}에서 진행하기 전에 변경 사항을 확인해야 합니다.
====


== 구성 파일 비교

또한 시스템 세트 관리자(SSM)를 사용해 클라이언트에 배포된 구성 파일을 {productname} 서버에 저장된 구성 파일과 비교할 수 있습니다.

.절차 구성 파일 비교
[role=procedure]
____
. {productname} {webui}에서 menu:시스템[시스템 목록]으로 이동하여 비교하려는 구성 파일에 구독된 클라이언트를 선택합니다.

. menu:시스템[시스템 세트 관리자]로 이동하고, menu:구성[파일 비교] 하위 탭에서 사용 가능한 구성 파일 목록으로 이동합니다.

. 선택 사항: `Systems` 열의 숫자를 클릭하여 현재 어떤 클라이언트가 구성 파일에 구독되어 있는지 확인합니다.

. 비교할 구성 파일을 확인하고 btn:[파일 비교 예약]을 클릭합니다.
____


== 클라이언트의 Jinja 템플릿

Jinja 템플릿은 {salt} 클라이언트에서 가능합니다.
Jinja는 콘텐츠가 {salt} 엔진에 의해 처리될 때만 평가됩니다.
Jinja는 필러 또는 그레인에서 변수를 제공합니다.
이들은 {salt}에 의해 관리되는 파일(예: `file.managed`를 통해) 또는 {salt} 상태에서 사용할 수 있습니다.

[NOTE]
====
파일 구성 채널은 Jinja 처리가 없는 정적 파일을 전달합니다.
상태 채널 파일은 {salt} 상태에서 참조될 때만 적용됩니다.
====

자세한 내용은 이 예제와 함께 https://docs.saltproject.io/salt/user-guide/en/latest/topics/jinja.html을(를) 참조하십시오:

----
{% if grains.os_family == 'RedHat' %}
  {% set dns_cfg = '/etc/named.conf' %}
{% elif grains.os_family == 'Debian' %}
  {% set dns_cfg = '/etc/bind/named.conf' %}
{% else %}
  {% set dns_cfg = '/etc/named.conf' %}
{% endif %}
dns_conf:
  file.managed:
    - name: {{ dns_cfg }}
    - source: salt://dns/files/named.conf
    - template: jinja
----
