[[autoinst-distributions]]
= 자동 설치 가능한 배포판
:description: 클라이언트 설치를 위해 다양한 방법으로 구성할 수 있는 자동 설치 가능한 배포판을 생성합니다.
:revdate: 2025-02-06
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

자동 설치 프로세스는 설치를 시작하기 위해 여러 파일에 의존합니다.
이 파일에는 Linux 커널, 초기 RAM 디스크 및 설치 모드에서 운영 체제를 부팅하는 데 필요한 기타 파일이 포함됩니다.

{productname}은(는) `mgradm` 도구를 사용하여 설치 파일을 소스에서 서버 컨테이너로 복사합니다.

필요한 파일을 DVD 이미지에서 추출할 수 있습니다.
자세한 내용은 xref:client-configuration:autoinst-distributions.adoc#based-on-iso-image[ISO 이미지 기반 배포]를 참조하십시오.

또는 `tftpboot-installation` 패키지를 설치할 수 있습니다.
자세한 내용은 xref:client-configuration:autoinst-distributions.adoc#based-on-rpm-package[RPM 패키지 기반 배포]를 참조하십시오.

해당 파일과 동일한 운영 체제 버전의 {productname} 서버에서 동기화된 기본 채널이 있어야 합니다.

파일이 준비되고 기본 채널이 동기화되면 배포를 선언해야 합니다.
이 작업은 설치 파일을 기본 채널에 연결합니다.
배포는 하나 이상의 설치 프로필로 참조될 수 있습니다.
자세한 내용은 xref:client-configuration:autoinst-distributions.adoc#declare-distribution[자동 설치 가능한 배포 선언]을 참조하십시오.



[[based-on-iso-image]]
== ISO 이미지 기반 배포


이 방법은 클라이언트에 설치할 운영 체제의 설치 미디어가 있다고 가정합니다.
이는 일반적으로 Linux 커널, `.iso` 파일 및 설치 모드에서 운영 체제를 부팅하는 데 필요한 기타 파일을 포함하는 DVD `initrd` 이미지입니다.



.절차 설치 미디어에서 파일 가져오기
. `mgradm`을(를) 사용하여 ISO 이미지에서 설치 데이터를 가져옵니다:
+
----
# mgradm distribution copy <image_name>.iso <image_name>
----
+
. `mgradm`에서 보고된 배포 경로를 기록해 두십시오.
  배포를 {productname}에 선언할 때 필요합니다.



=== 배포 자동 감지 및 등록

`mgradm`은(는) 배포 이름을 자동으로 감지하고 서버에 등록할 수 있습니다.
제공된 ISO 이미지에는 `.treeinfo` 파일이 포함되어야 합니다.

.절차 자동 감지 및 등록을 사용한 배포 파일 가져오기
. `mgradm`을(를) 사용:
+
----
# mgradm distribution copy --api-user <username> --api-password <password> <image_name>.iso
----


[[based-on-rpm-package]]
== RPM 패키지 기반 배포

이 방법은 {suse} 시스템에 적용됩니다.
설치 시스템에서 미리 패키지화된 파일을 사용하므로 설치 미디어에서 컨텐츠를 가져오는 것보다 단순합니다.



.절차 설치 패키지에서 파일 추출
. {productname} 서버에서 `tftpboot-installation`(으)로 시작하는 패키지를 설치합니다.
  `zypper se tftpboot-installation` 명령을 사용하여 정확한 이름을 확인할 수 있습니다.
. 다음 명령을 사용하여 패키지를 다른 루트에 설치하여 다시 시작할 필요가 없도록 할 수 있습니다.
+
----
# mkdir /opt/tftpinstall
# zypper --installroot /opt/tftpinstall install tftpboot-installation-SLE-Micro-5.5-x86_64
----
. `ls -d /opt/tftpinstall/usr/share/tftpboot-installation/*` 명령으로 설치 파일을 찾으십시오.
. `mgradm`을(를) 사용하여 설치 파일을 복사합니다:
+
----
# mgradm distribution copy /opt/tftpinstall/usr/share/tftpboot-installation/SLE-Micro-5.5-x86_64 SLE-Micro-5.5-x86_64
----
. `mgradm` 도구가 보고한 배포 경로를 기록해 두십시오.
  배포를 {productname}에 선언할 때 필요합니다.
. `mgradm` 도구가 완료되면 `/opt/tftpinstall` 디렉토리를 제거할 수 있습니다.

이 절차는 {productname} 서버를 구동하는 것과 동일한 운영 체제 버전의 설치를 준비합니다.
클라이언트에 다른 운영 체제나 버전을 설치하려면 해당 배포에서 패키지 `tftpboot-installation-*`를 수동으로 가져와야 합니다.
`Package Search` 입력 상자 {productname}에서 `tftpboot-installation`(으)로 시작하는 패키지를 검색한 다음 패키지 세부 정보를 확인하십시오.
이들은 `/var/spacewalk/` 아래의 로컬 경로를 보여줍니다.



[[declare-distribution]]
== 자동 설치 가능한 배포판 선언

자동 설치 파일을 추출한 후의 단계는 자동 설치 가능한 배포판을 선언하는 것입니다.



.절차 자동 설치 가능한 배포판 선언
. {productname} {webui}에서 menu:시스템[자동 설치> 배포판]으로 이동합니다.
. `Create Distribution`을(를) 클릭하고 다음 필드를 작성하십시오:
+
* `Distribution Label` 필드에 자동 설치 가능한 배포를 식별할 수 있는 이름을 입력합니다.
* `Tree Path` 필드에 {productname} 서버에 저장된 설치 미디어의 경로를 입력합니다.
* 일치하는 `Base Channel`을(를) 선택하십시오.
  설치 미디어와 일치해야 합니다.
* `Installer Generation`을(를) 선택하십시오.
  설치 미디어와 일치해야 합니다.
* 선택 사항: 이 배포를 부팅할 때 사용할 커널 옵션을 지정합니다.
  커널 옵션을 제공하는 방법은 여러 가지가 있습니다.
  여기에는 배포에 일반적인 옵션만 추가하십시오.
. btn:[자동 설치 가능한 배포판 생성] 을 클릭합니다.

준비한 설치 파일에는 설치에 필요한 패키지가 포함되어 있지 않을 수 있습니다.
포함되어 있지 않으면 `Kernel Options` 필드에 `useonlinerepo=1`을(를) 추가하십시오.

패키지 저장소에는 서명되지 않은 메타데이터가 포함될 수 있습니다.
메타데이터가 서명되지 않은 경우 `insecure=1` 필드에 `Kernel Options`을(를) 추가하거나 xref:client-configuration:autoinst-owngpgkey.adoc[Use Your Own GPG Key]에 설명된 대로 고유한 GPG 키를 사용하십시오.

이러한 커널 옵션은 예를 들어 전체 DVD가 아닌 "online installer" ISO 이미지를 사용하거나 `tpboot-installation` 패키지를 사용할 때 필요합니다.

menu:시스템[자동 설치> 배포판]으로 이동하여 자동 설치 가능한 배포판을 관리합니다.

ifeval::[{mlm-content} == true]
[NOTE]
====
{productname} 프록시 4.3을 {sle} 클라이언트와 동일한 방식으로 자동 설치할 수 있습니다.
{sle} 설치 미디어를 사용하고 `SLE-Product-SUSE-Manager-Proxy-4.3-Pool for x86_64` 기본 채널을 선택했는지 확인하십시오.

== 배포판 및 프로파일의 커널 옵션 처리

{productname}은 할당한 커널 옵션을 결합할 수 있습니다.
이는 특별한 상속 논리를 사용하여 수행됩니다.
이와 관련된 객체 유형은 세 가지가 있습니다:

. 배포판(또는 줄임말 "Distros")
. 프로파일
. 시스템

최종 커널 옵션에 대한 네 번째 특별한 영향 요소는 Cobbler 설정 파일 `/etc/cobbler/settings.yaml`입니다.
Cobbler 설정 파일은 모든 배포에 대한 기본 커널 옵션을 정의합니다.
이것은 {productname}의 맥락에서 지원되지 않습니다.

커널 옵션을 효과적으로 관리하기 위해서는 원시 값과 확인된 값을 이해하는 것이 중요합니다.

. **원시 값**: 이 값은 특정 Cobbler 항목에 직접 연결되어 Cobbler의 내부 데이터베이스에 있는 그대로 저장된 값을 의미합니다.
. **확인된 값**: 이 값은 Cobbler 항목의 상속 계층 구조를 고려하여 런타임에 동적으로 생성됩니다.

옵션 앞에 `!`를 추가하면 최종 커널 명령줄에서 옵션이 제거됩니다.

{productname}는 프로파일과 시스템 모두에 대한 커널 옵션을 관리합니다.
따라서 배포판에 대한 커널 옵션만 편집할 수 있습니다.

=== 예시

==== 기본 상속 예제

배포판 원시 값

----
install=http://uyuni.server/ks/dist/SLES15SP4 self_update=0
----

프로파일 원시 값

----
console=tty1
----

시스템 원시 값

----
console=ttyS0
----

이 프로파일을 상속하는 시스템의 *확인된* 값

----
install=http://uyuni.server/ks/dist/SLES15SP4 self_update=0 console=ttyS0
----

==== 옵션 제거 예제

배포판 원시 값

----
install=http://uyuni.server/ks/dist/SLES15SP4 self_update=0
----

프로파일 원시 값

----
console=tty1
----

시스템 원시 값

----
!self_update
----

이 프로파일을 상속하는 시스템의 *확인된* 값

----
install=http://uyuni.server/ks/dist/SLES15SP4 console=ttyS0
----
