제품이 동기화될 때, 부팅 리포지토리는 {productname} 서버에서 자동으로 생성되고 재생성됩니다.

== 자동 모드를 위한 옵션

자동 부팅 리포지토리 생성 방식은 변경할 수 있습니다.
이 섹션에서는 다양한 설정에 대해 설명합니다.

플러시 모드::
기본적으로 기존 리포지토리는 최신 패키지만 업데이트됩니다.
대신 항상 빈 리포지토리로 시작하도록 구성할 수 있습니다.
이 동작을 활성화하려면 `/etc/rhn/rhn.conf`에서 이 값을 추가하거나 수정하십시오:
+
----
server.susemanager.bootstrap_repo_flush = 1
----

자동 모드::

기본적으로 부팅 리포지토리의 자동 재생성이 활성화되어 있습니다.
이를 비활성화하려면 `/etc/rhn/rhn.conf`에서 이 값을 추가하거나 수정하십시오:
+
----
server.susemanager.auto_generate_bootstrap_repo = 0
----



=== 부트스트랩 데이터 파일 구성

이 도구는 각 배포에 필요한 패키지에 대한 정보가 포함된 데이터 파일을 사용합니다.
데이터 파일은 `/usr/share/susemanager/mgr_bootstrap_data.py`에 저장됩니다. {suse}은(는) 이 파일을 정기적으로 업데이트합니다.
이 파일을 변경하려면 직접 수정하지 마십시오.
대신, 지속적인 디렉토리에 복사본을 만들고 복사본을 수정하십시오:

----
cp /usr/share/susemanager/mgr_bootstrap_data.py /srv/susemanager/my_data.py
----

변경을 완료한 후 {productname}이 새 파일을 사용하도록 구성하십시오.
`/etc/rhn/rhn.conf`에서 이 값을 추가하거나 수정하십시오:

----
server.susemanager.bootstrap_repo_datamodule = my_data
----

[IMPORTANT]
====
다음 업데이트에서 {suse}의 새 데이터가 원본 데이터 파일을 덮어쓰고 새 파일은 덮어쓰지 않습니다.
새 파일을 {suse}에서 제공하는 변경 사항으로 최신 상태로 유지해야 합니다.
====



== 부트스트랩 리포지토리를 수동으로 생성

기본적으로 부팅 리포지토리는 매일 재생성됩니다.
명령 프롬프트에서 부팅 리포지토리를 수동으로 생성할 수 있습니다.



.절차 {sle}용 부팅 리포지토리 생성
[role=procedure]
_____

. {productname} 서버의 명령 프롬프트에서 루트 사용자로서 부팅 리포지토리를 생성하는 데 사용 가능한 배포를 나열합니다:

+

----
mgr-create-bootstrap-repo -l
----

. 다음과 같이 적절한 리포지토리 이름을 제품 레이블로 사용해 부트스트랩 리포지토리를 생성합니다.

+

----
mgr-create-bootstrap-repo -c SLE-version-x86_64
----

+

. 또는 사용 가능한 배포 목록에서 배포 이름 옆에 표시된 번호를 사용합니다.

_____

클라이언트 리포지토리는 `/srv/www/htdocs/pub/repositories/`에 위치합니다.



여러 제품(SLES 및 SLES for SAP 등)을 미러링했거나 사용자 지정 채널을 사용하는 경우, 부팅 리포지토리를 생성할 때 사용할 상위 채널을 지정해야 할 수 있습니다.
이는 모든 상황에서 필요하지 않습니다.
예를 들어, 일부 SLES{nbsp}15 버전은 공통 코드 기반을 가지고 있으므로 상위 채널을 지정할 필요가 없습니다.
환경에서 필요할 경우에만 이 절차를 사용하십시오.



.선택 사항 절차: 부팅 리포지토리에 상위 채널 지정
[role=procedure]
_____

. 사용 가능한 상위 채널을 확인합니다.

+

----
mgr-create-bootstrap-repo -c SLE-15-x86_64
Multiple options for parent channel found. Please use option
--with-parent-channel <label> and choose one of:
- sle-product-sles15-pool-x86_64
- sle-product-sles_sap15-pool-x86_64
- sle-product-sled15-pool-x86_64
----

. 다음과 같이 적절한 상위 채널을 지정합니다.

+

----
mgr-create-bootstrap-repo -c SLE-15-x86_64 --with-parent-channel sle-product-sled15-pool-x86_64
----

_____


=== 아키텍처가 여러 개인 리포지토리


여러 다른 아키텍처를 포함하는 부팅 리포지토리를 생성하는 경우, 모든 아키텍처가 올바르게 업데이트되도록 주의해야 합니다.
예를 들어, {slea}에 대한 {x86_64} 및 {ibmz} 아키텍처는 `/srv/www/htdocs/pub/repositories/sle/15/2/bootstrap/`에서 동일한 부팅 리포지토리 URL을 사용합니다.

`flush` 옵션이 활성화된 경우, 여러 아키텍처에 대한 부팅 리포지토리를 생성하려고 하면 하나의 아키텍처만 생성됩니다.
이를 피하려면 추가 아키텍처를 생성할 때 명령 프롬프트에서 `--no-flush` 옵션을 사용하십시오.
예:

----
mgr-create-bootstrap-repo -c SLE-15-SP2-x86_64
mgr-create-bootstrap-repo --no-flush -c SLE-15-SP2-s390x
----



== 부트스트랩 및 사용자 지정 채널

사용자 지정 채널을 사용하는 경우 `mgr-create-bootstrap-repo` 명령과 함께 `--with-custom-channels` 옵션을 사용할 수 있습니다.
이 경우, 사용할 상위 채널을 지정해야 합니다.

사용자 지정 채널을 사용하는 경우, 부팅 리포지토리의 자동 생성이 실패할 수 있습니다.
이 경우, 리포지토리를 수동으로 생성해야 합니다.

사용자 지정 채널에 대한 자세한 내용은 xref:administration:custom-channels.adoc[]을 참조하십시오.
