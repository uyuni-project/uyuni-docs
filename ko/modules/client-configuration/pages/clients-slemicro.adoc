= 클라이언트 {sle-micro} 등록
:description: SUSE Multi-Linux Manager 서버에 소프트웨어 채널을 동기화하고 Web UI 또는 명령줄 인터페이스를 통해 SLE Micro 클라이언트를 등록합니다.
:revdate: 2026-06-01
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

이 섹션에는 {x86_64}, {arm64} 및 {zsystems} (s390x) 아키텍처에서 {sle-micro} 운영 체제 5.1, 5.2, 5.3, 5.4 및 5.5를 실행하는 클라이언트를 등록하는 방법에 대한 정보가 포함되어 있습니다.

{sle-micro}은 트랜잭션 업데이트를 사용합니다.
트랜잭션 업데이트는 원자적이며(모든 업데이트가 성공해야만 모든 업데이트가 적용됨) 롤백을 지원합니다.
시스템이 재부팅될 때까지 변경 사항이 활성화되지 않으므로 실행 중인 시스템에 영향을 미치지 않습니다.

트랜잭션 업데이트 및 재부팅에 대한 자세한 내용은 https://documentation.suse.com/sles/15-SP7/html/SLES-all/cha-transactional-updates.html.을 참조하십시오.

* {webui} 또는 명령줄 인터페이스를 사용하여 {productname} 서버에 클라이언트를 등록합니다.

추가 섹션에서는 등록 절차를 확인하고 검증하는 방법을 설명합니다.

== 클라이언트 등록

[WARNING]
====
{sle-micro} 클라이언트는 등록 후 재부팅해야 합니다.

부팅 프로세스 후, {sle-micro} 클라이언트에서는 자동 부팅이 비활성화됩니다. 이 변경은 {salt}가 부팅 {salt} 상태 적용 결과를 전달하기 전에 발생할 수 있는 간헐적인 자동 재부팅으로 인해 구현되었습니다.

등록이 완료된 후 자동으로 재부팅이 예약되지만, 기본 재부팅 관리자 유지 관리 창을 존중합니다.
이 창은 클라이언트가 등록된 후 몇 시간일 수 있습니다.
등록 스크립트가 완료된 후 클라이언트를 수동으로 재부팅하는 것이 좋습니다. 이는 등록 속도를 높이고 시스템 목록에 시스템이 나타나는 것을 확인하기 위함입니다.
====

== 재부팅 {sle-micro}

{sle-micro}은 트랜잭션 시스템입니다.
트랜잭션 업데이트는 일반적으로 여러 재부팅 방법을 지원합니다.
`systemd`에 의해 관리되는 시스템에서 재부팅할 때는 {productname}를 사용하는 것이 권장됩니다.
다른 방법을 사용하면 원치 않는 동작이 발생할 수 있습니다.

{productname}에서 트랜잭션 시스템을 부팅할 때, 시스템이 기본 구성에 있는 경우 `systemd`가 재부팅 방법(`REBOOT_METHOD`)으로 구성됩니다.
이러한 구성은 {productname}가 재부팅 작업을 제어할 수 있게 하며, 재부팅은 즉시 수행되거나 원할 경우 {productname}로 예약할 수 있습니다.


=== 배경 정보

기본적으로 클라이언트 설치 중 재부팅 방법은 `auto`로 설정됩니다.
`auto` 부팅 방법을 사용하면, 서비스가 실행 중인 경우 `rebootmgrd`가 구성된 정책에 따라 시스템을 재부팅하는 데 사용됩니다.
정책은 즉시 재부팅하거나 유지 관리 창 동안 재부팅할 수 있습니다.
자세한 내용은 `rebootmgrd`(8) man 페이지를 참조하십시오.
그렇지 않고 `rebootmgrd`가 실행 중이 아니면, {productname}은 `systemctl reboot`을(를) 호출합니다.

[NOTE]
====
`systemd`와 다른 방법을 사용하면 원치 않는 동작이 발생할 수 있습니다.
====
