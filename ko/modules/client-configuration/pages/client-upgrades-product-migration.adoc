[[client-upgrades-spmigration]]
= 제품 마이그레이션
:description: SLE 기반 클라이언트 시스템을 업그레이드하는 것은 제품 마이그레이션을 사용하여 하나의 서비스 팩 버전에서 이후 서비스 팩 버전으로 마이그레이션하는 것을 포함합니다.
:revdate: 2025-03-10
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

== 소개


제품 마이그레이션을 통해 SLE 기반 클라이언트 시스템을 서비스 팩(SP) 수준에서 이후 수준으로 업그레이드할 수 있습니다.
예를 들어, {sles}{nbsp}15{nbsp}SP5를 {sles}{nbsp}15{nbsp}SP6으로 마이그레이션할 수 있습니다.
또한 {rhel} 또는 {centos}과 같은 클라이언트를 {sll}로 업그레이드할 수 있습니다.

[IMPORTANT]
====
제품 마이그레이션을 위해서는 클라이언트에서 활성화되고 마이그레이션되어야 하는 모든 리포지토리가 유효해야 합니다. 그렇지 않으면 마이그레이션이 중단됩니다.
====

{productname}은(는) 다음 마이그레이션 경로를 지원합니다:

=== 동일한 주 버전 내에서 마이그레이션

제품 마이그레이션은 동일한 주요 버전 내에서 업그레이드를 위한 것입니다.
제품 마이그레이션을 사용하여 {sles}{nbsp}12에서 {sles}{nbsp}15로 마이그레이션할 수 없습니다.
주요 업그레이드에 대한 자세한 내용은 xref:client-configuration:client-upgrades-major.adoc[]를 참조하십시오.


=== {leap}의 이후 부 버전 또는 해당 {sles} SP로의 마이그레이션

또한 {leap}을 이후 부 버전으로 또는 해당 {sles} SP 수준으로 마이그레이션할 수도 있습니다. 예:

* {leap} 15.5에서 15.6으로
* {leap} 15.6에서 {sles} 15 SP6으로


=== {sles}을(를) 해당 {slesforsap} SP로 마이그레이션

또한 {sles}을(를) 해당 {slesforsap} SP 수준으로 마이그레이션할 수 있습니다. 예:

* {sles} 15 SP5에서 {slesforsap} 15 SP5로


{sles}{nbsp}12 및 이후 버전에서 {suse}은(는) {scclongform}가 제공하는 경우 서비스 팩 건너뛰기를 지원합니다.
예를 들어, {sles}{nbsp}15에서 SP2로 업그레이드할 수 있으며, SP1을 설치할 필요가 없습니다.

지원되는 {sles} 업그레이드 경로에 대한 내용은 link:https://documentation.suse.com/en-us/sles/15-SP6/html/SLES-all/cha-upgrade-paths.html#sec-upgrade-paths-supported[]를 참조하십시오.


[NOTE]
====
* 모든 추천 확장 프로그램은 마이그레이션할 때 자동으로 활성화됩니다.

* 마이그레이션 과정에서 {productname}는 설치 전에 필요한 모든 사용 조건을 자동으로 수락합니다.
====



== 단일 시스템 마이그레이션

제품 마이그레이션을 시작하기 전에:

* 대기 중인 업데이트나 패치가 없는지 확인하십시오.
  클라이언트 시스템의 `System Status`menu:세부정보[개요] 페이지에서 을(를) 확인하고, 제공된 모든 업데이트나 패치를 설치하십시오.
  클라이언트 시스템이 최신 상태가 아니면 제품 마이그레이션이 실패할 수 있습니다.
* 대상 제품의 모든 채널이 완전히 동기화되어 있는지 확인하십시오.
  {webui}에서 동기화 상태를 확인하려면 menu:관리자[설정 마법사> 제품] 페이지로 이동하십시오.
* 비상 사태에 대비하여 작동하는 시스템 백업이 있는지 확인하십시오.
  제품 마이그레이션에는 롤백 기능이 없습니다.
  마이그레이션 절차가 시작되면 롤백이 불가능합니다.


.절차 단일 시스템 마이그레이션 수행
. menu:시스템[개요] 페이지에서 클라이언트를 선택하십시오.
. 클라이언트의 시스템 세부정보 페이지에서 menu:소프트웨어[제품 마이그레이션] 탭으로 이동하십시오.
. 대상 마이그레이션 경로를 선택하고 btn:[채널 선택]을 클릭하십시오.
. `Product Migration - Channels` 페이지에서 `Mandatory Child Channels` 및 추가 `Optional Child Channels`을(를) 포함하여 올바른 기본 채널을 선택하십시오.
. 선택 사항: 공급업체가 변경된 패키지를 설치할 수 있도록 `Allow Vendor Change`을(를) 선택하십시오.
  이 경우 마이그레이션이 시작되기 전에 세부정보와 함께 알림이 표시됩니다.
+
[IMPORTANT]
====
openSUSE Leap을(를) {sles}(으)로 마이그레이션하려면 `Allow Vendor Change` 옵션을 선택해야 합니다.
====
. 채널이 적절히 구성되었으면 btn:[마이그레이션 예약]을 클릭하십시오.



== 제품 대량 마이그레이션

다수의 클라이언트를 다음 SP 버전으로 마이그레이션하고 싶은 경우 {productname} API 호출을 사용하면 됩니다.

`spacecmd` 명령행 도구는 `system_scheduleproductmigration` 하위 명령을 제공하며, 이는 다수의 클라이언트에 대한 마이그레이션을 다음 마이너 버전으로 예약하는 데 사용할 수 있습니다.



=== 제품 대량 마이그레이션 수행

[WARNING]
====
제품 대량 마이그레이션 작업은 위험하므로 프로세스를 철저히 테스트해야 합니다.
최소한 시험 실행을 우선 수행하십시오.

실수로 시스템을 업그레이드하지 않도록 주의하십시오.
====

.절차 제품 대량 마이그레이션 수행
. 다음과 같이 사용 가능한 마이그레이션 대상을 나열하고, 마이그레이션하려는 시스템 ID를 적어 둡니다.
+
----
spacecmd api -- system.listMigrationTargets -A 1000010001
----
. 각 시스템 ID에 대해 `listMigrationTarget`을(를) 호출하고 원하는 대상 제품이 사용 가능한지 확인하십시오.
+
* 시스템 ID에 사용 가능한 대상이 있는 경우 `system.scheduleProductMigration`을(를) 호출하십시오.
* 원하는 대상을 사용할 수 없는 경우 시스템을 건너뛰십시오.
+
. 다음 템플릿을 환경에 맞게 조정합니다.
+
----
target = '[....]'
basechannel = 'channel-label'
system_ids = [1, 2, 3]

session = auth.login(user, pass)
for system in system_ids
  if system.listMigrationTargets(session, system).ident == target
    system.scheduleProductMigration(session, system, target, basechannel, [], False, <now>)
  else
    print "Cannot migrate to requested target -- skipping system"
  endif
endfor
----


=== 예: SLES 15 SP2에서 SLES 15 SP3로

이 예에서는 대량 마이그레이션을 편리하게 수행할 수 있도록 그룹이 임시로 생성됩니다.

.절차 대량 제품 마이그레이션 그룹 생성
. {productname} {webui}에서 menu:시스템[시스템 그룹]으로 이동한 후 btn:[그룹 생성]을 클릭합니다.
. 그룹 이름을 `mpm-target-sles15sp3`(으)로 지정하십시오.
+
* 동일한 기본 채널에 가입된 시스템만 생성된 그룹에 추가해야 합니다.
  예제에서는 `SLE-Product-SLES15-SP2-Pool for x86_64`에 구독된 시스템만 그룹에 추가해야 합니다.
+
클라이언트를 그룹에 추가하는 방법에 대한 자세한 내용은 xref:client-configuration:system-groups.adoc[]을 참조하십시오.

////
Note or warning about taking normal precautions (backups, make sure fully patched, etc)
////

.절차 시스템을 그룹에 추가
. 그룹 내의 모든 시스템에 대한 대상을 가져오려면 다음을 실행합니다.
+
----
spacecmd -- system_listmigrationtargets group:mpm-target-sles15sp3
----
+
. 이 명령을 수행하면 "ID" 스트링이 출력됩니다.
  * *all* 시스템에 대해 보고되는 대상만 선택하십시오.
  * 문자열은 다른 명령의 `MIGRATIONTARGET`에 대한 식별자입니다.
+
[IMPORTANT]
====
`spacecmd` 하위 명령인 `system_scheduleproductmigration` 및 `system_listmigrationtargets`은(는) 그룹에 속한 모든 시스템을 반복합니다.

그룹에 시스템 100개가 있는 경우 작업 100개가 예약으로 표시됩니다.

그룹의 모든 시스템은 동일한 마이그레이션 대상을 지원해야 합니다.
====



.절차 대량 마이그레이션 명령 실행

. `system_scheduleproductmigration` 명령의 구문은 다음과 같습니다:
+
----
spacecmd -- system_scheduleproductmigration <SYSTEM> <BASE_CHANNEL_LABEL> \
    <MIGRATION_TARGET> [options]
----
. 이 예에서 `mpm-target-sles15sp3` 그룹의 모든 시스템을 SLES 12 SP2에서 SLES 15 SP로 업그레이드하려면 명령줄에 다음을 입력하십시오:
+
----
spacecmd -- system_scheduleproductmigration group:mpm-target-sles15sp3 \
    sle-product-sles15-sp3-pool-x86_64 "[190,203,195,1242]" -d
----


==== 필수 구문 설명

`system_scheduleproductmigration`의 구문 사용법 및 옵션을 보려면 다음을 실행하십시오:

----
spacecmd system_scheduleproductmigration help
----

<SYSTEM>::
이 예에서는 해당 그룹에서 모든 시스템을 선택하기 위해 생성한 그룹을 사용합니다.
+
----
group:mpm-target-sles15sp3
----

<BASE_CHANNEL_LABEL>::
이것은 대상 기본 채널의 레이블입니다.
이 경우 시스템은 SLES 15 SP3으로 업그레이드되며 레이블은 `sle-product-sles15-sp3-pool-x86_64`입니다.
+
현재 미러링되는 모든 기본 채널 목록을 보려면 다음을 실행:
+
----
spacecmd softwarechannel_listbasechannels
----
+
현재 기본 채널에 대해 사용할 수 있는 대상이 아닌 경우 채널로 업그레이드할 수 없음에 유의하십시오.

<MIGRATION_TARGET>::
`group:mpm-target-sles15sp3` 그룹에 있는 시스템의 이 값을 식별하려면 다음을 실행하십시오:
+
----
spacecmd -- system_listmigrationtargets group:mpm-target-sles15sp3
----
+
`MIGRATION_TARGET` 매개변수는 다음 형식으로 전달되어야 하며, 괄호로 인한 부작용을 방지하려면 셸 인용이 필요함을 유의하십시오.
+
----
"[190,203,195,1242]"
----

옵션::

* `-s` START_TIME
* 시험 실행을 원하신다면, `-d`에 이 플래그를 전달하십시오(실제 마이그레이션 전에 시험 실행을 권장합니다).
* `-c` CHILD_CHANNELS (공백 없이 쉼표로 구분된 하위 채널 레이블)
+
이 경우 성공적인 시험 실행 후 제거할 수 있는 `-d` 옵션이 포함되었습니다.

성공하면 예약된 각 시스템의 명령 출력은 다음과 같습니다.
----
Scheduling Product migration for system mpm-sles152-1
Scheduled action ID: 66
----

주어진 그룹의 시스템에서 이 경우 시험 실행을 포함한 작업을 {webui}에서 추적할 수 있습니다.
클라이언트의 시스템 세부정보 페이지에서 menu:이벤트[이력]로 이동하십시오.
시험 실행 중 오류가 발생하면 시스템을 조사해야 합니다.

모두 정상이면 명령에서 `-d` 옵션을 제거하여 실제 마이그레이션을 실행할 수 있습니다.
마이그레이션이 완료되면 {productname} {webui}에서 시스템을 재부팅할 수 있습니다.
