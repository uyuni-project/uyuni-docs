= 무인 프로비저닝
:description: SUSE 시스템에서 system.createSystemRecord API 호출을 사용하여 자동 설치를 구성하는 방법을 배웁니다.
:revdate: 2024-03-14
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀

MAC 주소로 식별된 클라이언트와 자동 설치 프로파일 간의 연관성을 선언하기 위해 API 호출을 사용할 수 있습니다.
다음 번 시스템이 재부팅되면 지정된 프로파일을 기반으로 설치가 시작됩니다.


.절차 수동으로 선언된 프로파일에서 다시 설치

. {productname} 서버의 명령 프롬프트에서 `system.createSystemRecord` API 호출을 사용하십시오.
  이 예제에서는 `name`을(를) 클라이언트의 이름으로, `<profile>`을(를) 프로파일 레이블로, `<iface>`을(를) 클라이언트의 인터페이스 이름(예: `eth0`)으로, `<hw_addr>`을(를) 하드웨어 주소(예: `00:25:22:71:e7:c6`)로 교체하십시오:
+
----
$ spacecmd api -- --args '["<name>", "<profile>", "", "", \
  [ {"name": "<iface>", "mac": "<hw_addr>"} ]]' \
  system.createSystemRecord
----
. 클라이언트의 전원을 켭니다.
  네트워크에서 부팅되며 설치에 맞는 올바른 프로파일이 선택됩니다.

이 명령은 {cobbler}에 시스템 레코드를 생성합니다.
커널 옵션, 클라이언트의 IP 주소 및 도메인 이름과 같은 추가 매개변수를 지정할 수도 있습니다.
자세한 내용은 `createSystemRecord call`에 대한 API 설명서를 참조하십시오.
