[[client-upgrades]]
= 클라이언트 업그레이드
:description: SUSE Multi-Linux Manager 웹 UI의 YaST 도구 또는 콘텐츠 라이프사이클 관리자를 사용하여 클라이언트 업그레이드를 자동화하고 수행하십시오.
:revdate: 2024-11-29
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

클라이언트는 자체 기본 운영 체제의 버전 관리 시스템을 사용하며 정기적인 업그레이드가 필요합니다.

{scc}개의 등록된 클라이언트가 {suse} 운영 체제를 사용하는 경우, {productname} {webui} 내에서 업그레이드를 수행할 수 있습니다.
지원되는 {sle}{nbsp}15 업그레이드 경로에 대한 내용은 {sles-base-os-documentation}/cha-upgrade-paths.html을 참조하십시오.

SLE{nbsp}12에서 SLE{nbsp}15로 클라이언트를 업그레이드하려면 업그레이드가 자동화되지만, 시작하기 전에 몇 가지 준비 단계를 수행해야 합니다.
자세한 내용은 xref:client-configuration:client-upgrades-major.adoc[]을 참조하십시오.

콘텐츠 라이프사이클 관리자를 사용하여 클라이언트 업그레이드를 자동화할 수도 있습니다.
자세한 내용은 xref:client-configuration:client-upgrades-lifecycle.adoc[]을 참조하십시오.

서비스 팩 업그레이드, openSUSE Leap 마이너 버전 업그레이드 또는 openSUSE Leap에서 {sle}으로의 마이그레이션과 같은 제품 마이그레이션에 대한 자세한 내용은 xref:client-configuration:client-upgrades-product-migration.adoc[]을 참조하십시오.

ifeval::[{uyuni-content} == true]

등록되지 않은 openSUSE Leap 클라이언트 업그레이드에 대한 자세한 내용은 xref:client-configuration:client-upgrades-uyuni.adoc[]을 참조하십시오.
endif::[]
