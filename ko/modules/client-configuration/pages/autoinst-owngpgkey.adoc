= 자체 GPG 키 사용
:description: 자체 GPG 키를 구성하여 패키지 메타데이터를 안전하게 서명하고, 이를 설치 미디어의 초기 RAM 디스크에 추가하여 더 안전한
:revdate: 2024-07-10
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

자동 설치에 사용하는 리포지토리에 서명되지 않은 메타데이터가 있는 경우, 일반적으로 `insecure=1` 커널 파라미터를 자동 설치 가능한 배포판 옵션으로 사용하고 `spacewalk/sles_no_signature_checks` 코드 조각을 {ay} 설치 파일에 사용해야 합니다.

더 안전한 방법은 자체 GPG 키를 제공하는 것입니다.

[NOTE]
====
이 기술은 {suse} 클라이언트에만 해당합니다.
====

.절차 자체 GPG 키 포함
. GPG 키를 생성합니다.
. 이 키를 사용하여 패키지의 메타데이터에 서명합니다.
. 설치 미디어의 초기 RAM 디스크에 추가합니다.
. GPG 키를 컨테이너에 복사하려면 `mgradm gpg add`를 사용하십시오.
  이 명령은 GPG 키를 복사하여 고객 키링에 추가합니다.
. `mgradm restart` 명령을 실행하십시오.
  이 명령은 SUSE와 고객 키링을 사용하여 고유한 키링을 생성합니다.

[NOTE]
====
메타데이터를 새 GPG 키로 서명하면 이미 온보딩된 클라이언트는 새 키에 대해 알지 못합니다.
이상적으로는 클라이언트를 등록하기 전에 메타데이터에 서명해야 합니다.

그러한 리포지토리를 사용하는 클라이언트를 이미 온보딩한 경우 해결 방법은 해당 클라이언트에서 GPG 키 확인을 비활성화하는 것입니다.
====
