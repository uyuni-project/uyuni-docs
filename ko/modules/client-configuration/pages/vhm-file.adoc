= 다른 타사 공급자를 통한 가상화
:description: SUSE Multi-Linux Manager에서 VMware와 같은 타사 가상화 공급자를 포함하도록 가져오기 설정을 구성합니다.
:revdate: 2024-09-03
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

Xen, KVM 또는 VMware가 아닌 타사 가상화 공급자를 사용하려면 JSON 구성 파일을 {productname}로 가져올 수 있습니다.

이와 마찬가지로 API에 대한 직접 액세스를 제공하지 않는 VMware가 설치된 경우 파일 기반 VHM은 몇 가지 기본 관리 기능을 제공합니다.

[NOTE]
====
이 옵션은 `virtual-host-gatherer` 도구로 생성된 파일을 가져오는 데 사용됩니다.
수동으로 생성된 파일을 위한 것이 아닙니다.
====

// Add instructions for custom JSON file, including example if possible. LKB 2019-10-23

.절차 JSON 파일 내보내기 및 가져오기

. VM 네트워크에서 을(를) 실행하여`virtual-host-gatherer` JSON 구성 파일을 내보냅니다.
. 생성한 파일을 {productname} 서버가 액세스할 수 있는 위치에 저장합니다.
. {productname} {webui}에서 menu:Systems[Virtual Host Managers]로 이동합니다.
. btn:[Create]를 클릭하고 `File-based`을(를) 선택합니다.
. `Add a file-based Virtual Host Manager` 섹션에서 다음 매개변수를 사용합니다.
* `Label` 필드에 VHM의 사용자 지정 이름을 입력합니다.
* `Url` 필드에 내보낸 JSON 구성 파일의 경로를 입력합니다.
. btn:[Create]를 클릭하여 변경 사항을 저장하고 VHM을 생성합니다.
. `Virtual Host Managers` 페이지에서 새 VHM을 선택합니다.
. `Properties` 페이지에서 btn:[데이터 새로 고침]을 클릭하여 새 VHM을 목록에 추가합니다.

자세한 내용은 {productname} 서버의 man 페이지에서 `virtual-host-gatherer`를 참조하십시오.

----
man virtual-host-gatherer
----


이 패키지의 `README` 파일은 하이퍼바이저의 `type` 등에 대한 배경 정보를 제공합니다:

----
/usr/share/doc/packages/virtual-host-gatherer/README.md
----

man 페이지 및 `README` 파일에는 예시 구성 파일도 포함되어 있습니다.
