= {centos} 클라이언트 등록
:description: SUSE Multi-Linux Manager에서 필요한 소프트웨어 리포지토리를 추가하여 CentOS 클라이언트 아키텍처(x86-64 및 AArch64 포함)를 구성합니다.
:revdate: 2025-03-10
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀

이 섹션에는 {centos} 운영 체제를 실행하는 클라이언트 등록에 대한 정보가 포함되어 있습니다.

ifeval::[{mlm-content} == true]

{centos} 클라이언트는 {centos}를 기반으로 하며 {sleses}, RES, {redhat} 또는 {es}와는 관련이 없습니다.
endif::[]

{centos} 기본 미디어 리포지토리 및 {centos} 설치 미디어에 대한 액세스를 준비하고 {productname} 서버를 {centos} 콘텐츠 전송 네트워크에 연결할 책임은 사용자에게 있습니다.
====

[NOTE]
====
{centos} 클라이언트를 {productname}에 등록하는 것은 `enforcing`의 기본 SELinux 구성과 `targeted` 정책으로 테스트되었습니다.
{productname}에 {centos} 클라이언트를 등록하기 위해 SELinux를 비활성화할 필요는 없습니다.
====



== 소프트웨어 채널 추가


귀하의 {productname} 서버에 {centos} 클라이언트를 등록하기 전에 필요한 소프트웨어 채널을 추가하고 이 채널을 동기화해야 합니다.

Python 3.6을 제공하지 않으면 `spacecmd` 패키지 설치가 실패합니다.

[NOTE]
====
업스트림과 {productname} 채널 간의 AppStream 채널에서 사용 가능한 패키지 수에 차이가 있을 수 있습니다.
같은 채널을 다른 시점에 추가한 경우 비교할 때 다른 숫자가 표시될 수 있습니다.
이는 {centos}가 리포지토리를 관리하는 방식 때문입니다. {centos}는 새 버전이 출시될 때 이전 버전의 패키지를 제거하는 반면, {productname}는 나이에 관계없이 모든 패키지를 유지합니다.
====

== 활성화 키 생성

{centos} 채널에 연결된 활성화 키를 생성해야 합니다.

활성화 키에 대한 자세한 내용은 xref:client-configuration:activation-keys.adoc[]을 참조하십시오.



== GPG 키 관리

클라이언트는 소프트웨어 패키지가 설치되기 전에 GPG 키를 사용하여 패키지의 진위를 확인합니다.
신뢰할 수 있는 소프트웨어만 클라이언트에 설치할 수 있습니다.

[IMPORTANT]
====
GPG 키를 신뢰하는 것은 클라이언트의 보안을 위해 중요합니다.
어떤 키가 필요하고 신뢰할 수 있는지를 결정하는 것은 관리자의 임무입니다.
GPG 키가 신뢰되지 않을 때 소프트웨어 채널을 사용할 수 없기 때문에, 클라이언트에 채널을 할당하는 결정은 키를 신뢰하는 결정에 달려 있습니다.
====

GPG 키에 대한 자세한 정보는 xref:client-configuration:gpg-keys.adoc[]을(를) 참조하십시오.



== 클라이언트 등록


[NOTE]
====
CentOS 7은 `/etc/yum.repos.d/CentOS-Base.repo`에 URL이 활성화되어 있어 `yum`이(가) fastmirror를 사용하여 리포지토리를 찾으려고 시도하기 때문에 온보딩에 실패할 것입니다.
이로 인해 부팅이 다음과 같은 오류 메시지와 함께 실패하게 됩니다:

----
Could not resolve host: mirrorlist.centos.org; Unknown error
----

이 문제를 방지하려면 다음 명령을 사용하여 `CentOS-Base.repo` 파일을 제거하면 됩니다:

[source,shell]
----
rm -f /etc/yum.repos.d/CentOS-Base.repo
----

또는 파일을 유지해야 하는 경우에는 각 URL 섹션에 `enabled=0`을(를) 추가하면 됩니다.

그런 다음, 어떤 경우든 다시 부트스트랩합니다.

== 정오표 관리

[NOTE]
====
이 섹션은 컨테이너화되지 않은 Uyuni에만 적용되며, {centos} 7의 서비스 종료일 이후에 제거됩니다.
====

{centos} 클라이언트를 업데이트할 때 패키지에는 업데이트에 대한 메타데이터가 포함되지 않습니다.
이 정보를 얻기 위해 서드파티 정오표 서비스를 사용할 수 있습니다.

[WARNING]
====

// ifeval::[{mlm-content} == true]
//
// The third-party errata service described here, CEFS, is provided and maintained by the community.
// It is not supported by {suse}.
//
// endif::[]

CEFS의 저자들은 유용할 것이라는 희망으로 최선을 다해 패치나 정오표를 제공하지만, 정확성이나 최신성에 대한 보장은 없습니다.
이는 패치 날짜가 부정확할 수 있음을 의미하며, 최소한 한 경우에는 발표된 데이터가 한 달 이상 오래된 것으로 나타났습니다.
이러한 사례에 대한 자세한 정보는 https://github.com/stevemeier/cefs/issues/28#issuecomment-656579382 및 https://github.com/stevemeier/cefs/issues/28#issuecomment-656573607.를 참조하십시오.

패치 데이터와 관련된 문제나 지연은 귀하의 {productname} 서버에 신뢰할 수 없는 패치 정보가 수입될 수 있습니다.
이로 인해 보고서, 감사, CVE 업데이트 또는 기타 패치 관련 정보도 부정확해질 수 있습니다.
보안 관련 요구 사항 및 인증 기준에 따라 이 서비스를 사용하는 대신 패치 데이터를 독립적으로 검증하거나 다른 운영 체제를 선택하는 등의 대안을 고려해 주십시오.
====




.절차 정오표 서비스 설치

. {productname} 서버의 명령 프롬프트에서 루트 사용자로서 `sle-module-development-tools` 모듈을 추가하십시오:
+
----
SUSEConnect --product sle-module-development-tools/15.2/x86_64
----
. 다음과 같이 오류 서비스 종속성을 설치합니다.
+
----
zypper in  perl-Text-Unidecode
----
. `/etc/rhn/rhn.conf`에서 이 줄을 추가하거나 편집하십시오:
+
----
java.allow_adding_patches_via_api = centos7-x86_64-updates,centos7-x86_64,centos7-x86_64-extras
----
. Tomcat을 다시 시작합니다.
+
----
systemctl restart tomcat
----
. 다음과 같이 오류 스크립트에 대해 파일을 생성합니다.
+
----
touch /usr/local/bin/cent-errata.sh
----
. 이 스크립트를 포함할 새 파일을 편집하고, 필요에 따라 리포지토리 상세 정보를 편집합니다.
    이 스크립트는 외부 오류 서비스에서 오류 상세 정보를 가져와 압축을 풀고 세부 정보를 게시합니다.
+
----
#!/bin/bash
mkdir -p /usr/local/centos
cd /usr/local/centos
rm *.xml
wget -c http://cefs.steve-meier.de/errata.latest.xml
wget -c https://www.redhat.com/security/data/oval/v2/RHEL7/rhel-7.oval.xml.bz2
bzip2 -d rhel-7.oval.xml.bz2
wget -c http://cefs.steve-meier.de/errata-import.tar
tar xvf errata-import.tar
chmod +x /usr/local/centos/errata-import.pl
export SPACEWALK_USER='<adminname>';export SPACEWALK_PASS='<password>'
/usr/local/centos/errata-import.pl --server '<servername>' \
--errata /usr/local/centos/errata.latest.xml  \
--include-channels=centos7-x86_64-updates,centos7-x86_64,centos7-x86_64-extras \
--publish --rhsa-oval /usr/local/centos/rhel-7.oval.xml
----
. 다음과 같이 cron 작업을 설정하여 매일 스크립트를 실행합니다.
+
----
ln -s /usr/local/bin/cent-errata.sh /etc/cron.daily
----

이 도구에 대한 자세한 정보는 https://cefs.steve-meier.de/.을 참조하십시오.

endif::[]
