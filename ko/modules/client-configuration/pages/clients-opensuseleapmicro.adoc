= {leapmicro} 클라이언트 등록
:description: openSUSE Leap Micro를 실행하는 클라이언트를 등록하고 신뢰할 수 있는 에지 컴퓨팅 환경을 위한 채널 관리를 구성하는 방법을 알아보세요.
:revdate: 2026-06-01
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

이 섹션에는 {leapmicro} 운영 체제를 실행하는 클라이언트를 {x86_64} 및 {aarch64} 아키텍처에서 등록하는 방법에 대한 정보가 포함되어 있습니다.

{leapmicro}은 에지 컴퓨팅을 위해 특별히 제작된 초신뢰성 경량 운영 체제입니다.
SUSE Linux Enterprise의 기업용 보안 및 규정 준수 구성 요소를 활용하고 이를 현대적이고 불변의 개발자 친화적인 OS 플랫폼과 결합합니다.

{leapmicro}은 트랜잭션 업데이트를 사용합니다.
트랜잭션 업데이트는 원자적이며(모든 업데이트가 성공해야만 모든 업데이트가 적용됨) 롤백을 지원합니다.
시스템이 재부팅될 때까지 변경 사항이 활성화되지 않기 때문에 실행 중인 시스템에 영향을 미치지 않습니다.
이 정보는 menu:시스템[세부정보> 개요] 하위 탭에 표시됩니다.

트랜잭션 업데이트 및 재부팅에 대한 자세한 내용은 https://documentation.suse.com/sles/15-SP7/html/SLES-all/cha-transactional-updates.html.을 참조하세요.

== 클라이언트 등록

[WARNING]
====
{leapmicro} 클라이언트는 등록 후 재부팅해야 합니다.

부팅 프로세스가 완료된 후 {leapmicro}에서 자동 부팅이 비활성화됩니다. 이 변경은 부팅 솔트 상태를 적용한 결과를 Salt가 전달하기 전에 발생할 수 있는 간헐적인 자동 재부팅으로 인해 구현되었습니다.

등록이 완료된 후 재부팅이 자동으로 예약되지만, 기본 재부팅 관리자 유지 관리 창을 존중합니다.
이 창은 클라이언트가 등록된 후 몇 시간일 수 있습니다.
{leapmicro}등록을 가속화하려면 등록 스크립트가 완료된 후 클라이언트를 수동으로 재부팅하십시오.
====
