[[autoinst-reinstall]]
= 등록된 시스템 재설치
:description: 인플레이스 재설치를 위해 등록된 시스템을 자동 설치 가능한 배포판과 프로파일로 구성합니다.
:revdate: 2024-02-19
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

인플레이스 재설치는 로컬 클라이언트 시스템에서 시작됩니다.
따라서 클라이언트가 네트워크에서 PXE 부팅을 할 필요는 없습니다.

등록된 클라이언트를 인플레이스 재설치하려면 자동 설치 가능한 배포판과 자동 설치 프로파일을 정의해야 합니다.
자세한 내용은 xref:client-configuration:autoinst-distributions.adoc[자동 설치 가능한 배포판] 및 xref:client-configuration:autoinst-profiles.adoc[자동 설치 프로파일]을 참조하십시오.

자동 설치 프로파일 및 배포판을 정의한 후 다시 설치를 실행할 수 있습니다.



.절차 이미 등록된 클라이언트 재설치
. {productname} {webui}에서 menu:시스템[시스템 목록]으로 이동하여 재설치할 클라이언트를 선택하고 menu:프로비저닝[자동 설치> 스케줄] 하위 탭으로 이동합니다.
. 준비한 자동 설치 프로파일을 선택하고 필요한 경우 프록시를 선택한 후 btn:[자동 설치 예약 및 완료]를 클릭합니다.
. 설치 진행 상황은 menu:프로비저닝[자동 설치> 세션 상태]로 이동하거나 클라이언트에서 직접 모니터링할 수 있습니다.
  클라이언트는 재부팅한 후, 부팅 메뉴에서 `reinstall-system`라는 새로운 선택 항목을 선택합니다.

그러면 HTTP 프로토콜을 통해 설치가 진행됩니다.
