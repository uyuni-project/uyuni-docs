= 소스 패키지 미러링
:description: SUSE Multi-Linux Manager 서버에서 소스 패키지를 미러링하도록 구성하려면 rhn.conf 구성 파일에서 이 기능을 전역적으로 활성화한 후, 서버를 재시작하십시오.
:revdate: 2024-05-10
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

로컬에서 자체 패키지를 빌드하거나 법적인 이유로 패키지에 대한 소스 코드가 필요한 경우 {productname} 서버에서 소스 패키지를 미러링할 수 있습니다.

[NOTE]
====
미러링 소스 패키지는 상당한 양의 디스크 공간을 사용할 수 있습니다.
====

[IMPORTANT]
====
서버 컨테이너 내에서 단계를 실행하기 전에 `mgrctl term`을(를) 사용하십시오.
====

.절차 소스 패키지 미러링
. `/etc/rhn/rhn.conf` 구성 파일을 열고 다음 줄을 추가하십시오:
+

----
server.sync_source_packages = 1
----
. Spacewalk 서비스를 다시 시작하여 변경 사항을 적용합니다.
+

----
mgradm restart
----

현재 이 기능은 모든 리포지토리에 대해 전역적으로만 활성화할 수 있습니다.
개별 리포지토리를 선택하여 미러링하는 것은 불가능합니다.

이 기능이 활성화되면, 소스 패키지는 다음 리포지토리 동기화 후 {productname} {webui}에서 사용할 수 있게 됩니다.
이들은 바이너리 패키지의 소스로 표시되며, {webui}에서 직접 다운로드할 수 있습니다.
소스 패키지는 {webui}를 사용하는 클라이언트에 설치할 수 없습니다.
