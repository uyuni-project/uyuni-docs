[[third-party-channels]]
= 타사 채널
:description: SUSE Multi-Linux Manager 명령을 사용하여 리포지토리 동기화를 위해 제3자 GPG 키를 안전하게 가져오고 관리합니다.
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

{productname}에는 선택 사항 제3자 리포지토리의 콘텐츠를 해당 제품이 관리할 수 있는 일부 제품과 동기화하는 기능이 포함되어 있습니다.

일부 제3자 GPG 키는 기본적으로 {productname} 데이터베이스에 포함되어 있으며, 일부는 포함되어 있지 않습니다. 
포함되지 않은 제3자 키의 경우, 해당 제3자 리포지토리를 동기화할 때 관련 키를 가져와야 합니다.

GPG 키를 가져오려면 다음 명령 구문을 사용하고, {productname} 호스트 서버에서 명령을 실행합니다:

----
mgradm gpg add <path-to-gpg-key-file-or-URL>
----

. 예: 파일에서 GPG 키 추가: 
+
----
mgradm gpg add repomd.xml.key
----

. 예: URL을 사용하여 원격 리포지토리에서 GPG 키 추가:
+
----
mgradm gpg add https://<3rd-party-domain>/path/to/repository/repodata/repomd.xml.key
----

{productname} GPG 데이터베이스에 현재 저장된 키를 나열하려면 다음 명령을 실행합니다:

----
mgrctl exec -ti -- gpg --homedir /var/lib/spacewalk/gpgdir --list-keys
----

[NOTE]
====
클라이언트에서 패키지를 설치할 때 클라이언트 자체에서 GPG 키를 신뢰해야 할 수도 있습니다.
이것은 GPG 키가 그곳에서도 사용 가능해야 함을 요구합니다.

자세한 내용은 다음을 참조하십시오: xref:client-configuration:gpg-keys.adoc[].
====
