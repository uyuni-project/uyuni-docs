RPC 연결은 느린 네트워크나 네트워크 링크가 끊어져서 시간 초과가 발생할 수 있습니다.
이로 인해 패키지 다운로드나 배치 작업이 멈추거나 예상보다 더 오랜 시간이 걸릴 수 있습니다.
구성 파일을 편집하여 RPC 연결이 소요될 수 있는 최대 시간을 조정할 수 있습니다.
이것이 네트워크 문제를 해결하지는 않지만, 프로세스가 멈추는 대신 실패하게 만듭니다.


.절차 RPC 연결 시간 제한 해결

. {productname} 서버에서 `/etc/rhn/rhn.conf` 파일을 열고 최대 시간 제한 값(초)을 설정합니다:

+

----
server.timeout =`number`
----

+

. {productname} 프록시에서 `/etc/uyuni/proxy/config.yaml` 파일을 열고 최대 시간 제한 값(초)을 설정합니다.
변경 사항이 적용되려면 프록시 컨테이너를 재시작해야 합니다:

+

----
timeout =`number`
----

+

. zypper를 사용하는 하나의 {sles} 클라이언트에서 `/etc/zypp/zypp.conf` 파일을 열고 최대 시간 제한 값(초)을 설정합니다:

+

----
## Valid values:  [0,3600]
## Default value: 180
download.transfer_timeout = 180
----

+

. yum을 사용하는 하나의 {rhel} 클라이언트에서 `/etc/yum.conf` 파일을 열고 최대 시간 제한 값(초)을 설정합니다:

+

----
timeout =`number`
----

[NOTE]
====
RPC 시간 제한을 `180` 초 미만으로 제한하면, 완전히 정상적인 작업이 중단될 위험이 있습니다.
====
