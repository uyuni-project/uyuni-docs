[[troubleshooting-mounting-tmp-with-noexec]]
= noexec를 사용한 /tmp 마운트 문제 해결
:description: /tmp를 noexec로 마운트하는 문제를 해결하는 방법은 TMPDIR 환경 변수를 설정하거나 systemd 서비스 파일을 수정하는 것입니다.
:revdate: 2022-10-18
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

Salt는 클라이언트의 파일 시스템에서 `/tmp`의 원격 명령을 실행합니다.
따라서 `/tmp`를 `noexec` 옵션으로 마운트해서는 안 됩니다.
이 문제를 해결하는 또 다른 방법은 Salt 서비스에 대해 지정된 `TMPDIR` 환경 변수를 사용하여 임시 디렉토리 경로를 재정의하고, `noexec` 옵션이 설정되지 않은 디렉토리를 가리키도록 하는 것입니다.
Salt Bundle을 사용하는 경우 systemd 드롭인 구성 파일 `/etc/systemd/system/venv-salt-minion.service.d/10-TMPDIR.conf`을 사용하거나, 클라이언트에서 `/etc/systemd/system/salt-minion.service.d/10-TMPDIR.conf`을 사용하는 경우에는 `salt-minion`을 사용하는 것이 좋습니다.
드롭인 구성 파일 내용의 예:
----
[Service]
Environment=TMPDIR=/var/tmp
----
