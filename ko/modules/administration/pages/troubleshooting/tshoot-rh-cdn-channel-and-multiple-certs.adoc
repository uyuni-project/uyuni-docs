= 문제 해결 Red Hat CDN 채널 및 다중 인증서
:description: Red Hat CDN 채널을 구성하고 유효한 인증서를 단일 PEM 파일로 병합하여 SUSE Multi-Linux Manager와 함께 사용합니다.
:revdate: 2022-10-18
:page-revdate: {revdate}
:page-author: SUSE 제품 및 솔루션 문서 팀
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

{redhat} 콘텐츠 전송 네트워크(CDN) 채널은 때때로 여러 인증서를 제공하지만, {productname} {webui}는 단일 인증서만 가져올 수 있습니다.
CDN이 {productname} {webui}가 알고 있는 인증서와 다른 인증서를 제시하면, 검증이 실패하고 리포지토리에 접근할 수 있는 권한이 거부됩니다. 인증서는 정확하더라도 마찬가지입니다.
수신된 오류 메시지는 다음과 같습니다.

----
[error]
Repository '<repo_name>' is invalid.
<repo.pem> Valid metadata not found at specified URL
History:
 - [|] Error trying to read from '<repo.pem>'
 - Permission to access '<repo.pem>' denied.
Please check if the URIs defined for this repository are pointing to a valid repository.
Skipping repository '<repo_nam' because of the above error.
Could not refresh the repositories because of errors.
HH:MM:SS RepoMDError: Cannot access repository. Maybe repository GPG keys are not imported
----

이 문제를 해결하려면 유효한 모든 인증서를 단일 `.pem` 파일로 병합한 후 {productname}에서 사용하도록 인증서를 다시 빌드하십시오.



.절차 여러 {redhat} CDN 인증서 해결
. {redhat} 클라이언트의 명령 프롬프트에서 루트 사용자로 `/etc/pki/entitlement/`의 모든 현재 인증서를 단일 `rh-cert.pem` 파일로 수집하십시오.
+
----
cat 866705146090697087.pem 3539668047766796506.pem redhat-entitlement-authority.pem > rh-cert.pem
----
. `/etc/pki/entitlement/`의 모든 현재 키를 단일 `rh-key.pem` 파일로 수집하십시오.
+
----
cat 866705146090697087-key.pem 3539668047766796506-key.pem > rh-key.pem
----

이제 {productname} 서버에 새로운 인증서를 가져올 수 있으며, xref:client-configuration:clients-rh-cdn.adoc[]의 지침을 사용합니다.
