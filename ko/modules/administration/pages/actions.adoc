= 작업
:description: SUSE Multi-Linux Manager를 사용하여 클라이언트에서 정기적 작업, 원격 명령 및 자동화된 작업을 구성하여 하이 스테이트 자동화 및 사용자 정의 상태를 실행합니다.

클라이언트 작업은 다음과 같은 다양한 방법으로 관리할 수 있습니다.

* 자동화된 반복 작업을 예약하여 지정된 일정에 따라 Highstate 상태 또는 임의의 사용자 정의 상태 집합을 클라이언트에 적용할 수 있습니다.
* 반복 작업은 개별 클라이언트, 시스템 그룹 내 모든 클라이언트 또는 전체 조직에도 적용 가능합니다.
* 작업 체인을 생성하여 특정 순서로 수행하도록 작업을 설정할 수 있습니다.
  ** 작업 체인은 미리 생성 및 편집할 수 있으며 적절한 시간에 실행되도록 일정을 예약할 수 있습니다.
* 하나 이상의 클라이언트에서 원격 명령을 수행할 수도 있습니다.
  ** 원격 명령을 사용하면 개별 클라이언트 또는 검색어와 일치하는 모든 클라이언트에 명령을 실행할 수 있습니다.


[[recurring_actions]]
== 반복 작업


정기적 작업은 개별 클라이언트, 시스템 그룹 또는 조직 내 모든 클라이언트에도 적용 가능합니다.

현재 {productname}에서 정기적 작업으로 지원하는 작업 유형은 다음과 같습니다:

- `Highstate:` 하이스테이트를 실행합니다.
- `Custom state:` 사용자 정의 상태 집합을 실행합니다. 사용자 정의 상태는 {productname}에서 제공하는 내부 상태이거나 사용자가 생성한 구성 채널일 수 있습니다.

구성 채널에 대한 자세한 내용은 xref:client-configuration:configuration-management.adoc[]을 참조하십시오.



.절차 새 정기적 작업 생성하기
. 개별 클라이언트에 정기적 작업을 적용하려면 `Systems`으로 이동하여 일정을 구성할 클라이언트를 클릭하고 `Recurring Actions` 탭으로 이동합니다.
. 시스템 그룹에 정기적 작업을 적용하려면 menu:시스템[시스템 그룹]으로 이동하여 일정을 구성할 그룹을 선택한 후 `Recurring Actions` 탭으로 이동합니다.
. btn:[생성]을 클릭합니다.
. `Action Type` 드롭다운에서 작업 유형을 선택합니다.
. 새 일정의 이름을 입력합니다.
. 정기적 작업 빈도를 선택합니다.
+
* `Hourly:` 매시간 분을 입력합니다.
    예를 들어, `15`는 매시 15분에 작업을 실행합니다.
* `Daily:` 매일 실행할 시간을 선택합니다.
    예를 들어, `01:00`는 매일 01:00에 작업을 실행하며, 이는 {productname} 서버의 시간대 기준으로 실행됩니다.
* `Weekly:` 매주 지정된 시간에 작업을 실행할 요일과 시간을 선택합니다.
* `Monthly:` 매월 지정된 시간에 작업을 실행할 날짜와 시간을 선택합니다.
* `Custom Quartz format:` 더 자세한 옵션을 보려면 사용자 정의 quartz 문자열을 입력합니다.
    예를 들어, 매월 토요일 0215에 정기적 작업을 실행하려면 다음을 입력합니다.
+
----
0 15 2 ? * 7
----
. 선택 사항: `Test mode` 스위치를 켜서 테스트 모드에서 일정을 실행합니다.
. `Custom state` 유형의 작업에 대해 사용 가능한 상태 목록에서 상태를 선택하고 btn:[변경 사항 저장]을 클릭합니다. 이 작업은 현재 상태 선택만 저장하고 일정을 저장하지 않습니다.
. 다음 패널에서 선택한 상태를 드래그 앤 드롭하여 실행 순서에 넣고 btn:[확인]을 클릭합니다.
. btn:[일정 생성]을 클릭하여 저장하고 기존 일정의 전체 목록을 확인합니다.


조직 관리자들은 조직의 모든 클라이언트에 대해 정기적 작업을 설정하고 편집할 수 있습니다.
menu:홈[내 조직> 정기적 작업]으로 이동하여 전체 조직에 적용되는 모든 정기적 작업을 확인합니다.

{productname} 관리자는 모든 조직의 모든 클라이언트에 대해 정기적 작업을 설정하고 편집할 수 있습니다.
menu:관리자[조직]로 이동하여 관리할 조직을 선택하고 menu:상태[정기적 작업] 탭으로 이동합니다.



== 동작 체인

클라이언트에서 순차 작업을 여러 개 수행해야 하는 경우 작업 체인을 생성하면 순서대로 작업할 수 있습니다.

기본적으로 대부분의 클라이언트는 명령이 발행되자마자 작업을 실행합니다.
일부 경우 작업이 오랜 시간이 걸릴 수 있으며, 이는 이후에 발행된 작업이 실패할 수 있음을 의미할 수 있습니다.
예를 들어, 클라이언트에게 재부팅을 지시한 후 두 번째 명령을 발행하면, 재부팅이 아직 진행 중이기 때문에 두 번째 작업이 실패할 수 있습니다.
작업이 올바른 순서로 발생하도록 하려면 작업 체인을 사용하십시오.

[IMPORTANT]
====
트랜잭션 업데이트 시스템의 경우 재부팅 작업이 발생할 때까지 단일 스냅샷 내에서 작업 체인이 실행됩니다.
이로 인해 일부 제한이 발생할 수 있습니다.

ifeval::[{uyuni-content} == true]
자세한 내용은 xref:client-configuration:clients-slemicro.adoc[] 및 xref:client-configuration:clients-opensuseleapmicro.adoc[]을 참조하십시오.
endif::[]

ifeval::[{mlm-content} == true]
자세한 내용은 xref:client-configuration:clients-slemicro.adoc[]을 참조하십시오.
endif::[]
====

모든 클라이언트에서 작업 체인을 사용할 수 있습니다.
작업 체인에는 이러한 작업을 아무 순서로든 포함할 수 있습니다:

* menu:시스템 세부정보[원격 명령]
* menu:시스템 세부정보[시스템 재부팅 예약]
* menu:시스템 세부정보[상태> 하이 스테이트]
* menu:시스템 세부정보[소프트웨어> 패키지> 목록/제거]
* menu:시스템 세부정보[소프트웨어> 패키지> 설치]
* menu:시스템 세부정보[소프트웨어> 패키지> 업그레이드]
* menu:시스템 세부정보[소프트웨어> 패치]
* menu:시스템 세부정보[소프트웨어> 소프트웨어 채널]
* menu:시스템 세부정보[구성]
* menu:이미지[빌드]



[NOTE]
====
작업 체인은 사용자별입니다.
작업 체인을 {webui}에서 보려면 작업 체인을 생성한 동일한 사용자로 로그인해야 합니다.
====



.절차 새 작업 체인 생성
. {productname} {webui}에서 작업 체인의 첫 번째 작업으로 이동합니다.
  예를 들어, 클라이언트의 `System Details`으로 이동한 후 btn:[시스템 재부팅 예약]을 클릭합니다.
. `Add to` 필드를 확인하고 추가할 작업 체인을 선택합니다:
+
* 첫 번째 작업 체인인 경우 `new action chain`을 선택합니다.
* 작업 체인이 이미 있는 경우 목록에서 선택합니다.
* 기존 작업 체인이 이미 있지만 새 작업 체인을 생성하려면 새 작업 체인의 이름을 입력하여 생성합니다.
. 작업을 확인합니다.
    작업은 즉시 수행하지는 않으며 새 작업 체인을 생성하고 이를 확인하는 파란색 막대가 화면 상단에 표시됩니다.
. `Add to` 필드를 확인하고 추가할 작업 체인의 이름을 선택하여 작업 체인에 작업 추가를 계속 진행합니다.
. 작업 추가가 완료되면 menu:일정[작업 체인]으로 이동하여 목록에서 작업 체인을 선택합니다.
. 작업을 올바른 위치에 끌어다 놓아 작업을 다시 정렬합니다.
    파란색 더하기 기호를 클릭하여 작업이 수행될 클라이언트를 확인합니다.
    변경 사항을 저장하려면 btn:[저장]을 클릭합니다.
. 작업 체인이 실행될 시간을 예약하고 btn:[저장 및 예약]을 클릭합니다.
    btn:[저장] 또는 btn:[저장 및 예약]을 클릭하지 않고 페이지를 나가면 저장되지 않은 모든 변경 사항이 삭제됩니다.

[NOTE]
====
작업 체인의 한 작업이 실패하면 작업 체인이 중지되고 더 이상 작업이 실행되지 않습니다.
====

menu:일정[대기 중 작업]으로 이동하여 작업 체인에서 예약된 작업을 볼 수 있습니다.



== 원격 명령

이 절차를 사용하여 {salt}을 통해 원격 명령을 실행합니다.

시작하기 전에 클라이언트가 설치된 운영 체제에 적합한 도구 자식 채널에 구독되어 있는지 확인하십시오.
소프트웨어 채널에 구독하는 방법에 대한 자세한 내용은 xref:client-configuration:channels.adoc[]을 참조하십시오.

[IMPORTANT]
====
//* For transactional update systems, consider that a remote command is run inside a single snapshot.
//  This can cause some limitations.

ifeval::[{uyuni-content} == true]
* 트랜잭션 업데이트 시스템의 경우 단일 스냅샷 내에서 원격 명령이 실행된다는 점에 유의해야 합니다.
  이로 인해 일부 제한이 발생할 수 있습니다.
  자세한 내용은 xref:client-configuration:clients-slemicro.adoc[] 및 xref:client-configuration:clients-opensuseleapmicro.adoc[]을 참조하십시오.
endif::[]

ifeval::[{mlm-content} == true]
* 트랜잭션 업데이트 시스템의 경우 단일 스냅샷 내에서 원격 명령이 실행된다는 점에 유의해야 합니다.
  이로 인해 일부 제한이 발생할 수 있습니다.
  자세한 내용은 xref:client-configuration:clients-slemicro.adoc[]을 참조하십시오.
endif::[]

* 원격 명령은 클라이언트의 `/tmp/` 디렉토리에서 실행됩니다.
  원격 명령이 정확하게 작동하도록 하려면 `noexec` 옵션으로 `/tmp`을(를) 마운트하지 마십시오. 
  자세한 내용은 xref:administration:troubleshooting/tshoot-mounting-tmp-with-noexec.adoc[Troubleshooting Mounting /tmp with noexec]을 참조하십시오.

* `Remote Commands` 페이지에서 실행되는 모든 명령은 클라이언트에서 {rootuser}(으)로 실행됩니다.
  와일드카드를 사용하여 여러 시스템에서 명령을 실행할 수 있습니다.
  명령을 발행하기 전에 항상 신중하게 확인하십시오.
====


.절차 Salt 클라이언트에서 원격 명령 실행
. menu:Salt[원격 명령]으로 이동하십시오.
. 첫 번째 필드에서 `@` 기호 앞에 실행할 명령을 입력합니다.
. 두 번째 필드에서 `@` 기호 뒤에 명령을 실행할 클라이언트를 입력합니다.
    개별 클라이언트의 `minion-id`를 입력하거나 와일드카드를 사용하여 클라이언트 범위를 지정할 수 있습니다.
. btn:[대상 찾기]를 클릭하여 대상으로 지정한 클라이언트를 확인하고 올바른 세부 사항을 사용했는지 확인합니다.
. btn:[명령 실행]을 클릭하여 대상 클라이언트에 명령을 내립니다.
