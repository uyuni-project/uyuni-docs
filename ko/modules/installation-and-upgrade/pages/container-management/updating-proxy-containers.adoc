업그레이드 명령을 실행하기 전에 호스트 운영 체제를 업데이트해야 합니다.
호스트 운영 체제를 업데이트하면 {productname} 도구와 같은 `mgrpxy` 도구도 업데이트됩니다.

.절차 프록시 업그레이드
[role=procedure]
____
. `zypper`을(를) 사용하여 소프트웨어 리포지토리를 새로 고치세요: 

+

[source,shell]
----
zypper ref
----

ifeval::[{mlm-content} == true]

. 호스트 운영 체제에 따라 다음 단계를 진행합니다.

+

{sl-micro}와 같은 트랜잭션 시스템의 경우: ::

+

--

. `transactional-update`을(를) 사용하여 사용 가능한 업데이트를 적용하세요: 

+

[source,shell]
----
transactional-update
----

. 업데이트가 적용되었다면 `reboot`을(를) 실행하세요.

--

+

endif::[]

ifeval::[{uyuni-content} == true]

. `transactional-update`을(를) 사용하여 사용 가능한 업데이트를 적용하세요: 

+

[source,shell]
----
transactional-update
----

. 업데이트가 적용되었다면 `reboot`을(를) 실행하세요.

endif::[]


ifeval::[{mlm-content} == true]

{sles}의 경우: ::

+

--

`zypper`을(를) 사용하여 설치된 소프트웨어를 업데이트하세요: 

[source,shell]
----
zypper up
----

--

endif::[]

+

. {productname} 프록시 컨테이너가 `podman`에서 실행 중인 경우, 다음 명령을 사용하여 업데이트할 수 있습니다:

+

[source,shell]
----
mgrpxy upgrade podman
----

+

또는 Kubernetes 클러스터에서 실행 중인 경우 다음을 사용하여 업데이트할 수 있습니다.

+

[source,shell]
----
mgrpxy upgrade kubernetes
----

+

. podman에서, 사용하지 않는 컨테이너 이미지를 정리하여 디스크 공간을 확보합니다.

+

[source,shell]
----
podman image prune -a
----
+

{kube}에서는 이미지 정리가 자동으로 처리되며, {kube} 배포판에 따라 달라질 수 있습니다.
+

[NOTE]
====
특정 버전으로 업그레이드할 때 태그 매개변수를 지정하지 않으면 가장 최근 버전으로 업그레이드됩니다.
특정 버전으로 업그레이드하려면 원하는 이미지 태그와 함께 태그 매개변수를 제공하십시오.
====

[IMPORTANT]
====

ifeval::[{mlm-content} == true]

특정 태그를 사용하여 특정 컨테이너를 업그레이드하는 옵션이 있지만, 이 기능은 PTF를 적용하는 경우에만 사용할 수 있습니다.  일반적인 상황에서는 일관성을 유지하기 위해 모든 프록시 컨테이너에 동일한 태그를 사용하는 것이 좋습니다.
endif::[]
