{productname} 프록시는 `/var/cache/` 디렉토리에 패키지를 캐시합니다.
`/var/cache/`에 사용 가능한 공간이 충분하지 않으면, 프록시는 오래된 사용하지 않는 패키지를 제거하고 새로운 패키지로 교체합니다.

이 동작의 결과:

* 프록시의 `/var/cache/` 디렉토리가 클수록, 프록시와 {productname} 서버 간의 트래픽은 줄어듭니다.

* 프록시의 `/var/cache/` 디렉토리 크기를 `/var/spacewalk/` 서버의 {productname}와 동일하게 설정하면, 최초 동기화 후 많은 트래픽을 피할 수 있습니다.

* `/var/cache/` 서버에 있는 {productname} 디렉토리는 프록시보다 작을 수 있습니다.
    크기 추정에 대한 가이드는 <<server-hardware-requirements>> 섹션을 참조하십시오.
