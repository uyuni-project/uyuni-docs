== air-gapped 배포란?


에어갭 배포란 물리적으로 불안전한 네트워크, 특히 인터넷과 격리된 네트워크 시스템의 설정 및 운영을 의미합니다.
이러한 유형의 배포는 군사 시설, 금융 시스템, 중요 인프라, 그리고 민감한 데이터가 처리되어 외부 위협으로부터 보호되어야 하는 모든 곳과 같은 고보안 환경에서 일반적으로 사용됩니다.

`Podman` 또는 `Docker`를 사용하여 인터넷에 연결된 컴퓨터에서 컨테이너 이미지를 쉽게 가져올 수 있습니다.

.절차 이미지 가져오기
. 원하는 이미지를 가져온 후, 이미지를 `tar` 아카이브로 저장합니다.
  예:

+
.Podman
----
podman pull registry.opensuse.org/uyuni/server:latest
podman save --output server.tar registry.opensuse.org/uyuni/server:latest
----
+
.Docker
----
docker pull registry.opensuse.org/uyuni/server:latest
docker save --output images.tar registry.opensuse.org/uyuni/server:latest
----
+
. 생성된 `server-image.tar`를 서버 컨테이너 호스트로 전송하고 다음 명령을 사용하여 로드합니다:
+
.서버 이미지 로드
----
podman load -i images.tar
----

== 에어갭 환경에서 {salt} 공식의 컨테이너 이미지 가져오기

Bind 및 DHCP (Kea)와 같은 일부 공식도 컨테이너를 사용합니다.
에어갭 환경에서 사용하려는 경우 해당 이미지를 가져오고, 아카이브에 저장한 후 {productname} 프록시 또는 다른 관리 시스템에 로드해야 합니다.

이미지는 `registry.opensuse.org`에서 사용할 수 있습니다.

.절차 에어갭 환경에서 공식 이미지 가져오기
. 인터넷에 연결된 시스템에서 필요한 이미지를 당겨옵니다. 예:
 +
 [source,shell]
 ----
 podman pull registry.opensuse.org/opensuse/bind:latest
 podman pull registry.opensuse.org/opensuse/kea:latest
 ----
 . 이미지를 TAR 아카이브에 저장합니다.
 +
 [source,shell]
 ----
 podman save -m -o formula-images.tar registry.opensuse.org/opensuse/bind:latest registry.opensuse.org/opensuse/kea:latest
 ----
 . `formula-images.tar` 파일을 에어갭 시스템으로 전송합니다.
 . air-gapped 시스템에 이미지를 로드합니다.
 +
 [source,shell]
 ----
 podman load -i formula-images.tar
 ----

=== {productname}을(를) {leapmicro}에 배포합니다.

{productname}은(는) 시스템에 설치할 수 있는 RPM 패키지로 필요한 모든 컨테이너 이미지를 제공합니다.

[NOTE]
====
사용자는 필요한 RPM을 내부 네트워크에서 사용할 수 있도록 해야 합니다. 이는 두 번째 {productname} 서버 또는 어떤 종류의 미러를 사용하여 수행할 수 있습니다.
====

.절차 에어갭 환경에서 {productname}을 {leapmicro}에 설치합니다.
. {leapmicro}을 설치합니다.
. 시스템을 업데이트합니다.
. 도구 패키지 및 이미지 패키지 설치($ARCH$를 올바른 아키텍처로 대체):

+

[source,shell]
----
transactional-update pkg install mgradm* mgrctl* uyuni-server*-image*
----
+
. 재부팅합니다.
. {productname}를 `mgradm`와 함께 배포합니다. 에어갭 환경에서는 `--pullPolicy Never` 옵션을 사용하고 싶을 수 있습니다.


{leapmicro}에 {productname} 서버를 설치하는 것에 대한 더 자세한 정보는 xref:container-deployment/uyuni/server-deployment-uyuni.adoc[서버 배포]를 참조하십시오.

{productname} 서버를 업그레이드하려면 사용자는 시스템의 모든 패키지를 업그레이드하고 xref:container-management/updating-server-containers.adoc[서버 업그레이드]에 정의된 절차를 따라야 합니다.
