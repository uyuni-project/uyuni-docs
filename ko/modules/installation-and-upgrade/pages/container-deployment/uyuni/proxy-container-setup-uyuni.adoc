[[proxy-setup-containers-transfer-config]]
== {productname} 프록시 구성 전송


`spacecmd` 명령과 {webui} 방법을 통해 생성하는 두 가지 방식 모두 구성 아카이브를 만듭니다.
이 아카이브는 컨테이너 호스트에서 사용할 수 있어야 합니다.
생성된 이 아카이브를 컨테이너 호스트로 전송합니다.



[[proxy-setup-containers-transfer-start]]
== {productname} 프록시 컨테이너 시작

컨테이너는 `mgrpxy` 명령으로 시작할 수 있습니다.

[[proc-setup-containers-setup-start]]
.절차 {productname} 프록시 컨테이너 시작

. 명령 실행:
+
----
mgrpxy start uyuni-proxy-pod
----
+
. 호출하여 모든 컨테이너가 예상대로 시작되었는지 확인하십시오.
+
----
podman ps
----

5개의 {productname} 프록시 컨테이너가 있어야 하며, `proxy-pod` 컨테이너 Pod의 일부여야 합니다.

* proxy-salt-broker
* proxy-httpd
* proxy-tftpd
* proxy-squid
* proxy-ssh
