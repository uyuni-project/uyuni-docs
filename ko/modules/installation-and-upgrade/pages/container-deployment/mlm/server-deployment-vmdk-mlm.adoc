[[quickstart.sect.vmware.settings]]
== {productname} 가상 머신 설정 - VMware

이 섹션에서는 VMware 환경 내의 {productname} 스토리지 파티션에 필수적인 추가 가상 디스크 생성에 중점을 둔 VMware 구성에 대해 설명합니다.

.절차 VMware 가상 머신 생성

. {productname} 서버 `.vmdk` 파일을 다운로드한 다음, 복사본을 VMware 스토리지로 전송하세요.

. VMware 웹 인터페이스를 사용하여 업로드된 `.vmdk` 파일의 복사본을 만듭니다.
  이 작업은 제공된 `.vmdk` 파일을 vSphere 하이퍼바이저에 적합한 형식으로 변환합니다.

. 게스트 OS 제품군 `Linux` 및 게스트 OS 버전 {sle} 15(64비트)를 기반으로 새 가상 머신을 생성하고 이름을 지정합니다.

// FIXME: do we want to keep this size here? \/

+

. `Hard Disk 2` 500 GB (또는 그 이상)을 추가합니다.

. RAM 및 CPU 수(RAM 16GB 및 CPU 4개 이상)를 구성합니다.

. 필요에 따라 네트워크 어댑터를 설정합니다.

. VM의 전원을 켜고 최초 부팅 대화 상자(키보드 레이아웃, 사용권 계약, 표준 시간대, 루트 비밀번호)를 따릅니다.

. 설치가 완료되면 루트로 로그인합니다.

. 다음 섹션을 진행합니다.

*) 최소 값은 xref:installation-and-upgrade:hardware-requirements.adoc#proxy-hardware-requirements[]을 참조하십시오.

== {sl-micro} 및 {productname} {productnumber} 서버를 등록합니다.

시작하기 전에 {scclongform} - https://scc.suse.com.에서 {productname} 활성화 코드를 받으십시오.

[NOTE]
====
{sl-micro} {microversion} 권한은 {productname} 권한에 포함되어 있으므로 별도의 활성화 코드가 필요하지 않습니다.
====

.절차 {sl-micro} 및 {productname} {productnumber} 등록
. 가상 머신을 부팅합니다.
. `root`로 로그인합니다.
. {sl-micro}을 {scc}에 등록합니다.
+

----
transactional-update register -r <REGCODE> -e <your_email>
----

. 재부팅합니다.

. {productname} {productnumber}를 {scclongform}에 등록합니다.
+

----
transactional-update register -p Multi-Linux-Manager-Server/5.1/x86_64 -r <REGCODE>
----

. 재부팅
. 시스템 업데이트:
+

----
transactional-update
----

. 업데이트가 적용된 경우 재부팅합니다.

. 이 단계는 선택 사항입니다.
  그러나 인프라에 사용자 정의 영구 스토리지가 필요한 경우 `mgr-storage-server` 도구를 사용하십시오.

+

자세한 내용은 `mgr-storage-server --help`을 참조하십시오.
자세한 내용은 [command]\``mgr-storage-server \--help``를 참조하십시오.  이 도구는 컨테이너 스토리지 및 데이터베이스 볼륨 생성을 간소화합니다.
다음 방법으로 명령 사용:

+

----
mgr-storage-server <storage-disk-device> [<database-disk-device>]
----

+
예:
+

----
mgr-storage-server /dev/nvme1n1 /dev/nvme2n1
----

+
[NOTE]
====
이 명령은 `/var/lib/containers/storage/volumes`에 영구 스토리지 볼륨을 생성합니다.

자세한 내용은 다음을 참조하십시오.

* xref:installation-and-upgrade:container-management/persistent-container-volumes.adoc[]
* xref:administration:troubleshooting/tshoot-container-full-disk.adoc[]
