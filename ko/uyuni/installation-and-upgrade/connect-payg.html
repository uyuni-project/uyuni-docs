Pay-as-you-go 인스턴스 연결

Table of Contents
  • 1. Pay-as-you-go 인스턴스 연결
    • 1.1. 인스턴스 연결 데이터 삭제
  • 2. 인스턴스 자격 증명 수집 상태

3곳의 주요 공용 클라우드 공급자(AWS, GCP 및 Azure)에서, SUSE:

  • SLES, SLES for SAP 등을 위한 맞춤형 Pay-as-you-go 제품 이미지를 제공합니다.

  • Pay-as-you-go로 제공되는 제품을 위한 지역별 RMT 서버 미러링 리포지토리를 운영합니다.

이 설명서에서는 기존 Pay-as-you-go 인스턴스를 Uyuni 서버에 연결하는 방법을 설명하고 인스턴스에서 인증서 모음과 관련한 기본적인 정보를 제공합니다. 연결은 Uyuni 서버가 클라우드 RMT 호스트에 연결할 수 있도록 인증 데이터를 추출하기 위해 수행됩니다.

Before using the Pay-as-you-go feature ensure:

  • the Pay-as-you-go instance is launched from the correct SUSE product image (e.g. SLES, SLES for SAP, SLE HPC) to allow access to the desired repositories

  • Uyuni server has connectivity to the Pay-as-you-go instance (ideally in the same region) either directly or via a bastion

  • 유효한 SUSE Customer Center 자격 증명이 Admin  설정 마법사  조직 자격 증명에 입력되었지만, 제품에 대한 제한된 액세스만 제공합니다.

1. Pay-as-you-go 인스턴스 연결

프로시저: 새 Pay-as-you-go 인스턴스 연결

  1. Uyuni Web UI에서 관리  설정 마법사  Pay-as-you-go로 이동한 후 Pay-as-you-go 추가를 클릭합니다.

  2. 페이지 섹션 Pay-as-you-go 연결 설명으로 시작하십시오.

  3. 설명 필드에 설명을 입력하십시오.

  4. 인스턴스 SSH 연결 데이터 페이지 섹션으로 이동하십시오.

  5. Uyuni에서 연결할 인스턴스 DNS 또는 IP 주소를 호스트 필드에 입력하십시오.

  6. SSH 포트 필드에 포트 번호를 입력하거나 기본값인 22를 사용하십시오.

  7. 사용자 필드에 클라우드에 지정된 대로 사용자 이름을 입력하십시오.

  8. 암호 필드에 암호를 입력하십시오.

  9. SSH 개인 키 필드에 인스턴스 키를 입력하십시오.

  10. SSH 개인 키 암호 구문 필드에 키 암호 문구를 입력하십시오.

인증 키는 항상 PEM 형식이어야 합니다.

인스턴스에 직접 연결하지 않고 SSH 배스천을 통해 연결하는 경우 프로시저: SSH 배스천 연결 데이터 추가 명령을 수행하십시오.

그렇지 않으면, 프로시저: Pay-as-you-go 연결 종료 명령을 수행하십시오.

프로시저: SSH 배스천 연결 데이터 추가

  1. 페이지 영역 배스천 SSH 연결 데이터로 이동하십시오.

  2. 호스트 필드에 배스천 호스트 이름을 입력하십시오.

  3. SSH 포트 필드에 배스천 포트 번호를 입력하십시오.

  4. 사용자 필드에 배스천 사용자 이름을 입력하십시오.

  5. 비밀번호 필드에 배스천 비밀번호를 입력하십시오.

  6. SSH 개인 키 필드에 배스천 키를 입력하십시오.

  7. SSH 개인 키 암호 구문 필드에 배스천 키 암호 구문을 입력하십시오.

프로시저: Pay-as-you-go 연결 종료 명령으로 설정 프로세스를 완료하십시오.

프로시저: Pay-as-you-go 연결 종료

  1. 새로운 Pay-as-you-go 연결 데이터 추가를 완료하려면 만들기를 클릭하십시오.

  2. Pay-as-you-go 연결 데이터 자세히 페이지로 돌아갑니다. 업데이트된 연결 상태는 정보의 상단 영역에 표시됩니다.

  3. 연결 상태는 관리자 > 설정 마법사 > Pay-as-you-go 화면에도 표시됩니다.

  4. 인스턴스에 대한 인증 데이터가 올바르면, 상태 열에 "자격 증명이 업데이트됨" 메시지가 표시됩니다.

언제든 잘못된 데이터를 입력하면 새로 생성된 인스턴스가 관리자 > 설정 마법사 > Pay-as-you-go에 표시되고 상태 열에 오류 메시지가 표시됩니다.

서버에서 인증 데이터를 사용할 수 있게 되면 사용 가능한 제품 목록이 업데이트됩니다.

사용 가능한 제품은 Pay-as-you-go 인스턴스에 설치된 것과 동일한 제품군 및 아키텍처의 모든 버전입니다. 예를 들어, 인스턴스에 SLES 15 SP1 제품이 설치되어 있으면 SLES 15 SP2, SLES 15 SP3 및 SLES 15 SP4가 관리자 > 설정 마법사 > 제품에 자동으로 표시됩니다.

제품이 사용 가능한 것으로 표시되면 사용자는 제품 이름 옆의 확인란을 선택하고 제품 추가를 클릭하여 Uyuni에 제품을 추가할 수 있습니다.

성공 메시지가 표시되면 소프트웨어 > 채널 목록 > 전체로 이동하여 Web UI에서 새로 추가된 채널을 확인할 수 있습니다.

각 채널의 동기화 진행 상황을 모니터링하려면 Uyuni 서버의 /var/log/rhn/reposync 디렉토리에서 로그 파일을 확인하십시오.

1.1. 인스턴스 연결 데이터 삭제

다음 프로시저는 인스턴스의 SSH 연결 데이터를 삭제하는 방법을 설명합니다.

프로시저: 인스턴스에 대한 연결 데이터 삭제

  1. 관리자 > 설정 마법사 > Pay-as-you-go를 여십시오.

  2. 기존 인스턴스 목록에서 인스턴스를 검색하십시오.

  3. 인스턴스 세부 정보를 클릭하십시오.

  4. 삭제 버튼을 선택하고 선택 항목을 확인하십시오.

  5. 인스턴스 목록으로 되돌아갑니다. 방금 삭제한 항목은 더 이상 표시되지 않습니다.

2. 인스턴스 자격 증명 수집 상태

Uyuni 서버는 인스턴스에서 수집한 자격 증명을 사용하여 RMT 서버에 연결하고 reposync를 사용하여 패키지를 다운로드합니다. 이러한 자격 증명은 정의된 SSH 연결 데이터를 사용하여 taskomatic에 의해 10분마다 새로 고쳐집니다. RMT 서버에 대한 연결은 Pay-as-you-go 인스턴스에서 수집된 마지막으로 알려진 인증 자격 증명을 항상 사용합니다.

수집한 Pay-as-you-go 인스턴스 자격 증명의 최신 상태는 상태 열 또는 인스턴스 세부 정보 페이지에 표시됩니다. 인스턴스에 연결할 수 없으면 자격 증명 업데이트 프로세스가 실패합니다.

SSH 연결 데이터가 명시적으로 삭제된 경우를 제외하고, Pay-as-you-go 인스턴스는 Uyuni 서버에 연결된 상태를 유지합니다. 인스턴스에 대한 SSH 연결 데이터를 삭제하려면 프로시저: 인스턴스에 대한 연결 데이터 삭제 명령을 사용하십시오.

항상 Uyuni 서버에서 Pay-as-you-go 인스턴스에 액세스할 수 있는 것은 아닙니다.

  1. 인스턴스가 존재하지만 중지된 경우에는 마지막으로 알려진 자격 증명을 사용하여 인스턴스에 연결을 시도합니다. 자격 증명 유효 기간은 클라우드 공급자에 따라 다릅니다.

  2. 인스턴스가 더 이상 존재하지 않지만 여전히 SUMA에 등록되어 있으면, 해당 인증서는 더 이상 유효하지 않으며 인증이 실패합니다. 오류 메시지는 상태 열에 표시됩니다.

    오류 메시지는 인스턴스를 사용할 수 없다는 것만을 나타냅니다. 인스턴스 상태에 대한 자세한 진단은 클라우드 공급자에서 수행해야 합니다.
