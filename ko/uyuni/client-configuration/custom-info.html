사용자 정의 시스템 정보

클라이언트에 대한 사용자 정의 시스템 정보를 포함할 수 있습니다. 시스템 정보는 클라이언트에 할당할 수 있는 키:값 페어로 정의됩니다. 예를 들어 특정 프로세서에 대한 키:값 페어를 정의한 다음, 이 키를 프로세서가 설치된 모든 클라이언트에 할당할 수 있습니다. 사용자 정의 시스템 정보는 분류되며 Uyuni Web UI를 사용해 검색할 수 있습니다.

시작하기 전에 사용자 정의를 저장할 수 있게 허용하는 키를 생성해야 합니다.

프로시저: 사용자 정의 시스템 정보 키 생성

  1. Uyuni Web UI에서 시스템  사용자 정의 시스템 정보로 이동하여 키 생성을 클릭합니다.

  2. 키 레이블 필드에 키의 이름을 추가합니다. 공백을 사용하면 안 됩니다. 예를 들어, intel-x86_64-quadcore를 추가합니다.

  3. 설명 필드에 필요한 추가 정보를 입력합니다.

  4. 필요한 각 키에 이 작업을 반복합니다.

기존 클라이언트의 경우, 이 정보는 Uyuni 데이터베이스에 저장됩니다. Salt 클라이언트의 경우, 이 정보는 Salt 열에 저장됩니다. 이 정보는 다음과 같은 명령을 사용하여 Salt 클라이언트에서 검색할 수 있습니다.

salt $minionid pillar.get custom_info:key1

이 명령을 실행한 후의 출력은 다음과 같습니다.

$minionid:
    val1

사용자 정의 시스템 정보 키를 몇 개 생성했으면 이 키를 클라이언트에 적용할 수 있습니다.

프로시저: 사용자 정의 정보 키를 클라이언트에 적용

  1. Uyuni Web UI에서 시스템으로 이동하여 사용자 정의 정보를 적용할 클라이언트를 클릭하고, 상세 정보  사용자 정의 정보 탭으로 이동합니다.

  2. 값 생성을 클릭합니다.

  3. 적용하려는 값을 찾아 키 레이블을 클릭합니다.

  4. 값 필드에 추가 정보를 입력합니다.

  5. 키 업데이트를 클릭하여 사용자 정의 정보를 클라이언트에 적용합니다.

구성 관리에 대한 자세한 설명은 구성 관리에서 참조하십시오.
