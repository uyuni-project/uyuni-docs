= {sle-micro} 클라이언트 등록
:revdate: 2025-02-06
:page-revdate: {revdate}

이 섹션에는 {x86_64}, {arm64} 및 {zsystems}(s390x) 아키텍처에서 {sle-micro} 운영 체제 5.1, 5.2, 5.3, 5.4 및 5.5를 실행하는 클라이언트를 등록하는 방법에 대한 정보가 포함되어 있습니다.

{sle-micro}는 트랜잭션 업데이트를 사용합니다. 트랜잭션 업데이트는 원자적(모든 업데이트가 성공한 경우에만 모든 업데이트가 적용됨)이며 롤백을 지원합니다. 시스템이 재부팅될 때까지 변경 사항이 적용되지 않으므로 실행 중인 시스템에 영향을 주지 않습니다.

트랜잭션 업데이트 및 재부팅에 대한 자세한 내용은 link:https://documentation.suse.com/sles/html/SLES-all/cha-transactional-updates.html[]에서 확인할 수 있습니다.

이 장에서 설명하는 클라이언트 등록은 다음 단계로 구성됩니다.

* {webui} 또는 명령줄 인터페이스를 사용하여 새 클라이언트의 운영 체제 및 아키텍처에 대한 소프트웨어 채널을 동기화합니다.

ifeval::[{uyuni-content} == true]


* 명령줄 인터페이스를 사용하여 새로 추가된 운영 체제에 대한 클라이언트 도구 리포지토리를 동기화합니다.

endif::[]

* {webui} 또는 명령줄 인터페이스를 사용하여 {productname} 서버와 함께 클라이언트를 등록합니다.

추가 섹션에서는 등록 절차를 확인하고 검증하는 방법을 설명합니다.

== 클라이언트 등록

[WARNING]
====
{sle-micro} 클라이언트는 등록 후 재부팅해야 합니다.

Following the bootstrapping process, automatic booting is disabled on {sle-micro} clients. This change was implemented due to intermittent automatic reboots, which could occur before {salt} could relay the results of applying the bootstrap {salt} state.

등록이 완료되면 재부팅이 자동으로 예약되지만, 기본 재부팅 관리자 유지보수 기간이 적용됩니다. 이 기간은 클라이언트가 등록되고 몇 시간이 지난 후가 될 수 있습니다. 등록 스크립트가 완료된 후 등록 속도를 향상하고 시스템이 시스템 목록에 표시되도록 하려면 클라이언트를 수동으로 재부팅하는 것이 좋습니다.
====

[[clients-slemicro-reboot]]
== {sle-micro} 재부팅

{sle-micro}는 트랜잭션 시스템입니다. 일반적으로 트랜잭션 업데이트는 다양한 재부팅 방법을 지원합니다. {productname}에서 관리하는 시스템에서 재부팅을 수행하려면 [systemitem]``systemd``를 사용하는 것이 좋습니다. 다른 방법을 사용하면 원하지 않은 동작이 발생할 수 있습니다.

{productname}에서 트랜잭션 시스템을 부트스트랩할 때 시스템이 기본 구성인 경우 [systemitem]``systemd``는 재부팅 방법([literal]``REBOOT_METHOD``)대로 구성됩니다. 이러한 구성을 사용하면 {productname}가 재부팅 작업을 제어할 수 있으며 재부팅을 즉시 수행하거나 {productname}에서 원하는 일정을 예약할 수 있습니다.


=== 배경 정보

기본적으로 클라이언트를 설치하는 동안 재부팅 방법은 [옵션]``auto``로 설정됩니다. [option]``auto`` 부팅 방법을 사용하면 서비스가 실행 중인 경우 구성된 정책에 따라 시스템을 재부팅하기 위해 [command]``rebootmgrd``가 사용됩니다. 정책은 즉시 재부팅 또는 유지보수 기간 중 재부팅입니다. 자세한 내용은 [command]``rebootmgrd``(8) 사용자 지정 페이지를 참조하십시오. [command]``rebootmgrd``가 실행 중이 아닌 경우 {productname}는 [command]``systemctl reboot``를 호출합니다.

[NOTE]
====
[command]``systemd`` 이외의 방법을 사용하면 원하지 않은 동작이 발생할 수 있습니다.
====

// 2023-05-23, ke: The rest is commented and the user will not see the

Otherwise, the only thing mlm will be able to do is to ask for a reboot and it will be managed in the peculiarities of each reboot method.
