{oes} 클라이언트를 {productname} 서버에 등록하기 전에 필요한 소프트웨어 채널을 추가하고 이 채널을 동기화해야 합니다.

이 절차에 필요한 제품은 다음과 같습니다.

또는 명령 프롬프트에서 채널을 추가할 수 있습니다. 이 절차에 필요한 채널은 다음과 같습니다.

// "channel_label": "oes23.4-pool-x86_64"
// "channel_name": "OES23.4-Pool for x86_64",
//
// 24.4:
// "channel_label": "oes24.4-pool-x86_64",
// "channel_name": "OES24.4-Pool for x86_64",

[[oes-channels-cli]]
[cols="1,1,1", options="header"]
.OES 제품 - CLI
|===

| OS Version
| Base Channel
| Name

| {oes} 25.4
| oes25.4-pool-x86_64
| OES25.4-Pool for x86_64

| {oes} 24.4
| oes24.4-pool-x86_64
| OES24.4-Pool for x86_64

| {oes} 23.4
| oes23.4-pool-x86_64"
| OES23.4-Pool for x86_64

|===

이전 제품의 채널 이름을 찾으려면 {productname} 서버의 명령 프롬프트에서 루트 권한으로 [command]``mgr-sync`` 명령을 사용합니다.

----
mgr-sync list --help
----

그런 다음 원하는 인수를 지정합니다. 예를 들어, [literal]``channels``:

----
mgr-sync list channels [-c]
----

== GPG 키 관리

클라이언트는 소프트웨어 패키지가 설치되기 전에 GPG 키를 사용해 신뢰성을 확인합니다. 신뢰할 수 있는 소프트웨어만 클라이언트에 설치할 수 있습니다.

[IMPORTANT]
====
클라이언트의 보안을 위해 GPG 키를 신뢰하는 것이 중요합니다. 어떤 키가 필요하고 신뢰할 수 있는지 결정하는 것은 관리자가 수행해야 하는 작업입니다. GPG 키를 신뢰할 수 없는 경우 소프트웨어 채널을 사용할 수 없기 때문에 채널을 클라이언트에 할당하는 결정은 키를 신뢰하는 결정에 따라 달라집니다.
====

GPG 키에 대한 자세한 내용은 xref:client-configuration:gpg-keys.adoc[]에서 확인할 수 있습니다.

////
[NOTE]
====
Use the same GPG key for both {sles}{nbsp}15 and {sles}{nbsp}12 clients.
The correct key is called ``sle12-gpg-pubkey-39db7c82.key``.
====
////
