= {sle} 클라이언트 등록
:revdate: 2025-05-19
:page-revdate: {revdate}

이 섹션에는 {sle} 운영 체제를 실행하는 클라이언트를 등록하는 방법에 대한 정보가 포함되어 있습니다.

다음을 포함하여 모든 {sle} 제품을 준비하려면 이 장의 지침을 사용하십시오.

* SAP용 {sles}
* {sle} 데스크톱
* {sle}
* {sle} Real Time

이전 {sle} 버전 및 서비스 팩에서도 이러한 지침을 사용할 수 있습니다.

----
mgr-sync list --help
----

그런 다음 원하는 인수를 지정합니다. 예를 들어, [literal]``channels``:

----
mgr-sync list channels [-c]
----

== GPG 키 관리

클라이언트는 소프트웨어 패키지가 설치되기 전에 GPG 키를 사용해 신뢰성을 확인합니다. 신뢰할 수 있는 소프트웨어만 클라이언트에 설치할 수 있습니다.

[IMPORTANT]
====
클라이언트의 보안을 위해 GPG 키를 신뢰하는 것이 중요합니다. 어떤 키가 필요하고 신뢰할 수 있는지 결정하는 것은 관리자가 수행해야 하는 작업입니다. GPG 키를 신뢰할 수 없는 경우 소프트웨어 채널을 사용할 수 없기 때문에 채널을 클라이언트에 할당하는 결정은 키를 신뢰하는 결정에 따라 달라집니다.
====

GPG 키에 대한 자세한 내용은 xref:client-configuration:gpg-keys.adoc[]에서 확인할 수 있습니다.

[NOTE]
====
* {sles}{nbsp}15 및 {sles}{nbsp}12 클라이언트에 동일한 GPG 키를 사용합니다. 올바른 키는 ``sle12-gpg-pubkey-39db7c82.key``입니다.

* {sles}{nbsp}16 clients use a different GPG key. The correct key is called ``suse16-gpg-pubkey-09d9ea69.key``.
====


endif::[]
