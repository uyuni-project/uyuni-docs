기본적으로 {productname} {webui}에서는 사용자가 30{nbsp}분 후에 다시 로그인해야 합니다. 환경에 따라 로그인 시간 제한 값을 조정할 수 있습니다.

값을 조정하려면 [path]``rhn.conf`` 및 [path]``web.xml``을 모두 변경해야 합니다. [path]``/etc/rhn/rhn.conf``에서는 값을 초 단위로 설정하고 [path]``web.xml``에서는 분 단위로 값을 설정해야 합니다. 두 값은 시간의 양이 동일해야 합니다.

예를 들어, 제한 시간 값을 한 시간으로 변경하려면 [path]``rhn.conf``의 값을 3600초로, [path]``web.xml``의 값을 60분으로 설정합니다.



.절차: {webui} 로그인 시간 제한 값 조정
. 컨테이너 호스트에서, 서버 컨테이너 내의 명령줄을 엽니다.

+

[source, shell]
----
mgrctl term
----

+

--

.. [path]``/etc/rhn/rhn.conf``를 열고 새로운 시간 제한 값(초)을 포함하도록 다음 라인을 추가하거나 편집합니다.

+

----
web.session_database_lifetime = <Timeout_Value_in_Seconds>
----

.. 파일을 저장한 후 닫습니다.

.. [path]``/etc/tomcat/web.xml``를 열고 새로운 시간 제한 값(분)을 포함하도록 다음 라인을 추가하거나 편집합니다.

+

[source, xml]
----
<session-timeout>Timeout_Value_in_Minutes</session-timeout>
----

.. 파일을 저장한 후 닫습니다.

--

+

. 컨테이너 호스트에서 서버를 다시 시작하여 새 구성을 적용합니다.

+

[source, shell]
----
systemctl status uyuni-check-database.service
----
