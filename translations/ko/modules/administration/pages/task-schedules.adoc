[[ref.webui.admin.schedules]]
= 작업 스케줄
:revdate: 2025-07-03
:page-revdate: {revdate}


사전 정의된 모든 작업 묶음은 menu:관리자[작업 일정] 아래에 나열됩니다.

image::admin_task_schedules.png[scaledwidth=80%]


menu:{productname} 일정[일정 이름]을 클릭하면 menu:일정 이름[기본 일정 세부사항]이 열립니다. 비활성화하거나 빈도를 변경할 수 있습니다.

설정과 함께 일정을 업데이트하려면 btn:[일정 편집]을 클릭하십시오.

일정을 비활성화하려면 오른쪽 상단의 btn:[일정 비활성화]를 클릭합니다.

[WARNING]
====
일정은 {productname}의 올바른 작동을 위해 필수이므로 필요하다고 확신하는 경우에만 일정을 비활성화해야 합니다.
====

작업이 비활성화되어도 목록에는 계속 표시됩니다. menu:{productname} 일정[일정 이름]을 클릭하면 btn:[일정 활성화]를 눌러 작업을 다시 활성화할 수 있습니다.


무리 이름을 클릭하면 해당 묶음 유형의 실행 목록 및 해당 상태가 표시됩니다.

시작 시간 링크를 클릭하면 menu:일정 이름[기본 일정 세부 사항]으로 돌아갑니다.



== 사전 정의된 작업 묶음


다음과 같은 사전 정의 작업 묶음은 기본적으로 예약되며 구성할 수 있습니다.

menu:errata-cache-default:[]::
필요한 경우 자동 정오표 업데이트를 예약합니다.

menu:channel-repodata-default:[]::
리포지토리 메타데이터 파일을 (다시) 생성합니다.

menu:cleanup-data-default:[]::
데이터베이스에서 부실한 패키지 변경 로그 및 모니터링 시계열 데이터를 정리합니다.

menu:clear-taskologs-default:[]::
데이터베이스에서 작업 유형에 따라 지정된 일 수를 초과한 작업 엔진(taskomatic) 과거 기록 데이터를 지웁니다.

menu:cobbler-sync-default:[]::
{productname}에서 {cobbler}로 배포 및 프로파일 데이터를 동기화합니다. {cobbler}에서 제공하는 자동 설치에 대한 자세한 내용은 xref:client-configuration:autoinst-intro.adoc[]에서 확인할 수 있습니다.

menu:compare-configs-default:[]::
구성 채널에 저장된 구성 파일과 모든 구성 가능 서버에 저장된 파일을 비교합니다. 비교를 검토하려면 menu:시스템[] 탭을 클릭하고 관심이 있는 시스템을 선택합니다. menu:구성[파일 비교]로 이동합니다. 자세한 내용은 xref:reference:systems/system-details/sd-configuration.adoc#sd-config-compare-files[]에서 확인할 수 있습니다.

menu:cve-server-channels-default:[]::
menu:감사[CVE 감사] 페이지에 결과를 표시하기 위해 사용되는 미리 계산된 내부 CVE 데이터를 업데이트합니다. menu:감사[CVE 감사] 페이지의 검색 결과는 이 일정의 마지막 실행으로 업데이트됩니다. 자세한 내용은 xref:reference:audit/audit-cve-audit.adoc[]에서 확인할 수 있습니다.

menu:daily-status-default:[]::
관련 주소로 일일 보고서 이메일을 전송합니다. 특정 사용자에 대한 알림을 구성하는 방법에 대한 자세한 내용은 xref:reference:users/user-details.adoc[]에서 확인할 수 있습니다.

menu:errata-advisory-map-sync-default[]::
내부 {suse} 패치 공급업체 권고 사항 데이터베이스 테이블을 업데이트합니다. 가능한 경우, {suse}에서 제공한 원본 권고 사항은 각 패치 상세 정보의 공급업체 권고 사항 섹션에 표시됩니다.

menu:errata-cache-default:[]::
내부 패치 캐시 데이터베이스 테이블을 업데이트하며, 이는 각 서버에 대한 업데이트가 필요한 패키지를 찾기 위한 용도로 사용됩니다. 또한, 특정 패치에 관심이 있을 수 있는 사용자에게 알림 이메일도 전송합니다. 패치에 대한 자세한 내용은 xref:reference:patches/patches-menu.adoc[]에서 확인할 수 있습니다.

menu:errata-queue-default:[]::
자동 업데이트(패치)를 수신하도록 구성된 서버를 대기열에 추가합니다.

menu:gatherer-matcher-default[]::
Virtual Host Managers에 구성된 Virtual Host Gatherer를 실행하여 가상 호스트 데이터를 수집합니다. 업데이트된 데이터를 사용할 수 있게 되면 Subscription Matcher 작업이 실행됩니다.

menu:kickstart-cleanup-default:[]::
오래된 {kickstart} 세션 데이터를 정리합니다.

menu:kickstartfile-sync-default[]::
구성 마법사에서 생성한 {kickstart} 프로파일에 해당하는 {cobbler} 파일을 생성합니다.

menu:mgr-forward-registration-default[]::
클라이언트 등록 데이터를 {scclongform}과(와) 동기화합니다. 기본적으로 신규, 변경 또는 삭제된 클라이언트 데이터가 전달됩니다. [path]``/etc/rhn/rhn.conf``에 설정된 동기화를 비활성화하려면 다음을 수행합니다.
+
----
server.susemanager.forward_registration = 0
----
+

[WARNING]
====
{scc}와의 데이터 동기화를 비활성화하면 RMT, SMT, {productname} 및 {scc}에 직접 등록된 클라이언트 사이에서 관리형 클라이언트의 가시성이 저하됩니다.

데이터를 동기화하면 등록된 모든 클라이언트를 일관되게 확인할 수 있습니다.

https://suselinux.fra1.qualtrics.com/jfe/form/SV_0ooNnrY0rYuQScS[서비스 개선에 도움이 되도록 거부하시는 이유를 알려주십시오.]
====

menu:mgr-sync-refresh-default[]::
{scclongform}과(와)동기화(``mgr-sync-refresh``)합니다. 기본적으로 모든 사용자 정의 채널도 이 작업의 일부로 동기화됩니다. 사용자 정의 채널 동기화에 대한 자세한 내용은 xref:administration:custom-channels.adoc#_custom_channel_synchronization[]에서 확인할 수 있습니다.

menu:minion-action-chain-cleanup-default[]::
오래된 작업 체인 데이터를 정리합니다.

menu:minion-action-cleanup-default[]::
파일 시스템에서 부실한 클라이언트 작업 데이터를 삭제합니다. 우선, {salt} 작업 캐시에 저장된 해당 결과를 검색하여 완료되지 않은 작업의 완료를 시도합니다. 서버에서 작업 결과가 누락된 경우, 완료되지 않은 작업이 발생할 수 있습니다. 성공적으로 완료된 작업의 경우 실행된 스크립트 파일 등의 아티팩트를 제거합니다.

menu:minion-checkin-default[]::
클라이언트에 대한 정기 체크인을 수행합니다.

menu:notifications-cleanup-default[]::
만료된 알림 메시지를 정리합니다.

menu:oval-data-sync-default[]::
CVE 감사 쿼리의 정확도를 향상하기 위해 필수 OVAL 데이터를 생성합니다.

menu:package-cleanup-default[]::
파일 시스템에서 부실 패키지 파일을 삭제합니다.

menu:reboot-action-cleanup-default[]::
여섯 시간을 초과하여 대기 중인 모든 재부팅 작업은 실패한 것으로 표시되고 관련 데이터가 데이터베이스에서 정리됩니다. 재부팅 작업 예약에 대한 자세한 내용은 xref:reference:systems/system-details/sd-provisioning.adoc#sd-power-management[]에서 확인할 수 있습니다.

menu:session-cleanup-default[]::
부실 웹 인터페이스 세션, 일반적으로 사용자가 로그인할 때 임시로 저장한 후 로그아웃하기 전에 브라우저를 닫는 데이터를 정리합니다.

menu:ssh-service-default[]::
클라이언트가 `SSH 푸시` 연락 방법으로 구성된 경우 SSH를 통해 {productname}(으)로 체크인하라는 메시지가 표시됩니다. 또한 재부팅 후 작업 체인을 재개합니다.

menu:system-overview-update-queue-default[]::
시스템 개요 데이터를 업데이트합니다.

menu:system-profile-refresh-default[]::
모든 시스템에서 하드웨어 새로 고침을 실행합니다. 이 작업은 매월 수행되며 {productname} 서버의 부하를 증가시킬 수 있습니다. 이 작업에서는 xref:specialized-guides:salt/salt-rate-limiting.adoc[Salt 요청량 제한]을 사용합니다. 배치 크기 조정에 대한 내용은 xref:specialized-guides:large-deployments/tuning.adoc#java-salt-batch-size[]에서 확인할 수 있습니다.

menu:token-cleanup-default[]::
{salt} 클라이언트가 패키지 및 메타데이터를 다운로드하기 위해 사용하는 만료된 리포지토리 토큰을 삭제합니다.

menu:update-payg-default[]::
구성된 {payg} 클라우드 인스턴스에서 인증 데이터를 수집합니다.

menu:update-reporting-default[]::
로컬 보고 데이터베이스를 업데이트합니다.

menu:update-reporting-hub-default[]::
주변 {productname} 서버에서 모든 보고 데이터를 수집하고 Hub Reporting Database를 업데이트합니다.

menu:update-system-overview-default[]::
시스템 개요 데이터가 최신 상태인지 정기적으로 확인합니다.

menu:uuid-cleanup-default[]::
오래된 UUID 레코드를 정리합니다.
