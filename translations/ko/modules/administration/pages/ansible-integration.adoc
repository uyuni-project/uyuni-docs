= Ansible 통합
:revdate: 2025-03-07
:page-revdate: {revdate}

Ansible은 컴퓨터 클라이언트 시스템을 관리하는 도구입니다. 더 많은 정보를 원하시면, link:https://www.ansible.com[]을 참조하십시오.

{productname}은(는) Ansible 제어 노드 관리를 지원합니다. 자세한 내용은 xref:administration:ansible-setup-control-node.adoc#at.ansible.overview[]에서 확인할 수 있습니다.

Ansible 제어 노드에서 지원하는 버전은 Ansible 11.3입니다. 운영 체제 공급업체의 공식 리포지토리에서 설치해야 합니다. 예를 들어, {sle} 15 SP6 및 SP7에서 Ansible은 [literal]``Systems Management Module``을 통해 제공됩니다. {sle} 이외의 운영 체제를 실행하는 제어 노드의 경우, 해당 배포판과 함께 제공되는 Ansible을 사용하십시오.

Ansible 소프트웨어는 {productname} Proxy 및 {productname} {smr} Branch Server에도 사용할 수 있습니다.



[[at.ansible.overview]]
== 기능 개요


{productname}를 사용하면 시스템 관리자가 Ansible 제어 노드를 작동할 수 있습니다. 지원되는 기능은 다음과 같습니다.

* 인벤토리 파일 검사
* 플레이북 검색
* 플레이북 실행

자세한 내용은 다음을 참조하십시오.

* 인벤토리는 관리형 Ansible 노드의 정렬된 목록입니다. 인벤토리 구성에 대한 자세한 내용은 https://docs.ansible.com/ansible/latest/inventory_guide/intro_inventory.html[]에서 확인할 수 있습니다.
* 플레이북은 인벤토리 관리 방법을 설명합니다. 플레이북에 대한 자세한 내용은 https://docs.ansible.com/ansible/latest/playbook_guide/playbooks_intro.html[]에서 확인할 수 있습니다.

== 요구사항 및 기본 구성

Ansible 기능을 사용하려면 기존 Ansible Control Node를 {productname} 서버에 등록해야 합니다. {webui}에서 등록된 시스템의 menu:시스템 상세 사항[속성] 페이지에서 [guimenu]``추가 기능 시스템 유형`` 목록의 ``Ansible Control Node`` 시스템 유형을 활성화해야 합니다.

// FIXME: check it!
``Ansible Control Node`` 시스템 유형을 활성화하면 highstate에 추가하여 [package]``ansible`` 패키지를 시스템에 설치하고 menu:시스템 세부 사항[Ansible] 탭에서 Ansible 기능을 활성화할 수 있습니다.

다음 단계로 menu:시스템 세부 사항[Ansible > 제어 노드] 페이지에서 Ansible 플레이북 디렉토리 및 인벤토리 파일에 대한 경로를 구성합니다. 인벤토리 경로로 표준 Ansible 인벤토리 경로 [literal]``/etc/ansible/hosts``를 사용할 수 있습니다. 플레이북 디렉터리로는 플레이북 파일이 저장된 제어 노드의 모든 디렉토리를 사용할 수 있습니다. 플레이북 디렉토리에는 [literal]``.yml`` 파일 또는 [literal]``.yml`` 파일이 있는 하위 디렉토리가 포함됩니다.

Ansible 제어 노드 설치 및 설정에 대해서는 xref:administration:ansible-setup-control-node.adoc[]에서 확인할 수 있습니다.



[[at.ansible.inventory-inspection]]
== 인벤토리 검사

인벤토리 경로를 정의한 후에는 {productname}를 사용하여 내용을 검사할 수 있습니다.

.절차: {webui}에서 인벤토리 검사
. {productname} {webui}에서 menu:시스템 세부 사항[Ansible > 인벤토리]로 이동합니다.
. 인벤토리 경로를 클릭하면 제어 노드에서 실시간으로 인벤토리 검사를 실행할 수 있습니다.



[[at.ansible.playbook-discovery]]
== 플레이북 검색

플레이북 디렉토리를 정의한 후 menu:시스템 세부 사항[Ansible > 플레이북] 페이지에서 플레이북을 검색할 수 있습니다.

인벤토리 검사와 마찬가지로 플레이북 검색 작업은 제어 노드에서 실시간으로 실행됩니다.



[[at.ansible.playbook-execution]]
== 플레이북 실행

menu:시스템 세부 정보[Ansible > 플레이북] 페이지에서 플레이북 실행을 예약할 수 있습니다. 실행할 플레이북을 선택한 후 [guimenu]``인벤토리 경로`` 드롭다운 메뉴의 [guimenu]``플레이북 실행 예약`` 대화 상자에서 실행할 인벤토리 파일을 선택할 수 있습니다. 항목을 선택하지 않으면 제어 노드에 구성된 기본 인벤토리가 사용됩니다. 드롭다운 메뉴는 인벤토리 경로에 정의한 인벤토리와 플레이북 디렉토리에서 로컬로 검색된 인벤토리로 채워집니다. 해당 항목은 플레이북 세부 정보에서 [guimenu]``사용자 정의 인벤토리`` 항목으로 표시됩니다. 임의의 인벤토리 경로를 입력할 수도 있습니다.

그런 다음 플레이북 실행 시간을 선택하거나 작업 체인을 선택합니다. 최종적으로 {productname}는 플레이북을 제어 노드의 작업으로 실행합니다. 작업 결과는 작업 세부 사항 페이지에서 볼 수 있습니다.
