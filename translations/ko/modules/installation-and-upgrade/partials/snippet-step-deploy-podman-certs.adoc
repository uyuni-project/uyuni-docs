. {productname}을(를) 배포합니다.

+

[NOTE]
====
VM 이미지를 마이그레이션 타겟으로 사용하는 경우, 마지막 단계에서 [command]``mgradm install`` 대신 [command]``mgradm migrate`` 명령을 실행합니다.
====

+

SSL 인증서 유형(자체 서명 또는 타사)에 따라 다음 명령 중 하나를 실행합니다.
  ``<FQDN>``을 {productname} 서버의 정규화된 도메인 이름으로 바꿉니다.

+

--
* {productname}에서 제공하는 자체 서명 인증서 사용:

+

[source, shell]
----
mgradm install podman <FQDN>
----

* 타사 SSL 인증서 플래그를 사용하여 SSL 인증서 임포트(필요한 인증서가 모두 포함되지 않은 경우 예제 조정 가능):

+

[source, shell]
----
mgradm install podman <FQDN> \
  --ssl-ca-intermediate <strings> \
  --ssl-ca-root <string> \
  --ssl-server-cert <string> \
  --ssl-server-key <string> \
  --ssl-db-ca-intermediate <strings> \
  --ssl-db-ca-root <string> \
  --ssl-db-cert <string> \
  --ssl-db-key <string>
----

+

자세한 내용은 [command]``mgradm install podman --help``에서 확인할 수 있습니다.
