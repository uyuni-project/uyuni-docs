.사용할 수 있는 서버 이미지
[cols="3, 3", options="header"]
|===
| 아키텍처 | 이미지 형식

| aarch64| qcow2, vmdk

| x86_64  | qcow2, vmdk, 원시, 자체 설치 프로그램

| ppc64le | 원시, 자체 설치 프로그램

|===

__* s390x용으로 사용할 수 있는 2개의 스토리지 옵션: CDL DASD 및 FBA.__


[[quickstart.sect.kvm.settings]]
== 가상 머신 관리자(virt-manager) 설정

*virt-manager* 를 사용하여 새 가상 머신을 생성할 때 다음 설정을 입력합니다.

[NOTE]
====
이 표에는 최소 요구 사항이 명시되어 있습니다. 이는 1개의 클라이언트가 있는 서버와 같은 빠른 테스트 설치에 적합합니다. 프로덕션 환경을 사용하고 디스크 공간에 대한 배경 정보가 필요한 경우 xref:installation-and-upgrade:hardware-requirements.adoc[] 항목을 참조하십시오.
====


[cols="1,1", options="header"]
|===
2+<| KVM 설정
| 설치 방법 | 기존 디스크 이미지 임포트
 | OS:                 | Linux
 | 버전:            | {productname}-Server.x86_64-{productnumber}.*.qcow2
 | 메모리:             | 최소 *)
 | CPU:              | 최소 *)
 | 저장소 형식:     | _.qcow2_ 40GB(기본) 루트 파티션
 | 이름:               | test-setup
 | 네트워크             | Bridge _br0_
|===

*) 최솟값은 xref:installation-and-upgrade:hardware-requirements.adoc#server-hardware-requirements[]에서 확인할 수 있습니다.

.절차: {sl-micro} 및 {productname} {productnumber} 등록
. 가상 머신을 부팅합니다.
. **`root`** 로 로그인합니다.
+
// note for 5.0 backport: 5.0 should have sle-micro
. SCC를 사용하여 {sl-micro}를 등록합니다.
+

----
transactional-update register -r <REGCODE> -e <your_email>
----

. 재부팅합니다.

. {scclongform}을(를) 사용하여 {productname} {productnumber}을(를) 등록합니다.
+

----
transactional-update register -p Multi-Linux-Manager-Server/5.1/x86_64 -r <REGCODE>
----

. 재부팅합니다.

. 시스템 업데이트:
+

----
transactional-update
----

. 업데이트가 적용된 경우 재부팅합니다.

. 이 단계는 선택 사항입니다. 그러나 인프라에 사용자 정의 영구 스토리지가 필요한 경우 [command]``mgr-storage-server`` 도구를 사용하십시오.

+

자세한 내용은 [command]``mgr-storage-server --help``를 참조하십시오. 이 도구는 컨테이너 스토리지 및 데이터베이스 볼륨 생성을 간소화합니다. 이 명령은 다음 방법으로 사용하십시오.

+

----
mgr-storage-server <storage-disk-device> [<database-disk-device>]
----
+
예:
+

----
mgr-storage-server /dev/nvme1n1 /dev/nvme2n1
----

+
[NOTE]
====
이 명령은 [path]``/var/lib/containers/storage/volumes``의 영구 스토리지 볼륨을 지정된 스토리지 장치로 이동합니다.

자세한 내용은 다음을 참조하십시오.

* xref:installation-and-upgrade:container-management/persistent-container-volumes.adoc[]
* xref:administration:troubleshooting/tshoot-container-full-disk.adoc[]
