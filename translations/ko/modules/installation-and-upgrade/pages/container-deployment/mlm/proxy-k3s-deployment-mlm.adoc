[[installation-proxy-containers-k3s-k3s]]
== K3s 설치


[NOTE]
====
{productname} 프록시는 단일 노드 클러스터의 {sl-micro} 위에서 실행되는 K3s에서 지원됩니다. 이를 다른 Kubernetes 환경에 배포해야 하는 경우 지원팀에 문의하여 평가를 요청해야 합니다.
====

컨테이너 호스트 머신에 [literal]``K3s``를 설치합니다([literal]``<K3S_HOST_FQDN>``을 K3s 호스트의 FQDN으로 바꿈).

[source, shell]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--tls-san=<K3S_HOST_FQDN>" sh -
----

[[installation-proxy-containers-k3s-helm]]
== 도구 설치

설치하려면 [literal]``mgrpxy`` 및 [literal]``helm`` 패키지가 필요합니다.

[literal]``mgrpxy`` 및 ``helm`` 패키지는 {productname} 프록시 제품 리포지토리에서 사용할 수 있습니다.

. 설치하려면 다음을 실행:

+

[source, shell]
----
transactional-update pkg install helm mgrpxy
----

[[installation-proxy-containers-k3s-deploy]]
== {productname} 프록시 Helm 차트 배포

{productname} 프록시 Pod에서 사용할 볼륨의 스토리지를 구성하려면 다음 클레임에 대한 영구 볼륨을 정의합니다. 스토리지 구성을 사용자 정의하지 않으면 K3s가 자동으로 스토리지 볼륨을 생성합니다.

영구 볼륨 클레임의 이름은 다음과 같습니다.

* [literal]``squid-cache-pv-claim``
* [literal]``package-cache-pv-claim``
* [literal]``tftp-boot-pv-claim``

xref:installation-and-upgrade:container-deployment/mlm/proxy-deployment-mlm.adoc[]의 설명과 같이 {productname} 프록시에 대한 구성을 생성합니다. 구성 [literal]``tar.gz`` 파일을 복사한 다음 설치합니다.

----
mgrpxy install kubernetes /path/to/config.tar.gz
----

자세한 내용은 link:https://kubernetes.io/docs/concepts/storage/persistent-volumes/[](kubernetes) 또는 link:https://rancher.com/docs/k3s/latest/en/storage/[](K3s) 설명서를 참조하십시오.
