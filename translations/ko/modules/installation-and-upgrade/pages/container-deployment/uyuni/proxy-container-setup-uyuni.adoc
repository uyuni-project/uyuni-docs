{productname} 프록시 컨테이너에 대한 컨테이너 호스트가 준비되면 컨테이너 설정에서 구성을 완료하기 위해 몇 가지 추가 단계가 필요합니다.

.절차
. {productname} 프록시 구성 아카이브 파일 생성
. 설치 단계에서 준비한 컨테이너 호스트로 구성 아카이브를 전송하고 압축을 풉니다.
. [literal]``mgrpxy``를 실행하여 프록시 서비스를 시작합니다.

[[proxy-setup-containers-transfer-config]]
== {productname} 프록시 구성 전송


[command]``spacecmd`` 명령과 {webui} 방식을 통한 생성 모두 구성 아카이브를 생성합니다. 이 아카이브는 컨테이너 호스트에서 사용할 수 있도록 해야 합니다. 생성된 아카이브를 컨테이너 호스트로 전송합니다.



[[proxy-setup-containers-transfer-start]]
== {productname} 프록시 컨테이너 시작

컨테이너는 [literal]``mgrpxy`` 명령으로 시작할 수 있습니다.

[[proc-setup-containers-setup-start]]
.절차: {productname} 프록시 컨테이너 시작

. 다음 명령을 실행합니다.
+
----
mgrpxy start uyuni-proxy-pod
----
+
. 다음을 호출하여 모든 컨테이너가 예상대로 시작되었는지 확인하십시오.
+
----
podman ps
----

5개의 {productname} 프록시 컨테이너가 존재해야 하며, [literal]``proxy-pod`` 컨테이너 포드의 일부여야 합니다.

* proxy-salt-broker
* proxy-httpd
* proxy-tftpd
* proxy-squid
* proxy-ssh
