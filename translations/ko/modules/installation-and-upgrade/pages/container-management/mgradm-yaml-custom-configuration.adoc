= mgradm을 사용한 사용자 정의 YAML 구성 및 배포
:revdate: 2024-11-19
:page-revdate: {revdate}

배포 중에 [command]``mgradm`` 도구가 사용할 수 있는 사용자 지정 [filename]``mgradm.yaml`` 파일을 만들 수 있는 옵션이 제공됩니다.

[IMPORTANT]
====
명령줄 파라미터 또는 [filename]``mgradm.yaml`` 구성 파일을 사용하여 기본 변수를 제공하지 않은 경우 [command]``mgradm``은 기본 변수를 묻는 메시지를 표시합니다.

보안상 **명령줄 파라미터를 사용한 비밀번호 지정은 바람직하지 않습니다**. 대신 적절한 권한이 있는 구성 파일을 사용하십시오.
====

.절차: 사용자 정의 구성 파일을 사용하여 podman으로 {productname} 컨테이너 배포
. 다음 예제와 유사하게 [filename]``mgradm.yaml``이라는 이름의 구성 파일을 준비합니다.
+
[source, yaml]
....
# 데이터베이스 비밀번호. 기본적으로 무작위로 생성됨
db:
  password: MySuperSecretDBPass

# CA 인증서의 비밀번호
ssl:
  password: MySuperSecretSSLPassword

# SUSE 고객 센터 자격 증명
scc: 
  user: ccUsername
  password: ccPassword

# 조직 이름
organization: YourOrganization

# 알림을 전송하는 이메일 주소
emailFrom: notifications@example.com

# 관리자 계정 세부 정보
admin:
  password: MySuperSecretAdminPass
  login: LoginName
  firstName: Admin
  lastName: Admin
  email: email@example.com
....

. 터미널에서 루트 권한으로 다음 명령을 실행합니다. 서버의 FQDN 입력은 선택 사항입니다.
+
[source, shell]
----
mgradm -c mgradm.yaml install podman <FQDN>
----
+
[IMPORTANT]
====
컨테이너를 sudo 또는 루트로 배포해야 합니다. 이 단계를 건너뛰면 터미널에 다음 오류가 표시됩니다.

[source, shell]
----
INF Setting up uyuni network
9:58AM INF Enabling system service
9:58AM FTL Failed to open /etc/systemd/system/uyuni-server.service for writing 
error="open /etc/systemd/system/uyuni-server.service: permission denied"
----
====

. 배포가 완료될 때까지 기다립니다.

. 브라우저를 열고 서버의 FQDN 또는 IP 주소로 이동합니다.
