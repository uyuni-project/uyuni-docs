= 無人プロビジョニング
:revdate: 2024-03-14
:page-revdate: {revdate}

APIコールを使用して、MACアドレスによって識別されるクライアントと自動インストールプロファイル間の関連付けを宣言できます。 次にシステムを再起動したときに、指定したプロファイルに基づいてインストールが開始されます。


.プロシージャ: 手動で宣言したプロファイルからの再インインストール

. {productname}サーバのコマンドプロンプトで、[systemitem]``system.createSystemRecord`` APIコールを使用します。 この例では、[literal]``name``をクライアントの名前に、[literal]``<profile>``をプロファイルラベルに、[literal]``<iface>``を[literal]``eth0``などのクライアント上のインタフェース名に、[literal]``<hw_addr>``を[literal]``00:25:22:71:e7:c6``などのクライアントのハードウェアアドレスに置き換えます。
+
----
$ spacecmd api -- --args '["<name>", "<profile>", "", "", \
  [ {"name": "<iface>", "mac": "<hw_addr>"} ]]' \
  system.createSystemRecord
----
. クライアントの電源をオンにします。 ネットワークからブートすると、インストール用の正しいプロファイルが選択されます。

このコマンドによって、{cobbler}でシステムレコードが作成されます。カーネルオプション、クライアントのIPアドレス、クライアントのドメイン名など、追加のパラメータを指定することもできます。詳細については、[systemitem]``createSystemRecordコール``のAPIドキュメントを参照してください。
