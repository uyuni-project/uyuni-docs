= YAMLのカスタム設定とmgradmを使用した配備
:revdate: 2024-11-19
:page-revdate: {revdate}

カスタムの[filename]``mgradm.yaml``ファイルを作成し、配備時に[command]``mgradm``ツールでこのファイルを利用することができます。

[IMPORTANT]
====
コマンドラインパラメータまたは[filename]``mgradm.yaml``設定ファイルで基本的な変数が指定されていない場合、[command]``mgradm``によって入力を求められます。

セキュリティのため、**コマンドラインパラメータを使用してパスワードを指定するのは避けることをお勧めします**。代わりに、適切なパーミッションで設定ファイルを使用します。
====

.プロシージャ: カスタム設定ファイルを使用してPodmanで{productname}コンテナを配備する
. 次の例のような[filename]``mgradm.yaml``という名前の設定ファイルを準備します。
+
[source, yaml]
....
# データベースのパスワード。デフォルトでランダムに生成されます
db:
  password: MySuperSecretDBPass

# CA証明書のパスワード
ssl:
  password: MySuperSecretSSLPassword

# SUSEカスタマーセンターの資格情報
scc: 
  user: ccUsername
  password: ccPassword

# 組織名
organization: YourOrganization

# 通知を送信する電子メールアドレス
emailFrom: notifications@example.com

# 管理者アカウントの詳細
admin:
  password: MySuperSecretAdminPass
  login: LoginName
  firstName: Admin
  lastName: Admin
  email: email@example.com
....

. 端末からrootとして次のコマンドを実行します。サーバのFQDNの入力はオプションです。
+
[source, shell]
----
mgradm -c mgradm.yaml install podman <FQDN>
----
+
[IMPORTANT]
====
コンテナはsudoまたはrootとして配備する必要があります。このステップを省略すると、端末に次のエラーが表示されます。

[source, shell]
----
INF Setting up uyuni network
9:58AM INF Enabling system service
9:58AM FTL Failed to open /etc/systemd/system/uyuni-server.service for writing 
error="open /etc/systemd/system/uyuni-server.service: permission denied"
----
====

. 配備が完了するまで待ちます。

. ブラウザを開き、FQDNまたはIPアドレスの入力に進みます。
