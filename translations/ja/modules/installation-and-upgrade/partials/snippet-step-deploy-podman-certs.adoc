. {productname}を配備します。

+

[NOTE]
====
VMイメージを移行ターゲットとして使用する場合、最終ステップとしてここで、[command]``mgradm install``ではなく、[command]``mgradm migrate``コマンドを実行します。
====

+

SSL証明書の種類(自己署名またはサードパーティ)に応じて、以下のいずれかのコマンドを実行します。
  ``<FQDN>``を{productname}サーバの完全修飾ドメイン名に置き換えます。

+

--
* {productname}で提供される自己署名証明書を使用する場合:

+

[source, shell]
----
mgradm install podman <FQDN>
----

* サードパーティSSL証明書フラグを使用してSSL証明書をインポートする場合(これらすべての証明書が必要なわけではない場合は例を調整可能):

+

[source, shell]
----
mgradm install podman <FQDN> \
  --ssl-ca-intermediate <strings> \
  --ssl-ca-root <string> \
  --ssl-server-cert <string> \
  --ssl-server-key <string> \
  --ssl-db-ca-intermediate <strings> \
  --ssl-db-ca-root <string> \
  --ssl-db-cert <string> \
  --ssl-db-key <string>
----

+

詳細については、[command]``mgradm install podman --help``を参照してください。
