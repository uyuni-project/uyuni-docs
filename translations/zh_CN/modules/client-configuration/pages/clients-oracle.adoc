== 添加软件通道


将 {oracle} 客户端注册到您的 {productname} 服务器之前，您需要添加所需的软件通道，并同步这些通道。

当前支持的系统结构为 `x86_64` 和 `aarch64`。有关支持的产品和体系结构的完整列表，请参见 xref:client-configuration:supported-features.adoc[]。

== 创建激活密钥

您需要创建与您的 {oracle} 通道关联的激活密钥。

有关激活密钥的详细信息，请参见 xref:client-configuration:activation-keys.adoc[]。




== 管理 GPG 密钥

安装软件包之前，客户端会使用 GPG 密钥检查这些软件包的真实性。只有可信软件才能安装在客户端上。

[IMPORTANT]
====
信任 GPG 密钥对于客户端的安全非常重要。由管理员来决定需要哪些密钥，可以信任哪些密钥。这是因为如果不信任 GPG 密钥，便无法使用软件通道，是否将通道指派给客户端取决于是否信任密钥。
====

有关 GPG 密钥的详细信息，请参见 xref:client-configuration:gpg-keys.adoc[]。

[NOTE]
====
对于 {oracle} 9 或 {oracle} 8 客户端，请使用
----
ol8-gpg-pubkey-82562EA9AD986DA3.key
----

对于 {oracle} 7 客户端，请使用
----
ol67-gpg-pubkey-72F97B74EC551F0A3.key
----
====
