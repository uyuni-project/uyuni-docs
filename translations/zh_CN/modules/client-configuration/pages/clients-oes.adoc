将 {oes} 客户端注册到您的 {productname} 服务器之前，您需要添加所需的软件通道，并同步这些通道。

此过程所需的产品包括：

要查找较旧产品的通道名称，请在 {productname} 服务器上的命令提示符处以 root 身份使用 [command]``mgr-sync`` 命令：

----
mgr-sync list --help
----

然后指定您感兴趣的参数，例如 [literal]``channels``：

----
mgr-sync list channels [-c]
----

== 管理 GPG 密钥

安装软件包之前，客户端会使用 GPG 密钥检查这些软件包的真实性。只有可信软件才能安装在客户端上。

[IMPORTANT]
====
信任 GPG 密钥对于客户端的安全非常重要。由管理员来决定需要哪些密钥，可以信任哪些密钥。这是因为如果不信任 GPG 密钥，便无法使用软件通道，是否将通道指派给客户端取决于是否信任密钥。
====

有关 GPG 密钥的详细信息，请参见 xref:client-configuration:gpg-keys.adoc[]。

////
[NOTE]
====
Use the same GPG key for both {sles}{nbsp}15 and {sles}{nbsp}12 clients.
The correct key is called ``sle12-gpg-pubkey-39db7c82.key``.
====
////
