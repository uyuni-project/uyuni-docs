= 注册 {sle-micro} 客户端
:revdate: 2025-02-06
:page-revdate: {revdate}

本节包含有关注册运行 {sle-micro} 操作系统 5.1、5.2、5.3、5.4 和 5.5（{x86_64}、{arm64} 和 {zsystems} (s390x) 体系结构）的客户端的信息。

{sle-micro} 使用事务更新。事务更新是原子更新（仅当所有更新都成功时，才应用所有更新）且支持回滚。它们不会影响正在运行的系统，因为在系统重引导之前，任何更改都不会生效。

有关事务更新和重引导的详细信息，请访问 https://documentation.suse.com/sles/html/SLES-all/cha-transactional-updates.html[]。

== 注册客户端

[WARNING]
====
{sle-micro} 客户端在注册后需要重引导。

Following the bootstrapping process, automatic booting is disabled on {sle-micro} clients. This change was implemented due to intermittent automatic reboots, which could occur before {salt} could relay the results of applying the bootstrap {salt} state.

尽管系统在完成注册后会自动安排重引导，但会采用默认的重引导管理器维护时段。此时段在客户端注册后可能会持续几个小时。建议在注册脚本完成后手动重引导客户端，以加速注册过程并使系统显示在系统列表中。
====

----

----

[[clients-slemicro-reboot]]
== 重引导 {sle-micro}

{sle-micro} 是一种事务系统。事务更新一般支持数种重引导方法。建议在 {productname} 所管理的系统中使用 [systemitem]``systemd`` 来进行重引导。使用其他方法可能会导致系统出现意外行为。

如果 {productname} 上的事务系统使用的是默认配置，引导该系统时，[systemitem]``systemd`` 将会配置为引导方法 ([literal]``REBOOT_METHOD``)。该默认配置允许 {productname} 控制重引导操作，重引导可以立即执行或使用 {productname} 根据需要安排重引导时间。


=== 背景信息

默认情况下，在安装客户端的过程中会将重引导方法设置为 [option]``auto``。采用 [option]``auto`` 引导方法时，如果 [command]``rebootmgrd`` 正在运行，将使用该服务根据配置的策略来重引导系统。策略可能设置为立即重引导或在维护时段重引导。有关详细信息，请参见 [command]``rebootmgrd``(8) 手册页。如果 [command]``rebootmgrd`` 没有运行，{productname} 将会调用 [command]``systemctl reboot``。

[NOTE]
====
使用 [command]``systemd`` 之外的任何其他方法都可能会导致系统出现意外行为。
====
