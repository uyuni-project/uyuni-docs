= 任务日程安排
:revdate: 2025-07-03
:page-revdate: {revdate}


menu:管理[任务日程安排]下会列出所有预定义的任务组。

image::admin_task_schedules.png[scaledwidth=80%]


单击 menu:{scc} 日程安排[日程安排名称]打开menu:日程安排名称[日程安排基本细节]，在其中可以禁用该日程安排或更改其频率。

单击 btn:[编辑日程安排] 可使用您的设置更新该日程安排。

要禁用日程安排，请单击右上角的 btn:[禁用日程安排]。

[WARNING]
====
由于日程安排对于 {productname} 正常运行至关重要，因此请仅在绝对必要的情况下才禁用日程安排。
====

任务被禁用后仍会显示在列表中。单击menu:{productname} 日程安排[日程安排名称]后，可通过单击 btn:[激活日程安排] 重新激活该作业。


如果您单击某个组名称，将显示该组类型的运行列表和运行状态。

单击开始时间链接会返回menu:日程安排名称[日程安排基本细节]。



== 预定义的任务组


系统默认会安排以下预定义的任务组，您可对其进行配置：

menu:auto-errata-default[]::
根据需要安排自动勘误更新。

menu:channel-repodata-default[]::
（重新）生成储存库元数据文件。

menu:cleanup-data-default[]::
从数据库中清理已过时的软件包更改日志和监控时序数据。

menu:clear-taskologs-default[]::
根据作业类型，从数据库中清除超过指定天数的任务引擎 (taskomatic) 历史数据。

menu:cobbler-sync-default[]::
将 {productname} 中的分布数据和配置文件数据同步到 {cobbler}。有关由 {cobbler} 提供支持的自动安装的详细信息，请参见 xref:client-configuration:autoinst-intro.adoc[]。

menu:compare-configs-default[]::
将存储在配置通道中的配置文件与存储在所有已启用配置的服务器上的文件进行比较。要查看比较结果，请单击menu:系统[]选项卡并选择相关系统。转到menu:配置[比较文件]。有关详细信息，请参见 xref:reference:systems/system-details/sd-configuration.adoc#sd-config-compare-files[]。

menu:cve-server-channels-default[]::
更新用于在menu:审计[CVE 审计]页面上显示结果的内部预计算 CVE 数据。menu:审计[CVE 审计]页面中的搜索结果将根据此日程安排的上次运行情况更新。有关详细信息，请参见 xref:reference:audit/audit-cve-audit.adoc[]。

menu:daily-status-default[]::
将每日报告电子邮件发送到相关地址。有关如何为特定用户配置通知的详细信息，请参见 xref:reference:users/user-details.adoc[]。

menu:errata-advisory-map-sync-default[]::
更新内部的 {suse} 补丁供应商建议数据库表。如果有相关内容，每个补丁详细信息的“供应商建议”部分会显示 {suse} 提供的原始建议。

menu:errata-cache-default[]::
更新内部补丁缓存数据库表，这些表用于查找每个服务器的需要更新的软件包。此任务组还向可能对特定补丁感兴趣的用户发送通知电子邮件。有关补丁的详细信息，请参见 xref:reference:patches/patches-menu.adoc[]。

menu:errata-queue-default[]::
为配置为接收自动更新（补丁）的服务器将这些更新（补丁）排队。

menu:gatherer-matcher-default[]::
通过运行在虚拟主机管理器中配置的虚拟主机收集器，来收集虚拟主机数据。当有更新的数据后，订阅匹配器作业即会运行。

menu:kickstart-cleanup-default[]::
清理过时的 {kickstart} 会话数据。

menu:kickstartfile-sync-default[]::
生成与配置向导创建的 {kickstart} 配置文件对应的 {cobbler} 文件。

menu:mgr-forward-registration-default[]::
将客户端注册数据与 {scclongform} 同步。默认会转发新的、已更改的或已删除的客户端数据。要禁用 [path]``/etc/rhn/rhn.conf`` 中设置的同步，请运行：
+
----
server.susemanager.forward_registration = 0
----
+

[WARNING]
====
禁用与 {scc} 的数据同步将导致 RMT、SMT、{productname} 以及直接注册到 {scc} 的客户端之间，对受管客户端的可见性降低。

通过同步数据，您可以确保所有注册客户端具有统一的视图。

https://suselinux.fra1.qualtrics.com/jfe/form/SV_0ooNnrY0rYuQScS[告知我们您选择停用的原因，从而帮助改进我们的服务]
====

menu:mgr-sync-refresh-default[]::
与 {scclongform} 保持同步 (``mgr-sync-refresh``)。默认情况下，在此任务执行过程中也会同步所有自定义通道。有关自定义通道同步的详细信息，请参见 xref:administration:custom-channels.adoc#_custom_channel_synchronization[]。

menu:minion-action-chain-cleanup-default[]::
清理过时的操作链数据。

menu:minion-action-cleanup-default[]::
从文件系统中删除过时的客户端操作数据。首先，此任务组会尝试通过查找存储在 {salt} 作业缓存中的相应结果，来完成任何可能未完成的操作。如果服务器遗漏了操作结果，就可能发生操作未完成的情况。对于成功完成的操作，此任务组会去除已执行的脚本文件等项目。

menu:minion-checkin-default[]::
在客户端上执行常规的签入。

menu:notifications-cleanup-default[]::
清理失效的通知消息。

menu:oval-data-sync-default[]::
生成所需的 OVAL 数据，以提高 CVE 审计查询的准确度。

menu:package-cleanup-default[]::
从文件系统中删除过时的软件包文件。

menu:reboot-action-cleanup-default[]::
任何超过六个小时未处理的重引导操作都将标记为失败，数据库中的关联数据将被清理。有关安排重引导操作的详细信息，请参见 xref:reference:systems/system-details/sd-provisioning.adoc#sd-power-management[]。

menu:session-cleanup-default[]::
清理过时的 Web 界面会话，这通常是用户登录之后，在注销之前关闭浏览器时临时存储的数据。

menu:ssh-service-default[]::
如果为客户端配置了 ``SSH 推送``联系方法，则提示客户端通过 SSH 来与 {productname} 通信。另外，在重引导后继续执行操作链。

menu:system-overview-update-queue-default[]::
更新系统概览数据。

menu:system-profile-refresh-default[]::
在所有系统上执行硬件刷新。此任务组每月只执行一次，可能会增加 {productname} 服务器上的负荷。该作业会使用 xref:specialized-guides:salt/salt-rate-limiting.adoc[Salt 率限制]。如果要调整批次大小，请参见 xref:specialized-guides:large-deployments/tuning.adoc#java-salt-batch-size[]。

menu:token-cleanup-default[]::
删除 {salt} 客户端用来下载软件包和元数据的已失效储存库令牌。

menu:update-payg-default[]::
从配置的 {payg} 云实例收集身份验证数据。

menu:update-reporting-default[]::
更新本地报告数据库。

menu:update-reporting-hub-default[]::
从外围 {productname} 服务器收集所有报告数据，并更新中心报告数据库。

menu:update-system-overview-default[]::
定期检查系统概览数据是否保持最新。

menu:uuid-cleanup-default[]::
清理过时的 UUID 记录。
