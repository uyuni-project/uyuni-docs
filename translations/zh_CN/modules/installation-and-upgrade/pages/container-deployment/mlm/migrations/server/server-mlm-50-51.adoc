迁移过程中，会从源服务器复制服务器 SSL 证书和 CA 链，这意味着您只需准备数据库证书即可

[NOTE]
====
{productname} 5.0 peripheral servers are always using third-party SSL certificates. If the hub server has generated the certificates for the peripheral server, it needs to generate the certificate for the peripheral database too.

On the hub server, run the following command for each of the peripheral server to migrate.

[source, shell]
----
mgrctl exec -ti -- rhn-ssl-tool --gen-server --dir="/root/ssl-build" --set-country="COUNTRY" \
  --set-state="STATE" --set-city="CITY" --set-org="ORGANIZATION" \
  --set-org-unit="ORGANIZATION UNIT" --set-email="name@example.com" \
  --set-hostname=<hostname>-reportdb --set-cname="example.com" --set-cname=db --set-cname=reportdb
----

要使用的文件位于服务器容器内，需要复制到新的外围服务器主机上：

. [literal]``/root/ssl-build/RHN-ORG-TRUSTED-SSL-CERT`` as the root CA

. [literal]``/root/ssl-build/<hostname>-reportdb/server.crt`` as the db certificate

. [literal]``/root/ssl-build/<hostname>-reportdb/server.key`` as the db certificate's key.
====


== 客户端工具品牌更新

{productname} {productnumber} 为所有受支持的操作系统推出了一套重新命名的客户端工具。此过渡过程无缝衔接，执行新产品同步的用户仅会注意到通道名称已更新。

此前注册到 {productname} 4.3 或 5.0 的客户端所使用的 [literal]``SUSE Manager Client Tools for XYZ`` 通道在版本 5.1 中已不再提供，且不会在 5.1 中继续接收更新。

尽管迁移后旧版通道仍会分配给现有客户端，但对应的储存库已被去除。

为确保客户端能持续接收更新，用户必须执行以下操作：

* 为相关产品同步新的 [literal]``SUSE Multi-Linux Manager Client Tools for XYZ`` 通道，并将其分配给相应的客户端。
* 取消分配过时的 [literal]``SUSE Manager Client Tools for XYZ`` 通道。

这也意味着，任何基于旧客户端工具的 CLM 项目都必须相应调整。

示例工作流程请参见 xref:common-workflows:workflow-switch-to-new-client-tools-channels.adoc[]。



== 从 {sle-micro} 5.5 迁移到 {sl-micro} 6.1



本文提供了经过测试的步骤，用于将部署了 {productname} 5.0 服务器的 {sle-micro} 5.5 主机升级到 {sl-micro} 6.1 并迁移到 {productname} {productnumber}。


=== 先决条件

[IMPORTANT]
====
{sl-micro} 6.1 uses NetworkManager as the network managing service. If your server is using wicked as the network manager it **must** be migrated to NetworkManager before upgrading the Operating System. For more information on migrating to NetworkManager see: https://documentation.suse.com/sle-micro.

* {productname} 5.0 已在 {sle-micro} 5.5 上安装并运行。

* 系统已注册到 {scc} 且拥有有效的订阅。
* NetworkManager as the network managing service

====


=== 发行套件升级和服务器迁移

.过程：从 {productname} 5.0 迁移到 {productname} {productnumber}
[role=procedure]
____

. 校验当前产品状态。

+

[source, console]
----
SUSEConnect --status-text
----

+

确认以下信息：

* 基础操作系统：`SUSE Linux Enterprise Micro 5.5`

* 扩展：`SUSE Manager Server 5.0 Extension`

+

. 确保系统已更新。

+

[source, console]
----
transactional-update patch
----

+

--

* 如果已应用补丁，请在继续迁移前停止服务器，然后重引导系统：

+

[source, console]
----
mgradm stop
reboot
----

* 如果未发现更新，可直接进入迁移步骤。

--

+

. 启动迁移流程。

+

[source, console]
----
transactional-update migration --auto-agree-with-licenses --gpg-auto-import-keys
----

+

按照提示操作，选择可用的迁移目标：**SUSE Linux Micro 6.1** 和 **{productname} Server Extension 5.1**。

+

. 停止服务器后重引导，使更改生效。

+

[source, console]
----
mgradm stop
reboot
----

+

. 执行重引导后的检查。

+

校验升级后的操作系统和 {productname} 扩展：

+

[source, console]
----
cat /etc/os-release
SUSEConnect --status-text
----

+

您应看到以下内容：

* [literal]``PRETTY_NAME="SUSE Linux Micro 6.1"``

* [literal]``SUSE Multi-Linux Manager Server 5.1 Extension``

+

. Enable root SSH access (_if required_).
+
SL Micro 6.1 disables root login via SSH by default.
+
Edit `/etc/ssh/sshd_config.d/sshd.conf`:

+

[source, console]
----
PermitRootLogin yes
----

+

并重启服务：

+

[source, shell]
----
systemctl restart sshd
----

+

有关详细信息，请参见 xref:administration:troubleshooting/tshoot-remote-root-on-micro.adoc[]。

+

. 校验 {productname} 工具

+

[source, console]
----
mgradm --version
----

+

预期输出：

+

* 版本 `5.1.11` 或更高版本

* 引用版本 [literal]``5.1.0`` 或更高版本

. 升级服务器容器。

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source, console]
----
mgradm start
mgradm upgrade podman
----

+

按照提示拉取并配置新的 5.1.0 容器。


. 检查正在运行的容器：

+

[source, console]
----
podman ps
----

+

您应看到以下内容：

* [literal]``server:5.1.0`` 或更高版本

* [literal]``server-postgresql:5.1.0`` 或更高版本

____


[NOTE]
====
如果之前未安装 [literal]``uyuni-db`` 或 [litarl]``saline`` 等服务，升级过程中出现的相关缺失服务错误可忽略。
====


=== 迁移完成

系统现已在 SL Micro 6.1 上运行 {productname} 5.1。请在恢复生产操作前验证设置。如果您有 {productname} 5.0 代理连接至该服务器，请继续参考 xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-51.adoc[] 指南升级代理主机。

== 从 {sles} 15 SP6 升级到 15 SP7

本文提供了经过测试的步骤，用于将部署了 SUSE Multi-Linux Manager 5.0 服务器的 SUSE Linux Enterprise Server 15 SP6 主机升级到部署了 SUSE Multi-Linux Manager 5.1 的 SUSE Linux Enterprise Server 15 SP7。


=== 先决条件

* SUSE Multi-Linux Manager 5.0 已在 SUSE Linux Enterprise Server 15 SP6 上安装并运行。
* 系统已注册到 SUSE Customer Center (SCC) 且拥有有效的订阅。
* 继续操作前请务必创建备份。



=== 发行套件升级和服务器迁移

.过程：从 {productname} 5.0 迁移到 {productname} {productnumber}
[role=procedure]
____

. 校验当前产品状态。

+

[source, console]
----
SUSEConnect --status-text
----

+

确认以下信息：

+

* 基础操作系统：[literal]``SUSE Linux Enterprise Server 15 SP6``

* 扩展：[literal]``SUSE Manager Server 5.0 Extension``

+

. 应用所有系统补丁。

+

[source, console]
----
zypper patch
----

+

. 如果更新了更新栈，则停止服务器后重引导。

+

[source, console]
----
mgradm stop
reboot
----

+

. 启动 Zypper 迁移工具。

+

[source, console]
----
zypper migration
----

+

Zypper 会显示可能的迁移目标以及详细摘要。

+

. 选择适当的目标，并按照提示完成迁移。

. 完成迁移后，停止服务器，然后重引导系统。

+

[source, console]
----
mgradm stop
reboot
----

+

. 执行重引导后的检查：

+

校验升级后的操作系统和 {productname} 扩展。
+

[source, console]
----
cat /etc/os-release
SUSEConnect --status-text
----

+

您应看到以下内容：

* [literal]``VERSION="15-SP7"``

* [literal]``SUSE Multi-Linux Manager Server 5.1 Extension for SLE``

+

. 校验 {productname} 工具版本。

+

[source, console]
----
mgradm --version
----

+

预期输出：

* 版本 `5.1.11` 或更高版本

* 映像标记 [literal]``5.1.0`` 或更高版本

+

. 升级服务器容器。

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source, console]
----
mgradm start
mgradm upgrade podman
----

+

按照提示拉取新的容器映像并重新配置环境。

+

. 检查正在运行的容器。

+

[source, console]
----
podman ps
----

+

预期容器：

* [literal]``server:5.1.0`` 或更高版本

* [literal]``server-postgresql:5.1.0`` 或更高版本

____


=== 迁移完成

系统现已在 SUSE Linux Enterprise Server 15 SP7 上运行 {productname} 5.1。请在恢复生产操作前验证设置。如果您有 {productname} 5.0 代理连接至该服务器，请继续按照 xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-51.adoc[] 操作。
