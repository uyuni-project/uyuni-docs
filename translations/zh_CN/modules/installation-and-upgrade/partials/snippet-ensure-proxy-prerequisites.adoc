== 安装软件包并启用 podman

使用代理前，需要在主机上安装一些软件包，且 [package]``podman`` 需处于运行状态。


.过程：准备前置条件
[role=procedure]
_____

. 在代理主机上，确保已安装以下软件包：

* [package]``podman``
* [package]``mgrpxy-bash-completion``
* [package]``suse-multi-linux-manager-<version>-<arch>-proxy-httpd-image``
* [package]``suse-multi-linux-manager-<version>-<arch>-proxy-salt-broker-image``
* [package]``suse-multi-linux-manager-<version>-<arch>-proxy-squid-image``
* [package]``suse-multi-linux-manager-<version>-<arch>-proxy-ssh-image``
* [package]``suse-multi-linux-manager-<version>-<arch>-proxy-tftpd-image``

+

For example, for version 5.1 and architecture of x86_64, the package name would be [package]``suse-multi-linux-manager-5.1-x86_64-proxy-httpd-image``.

+

. 重引导系统或运行以下命令，启动代理主机上的 Podman 服务：

+

[source, shell]
----
systemctl enable --now podman.service
----

. 在代理主机上使用以下命令安装代理：

+

----
mgrpxy install podman config.tar.gz
----
