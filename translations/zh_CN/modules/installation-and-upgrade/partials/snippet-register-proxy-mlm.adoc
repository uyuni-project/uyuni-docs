== 注册 {sl-micro} 和 {productname} {productnumber} 代理

ifeval::[{uyuni-content} == true]
:noindex:
endif::[]
// Before starting obtain your SUSE Manager Registration Code from SUSE Customer Center - https://scc.suse.com.

.过程：注册 {sl-micro} 和 {productname} {productnumber} 代理
. 引导虚拟机。
. 以 `root` 身份登录。
. 在 SCC 中注册 {sl-micro}。
+

----
transactional-update register -r <注册代码> -e <您的电子邮件地址>
----

. 重引导。

. 在 {scclongform} 中注册 {productname} {productnumber} 代理。
+

----
transactional-update register -p Multi-Linux-Manager-Proxy/5.1/x86_64 -r <注册代码>
----

. 重引导。

. 更新系统：
+

----
transactional-update
----

. 如果已应用更新，请重引导。

. 此步骤是可选的。但是，如果您的基础架构需要自定义的永久性存储，请使用 [command]``mgr-storage-proxy`` 工具。
有关详细信息，请参见 [command]``mgr-storage-proxy --help``。此工具可以简化容器卷的创建过程。

** 如下所示使用命令：
+

----
mgr-storage-proxy <存储磁盘设备>
----
+
例如：
+
----
mgr-storage-proxy /dev/nvme1n1
----
+
[NOTE]
====
此命令会将 [path]``/var/lib/containers/storage/volumes`` 中的永久性存储卷移动到指定的存储设备。

有关详细信息，请参见

* xref:installation-and-upgrade:container-management/persistent-container-volumes.adoc[]
* xref:administration:troubleshooting/tshoot-container-full-disk.adoc[]
====
